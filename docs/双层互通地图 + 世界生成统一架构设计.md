# 2D 工厂游戏“双层互通地图 + 世界生成”统一架构设计文档

---

# 一、文档目标

本文档用于统一：

* 双层地图系统
* Chunk 世界生成系统
* 世界数据架构
* 地表 / 地下互通逻辑
* Godot 渲染层
* C++ Simulation Core

目标是构建一个：

```text
“真正立体化”
```

的 2D 工厂 / 探索游戏世界。

最终效果参考：

* 《奈斯启示录》
* 《Core Keeper》
* 《Terraria》
* 《Factorio》
* 《Mindustry》

但更强调：

```text
上下层互相影响
```

而不是：

```text
独立地图切换
```

---

# 二、核心设计思想

---

# 1. 世界不是“多个地图”

而是：

```text
同一个世界的不同高度层
```

因此：

* 地表
* 地下
* 遗迹
* 建筑内部
* 深层矿井

本质上都属于：

```text
同一个 WorldData
```

---

# 2. 统一二维坐标系

所有层：

共享：

```cpp
Vector2i world_cell
```

区别仅在于：

```cpp
LayerId layer_id
```

例如：

```text
surface      (12, 8)
underground  (12, 8)
```

代表：

```text
同一个世界位置的不同高度
```

这是整个双层设计最关键的核心。

---

# 3. Layer 是世界维度，不是场景

禁止：

```text
SurfaceScene
UndergroundScene
```

互相独立。

正确设计：

```text
WorldData
 ├ Layer(surface)
 ├ Layer(underground)
 ├ Layer(ruins)
 └ Layer(factory_depth_1)
```

Layer：

只是：

```text
世界高度维度
```

---

# 4. Chunk 是 Layer 内部分片

最终结构：

```text
World
 ├ Layer
 │   ├ Chunk
 │   ├ Chunk
 │   └ Chunk
 │
 ├ Layer
 │   ├ Chunk
 │   ├ Chunk
 │   └ Chunk
```

即：

```text
Layer + Chunk
```

双维管理。

# 四、核心数据结构

---

# LayerId

```cpp
using LayerId = StringName;
```

示例：

```text
surface
underground
ruins
facility_depth_1
```

---

# WorldPosition

```cpp
struct WorldPosition {
    LayerId layer;
    Vector2i cell;
};
```

这是：

```text
世界唯一位置标识
```

---

# ChunkCoord

```cpp
struct ChunkCoord {
    int x;
    int y;
};
```

---

# ChunkKey

```cpp
struct ChunkKey {
    LayerId layer;
    ChunkCoord coord;
};
```

用于：

```text
唯一标识 Chunk
```

---

# TerrainCell

```cpp
struct TerrainCell {
    TerrainMaterial material;
    uint32_t flags;
};
```

---

# TerrainMaterial

禁止：

```cpp
WALL
FLOOR
```

这种渲染语义。

改为：

```cpp
enum class TerrainMaterial {
    AIR,

    DIRT,
    STONE,
    SAND,
    WATER,
    LAVA,

    ORE_IRON,
    ORE_COPPER,
    ORE_COAL,

    RUINS_FLOOR,
    RUINS_WALL,

    METAL_FLOOR,
    METAL_WALL
};
```

---

# TerrainFlags

```cpp
enum TerrainFlags {
    WALKABLE = 1 << 0,
    SOLID    = 1 << 1,
    LIQUID   = 1 << 2,
    MINEABLE = 1 << 3,
    BLOCKS_LIGHT = 1 << 4
};
```

# TerrainChunkManager 职责

负责：

```text
1. 玩家当前位置
2. 当前 Layer
3. 当前可见 Chunk
4. 当前可见 Layer
5. Chunk 加载卸载
6. Layer 切换
```

---

# Chunk 加载规则

例如：

```text
当前 Layer = underground
```

则：

```text
优先加载 underground 周围 Chunk
```

同时：

```text
可选低精度加载 surface
```

用于：

* 光照
* 小地图
* 地表影响
* 世界联动

# 六、双层世界生成系统

---

# 核心思想

地表和地下：

```text
不能独立随机
```

必须：

```text
共享世界生成 Seed
```

并互相关联。

---

# 世界生成结构

```text
WorldGenerator
│
├── SurfaceGenerator
├── UndergroundGenerator
├── ConnectorGenerator
├── StructureGenerator
└── RegionGenerator
```

---

# ConnectorGenerator（关键）

负责生成：

```text
洞口
裂缝
井
楼梯
遗迹入口
矿井入口
电梯
```

这是：

```text
地表和地下互通的核心
```

---

# 连接点生成规则

例如：

```text
地表山脉区域
    ↓
地下生成大型洞穴
    ↓
在边缘生成洞口
```

或者：

```text
地表遗迹
    ↓
地下对应古代设施
```

---

# 推荐生成流程

```text
1. Generate Surface
2. Generate Underground
3. Analyze Shared Regions
4. Generate Connectors
5. Generate Structures
6. Generate Gameplay Objects
```

# 九、玩法联动设计

---

# 核心原则

双层地图：

最重要的是：

```text
层与层之间互相影响
```

而不是：

```text
只是上下切换
```

---

# 推荐联动

---

# 地形联动

```text
地表河流
    ↓
地下暗河

地表山脉
    ↓
地下大型矿脉
```

---

# 路径联动

```text
地表无法通过
    ↓
地下绕路

地下塌方
    ↓
地表出现裂缝
```

---

# 机关联动

```text
地下启动机器
    ↓
地表桥梁升起

地表祭坛激活
    ↓
地下封门开启
```

---

# 污染联动

```text
地下污染扩散
    ↓
地表植物枯萎
```

---

# 工厂联动（非常重要）

后期：

```text
地表：
生产区

地下：
能源区
矿井
流体区
危险设施
```

这会形成：

```text
真正立体工厂
```
