# 种植系统设计（分时代融合递进框架）

> 方向五：分时代融合递进。种植系统随时代递进，每个时代引入一层新机制，
> 最终汇聚工业 + 生态 + 魔法 + 基因四条线。最契合"科学与神学"双主题。

---

## 0. 设计总览

### 0.1 时代递进路线

```
石器时代          青铜/蒸汽         LV/MV            HV/EV            源律时代
   │                │                │                │                │
   ▼                ▼                ▼                ▼                ▼
[Tier 1 手工]    [Tier 2 简机]    [Tier 3 自动]   [Tier 4 温室基因] [Tier 5 源质]
  锄地播种         蒸汽洒水器       电动播种/收割     气候控制温室      五行源质植物
  骨粉催熟         堆肥机           农田机器人        基因测序仪        仪式催化
  轮作伴生         灌溉水道         AE2 闭环          育种杂交          炼金杂交
  野生采集         基础加工         加工链条          性状稳定          源质害兽
```

### 0.2 与现有系统的关系

| 现有系统 | 种植系统接入点 |
|---|---|
| `TreeGrowthSystem`（优先级 8） | 已实现树木生长；Tier 1 复用其模式新建 `CropGrowthSystem`（优先级 9） |
| `BlockEntityRegistry` | 新增 `FARMLAND` / `CROP` 实体类型，承载农田湿度/肥力与作物生长状态 |
| `EcosystemSystem` | 复用 `soil_fertility` / `water_availability` 调制作物生长速率 |
| `SeasonSystem` | 作物有 `plant_season` / `harvest_season`，反季节减速/失败 |
| `GDGameCommandServer` | 新增 `cmd_till_farmland` / `cmd_plant_crop` / `cmd_harvest_crop` / `cmd_fertilize` |
| `GDTerrainContentRegistry` | 新增 `register_crop_species()` + `register_farmland_material()` |
| `MachineSystem` + `RecipeDatabase` | Tier 2+ 新增 PLANTER / HARVESTER / SPRINKLER / FERTILIZER 机器类型 |
| `FluidNetwork` | Tier 2+ 灌溉水道、洒水器接入水流体网络 |
| `ItemPipeNetwork` + AE2 | Tier 3 自动化闭环 |
| `RuneRegistry` / `RitualRecipeRegistry` | Tier 5 仪式催化作物生长 |
| `QuestDatabase` | 每个时代对应种植任务线 |

### 0.3 核心设计原则

1. **数据驱动**：作物物种、农田材质、配方全部在 GDScript 注册，C++ 冻结快照后消费
2. **命令权威**：所有玩法写入经 `GDGameCommandServer`，客户端只提交命令
3. **模拟解耦**：生长逻辑在 `CropGrowthSystem`，跨系统数据通过 `TickContext` 只读
4. **休眠不生长**：作物只在活跃 chunk 生长（与 `TreeGrowthSystem` 一致）
5. **渐进复杂度**：Tier 1 无需电力/机器，前期体验平滑；后续时代叠加机制
6. **三档 tick 策略**：按植物类型选择不同 tick 模式，平衡性能与确定性

### 0.4 三档 tick 策略（植物分类调度）

不同植物类型对"确定性"和"数量"的需求不同，采用三种 tick 模式：

| 植物类型 | tick 策略 | 调度方式 | 适用场景 | 性能特征 |
|---|---|---|---|---|
| **普通植物** | 随机刻 | 每 tick 在活跃 chunk 内随机抽取 N 个方块处理 | 野草、花、装饰植物、野生作物扩散/枯萎 | 低成本、不精确、适合大量低价值方块 |
| **农作物/工业植物** | 计划刻 | 遍历所有 CROP 实体，确定性推进生长阶段 | 小麦、棉花、胡萝卜等玩家种植作物 | 可靠生长、中等成本、数量可控 |
| **特殊植物** | 实体刻 | 每个实体独立 tick（类似机器/生物），自带状态机 | 源质植物（Tier 5）、魔法作物、杂交种 | 高成本/实体、数量稀少、逻辑复杂 |

**实现映射**：

```
普通植物  →  EcosystemSystem / LightPlantSystem（随机刻）
             每 tick 在活跃 chunk 随机选 64 个方块，处理野草扩散/枯萎/季节变色
             不创建 BlockEntity，仅读写地形材质

农作物    →  CropGrowthSystem（计划刻，优先级 9）  ← Tier 1 当前实现
             遍历所有 CROP BlockEntity，按生长计时推进阶段
             数量受玩家种植行为控制，天然有上限

特殊植物  →  SpecialPlantSystem（实体刻，优先级 10）  ← Tier 5 实现
             每个源质植物实体独立状态机：吸收法力→生长→产出→吸引害兽
             类似 MachineSystem 的 per-entity tick 模式
```

**与现有系统的关系**：
- `TreeGrowthSystem`（优先级 8）本质是"实体刻"——遍历 TREE BlockEntity，因为树木数量中等且需精确生长
- `EcosystemSystem`（优先级 7）已含"随机刻"思想——per-chunk Lotka-Volterra 种群演化
- `MachineSystem`（优先级 2）是"实体刻"——per-machine tick

**Tier 1 落地**：
- 农作物 → `CropGrowthSystem`（计划刻）✓
- 野生作物 → 世界生成时直接放置为 MATURE 阶段，无需 tick（玩家采集获得种子）
- 普通野草/花 → 暂不实现独立随机刻系统，由 `EcosystemSystem.vegetation_density` 抽象表达

## 1. 核心数据结构（C++ 核心层）

### 1.1 作物物种定义 `CropSpeciesDef`

文件：`src/core/world_gen/crop_species_def.hpp`

```cpp
#pragma once
#include <cstdint>
#include <string>
#include <vector>

namespace science_and_theology {

// 作物生长阶段。草本作物通常 4 阶段（种子→幼苗→生长→成熟）。
enum class CropGrowthStage : uint8_t {
    SEED        = 0,   // 刚播种，未发芽
    SPROUT      = 1,   // 发芽
    GROWING     = 2,   // 生长中
    MATURE      = 3,   // 成熟，可收获
    COUNT       = 4,
};

constexpr const char* kCropGrowthStageNames[] = {
    "Seed", "Sprout", "Growing", "Mature",
};

// 作物类别：决定加工链与任务归属。
enum class CropCategory : uint8_t {
    GRAIN       = 0,   // 谷物（小麦、水稻）→ 面粉/淀粉
    ROOT        = 1,   // 根茎（胡萝卜、马铃薯）→ 直接食用/淀粉
    FIBER       = 2,   // 纤维（棉花、亚麻）→ 纤维/布
    HERB        = 3,   // 药草（薄荷、甘菊）→ 药剂
    FRUIT       = 4,   // 果实（南瓜、浆果）→ 食用/发酵
    MAGIC       = 5,   // 源质植物（Tier 5）
    COUNT       = 6,
};

// 季节枚举（与 SeasonSystem 一致）。
// 0=Spring, 1=Summer, 2=Autumn, 3=Winter, -1=Any
using SeasonId = int;

struct CropSpeciesDef {
    // 唯一物种 key，如 "wheat", "carrot", "cotton"。
    std::string species_key;
    // 翻译 key，如 "crop.wheat"。
    std::string title_key;

    // --- 类别 ---
    CropCategory category = CropCategory::GRAIN;

    // --- 生物群系约束（与 TreeSpeciesDef 一致）---
    float temperature_min = -1.0f;
    float temperature_max = 1.0f;
    float humidity_min = -1.0f;
    float humidity_max = 1.0f;

    // --- 季节约束 ---
    // 适合播种的季节（-1 = 任意季节）。
    SeasonId plant_season = -1;
    // 适合生长的季节（-1 = 任意季节）。反季节生长速率 ×0.3。
    SeasonId grow_season = -1;
    // 适合收获的季节（-1 = 任意季节）。
    SeasonId harvest_season = -1;

    // --- 生长 ---
    // 各阶段转换所需 ticks（受生态系统肥力/水分调制）。
    int64_t ticks_seed_to_sprout   = 3000;   // 2.5 分钟
    int64_t ticks_sprout_to_growing = 6000;   // 5 分钟
    int64_t ticks_growing_to_mature = 9000;   // 7.5 分钟

    // --- 物品产出 ---
    // 种子物品 key（播种消耗），如 "seed.wheat"。
    std::string seed_item_key;
    // 收获主产物物品 key，如 "crop.wheat"。
    std::string crop_item_key;
    // 收获副产物物品 key（可选，如小麦的种子），空字符串表示无。
    std::string byproduct_item_key;
    // 主产物最小/最大数量。
    int crop_min = 1;
    int crop_max = 2;
    // 副产物数量（固定）。
    int byproduct_count = 1;
    // 成熟后是否可多次收获（如浆果丛）。false = 收获后消失。
    bool repeat_harvest = false;
    // 多次收获作物的再生 ticks（repeat_harvest=true 时有效）。
    int64_t regrow_ticks = 6000;

    // --- 地形材质 ---
    // 各生长阶段对应的地形材质 key（4 个阶段）。
    // 如 wheat: ["snt:wheat_seed", "snt:wheat_sprout",
    //            "snt:wheat_growing", "snt:wheat_mature"]
    std::string stage_material_keys[4];

    // --- 肥力/水分敏感度 ---
    // 0.0 = 不敏感（杂草级），1.0 = 极敏感（贫瘠地绝收）。
    float fertility_sensitivity = 0.7f;
    float water_sensitivity = 0.7f;

    // --- 野生生成（Tier 1 野外采集）---
    // 是否在世界生成时作为野生作物放置。
    bool wild_spawn = false;
    // 野生生成密度权重（相对其他野生作物）。
    float wild_density_weight = 1.0f;

    // --- 视觉颜色（图标/渲染回退）---
    float crop_color_r = 0.85f;
    float crop_color_g = 0.75f;
    float crop_color_b = 0.25f;
};

} // namespace science_and_theology
```

### 1.5 `CropGrowthSystem` 模拟系统

文件：`src/core/simulation/crop_growth_system.hpp` / `.cpp`

```cpp
class CropGrowthSystem : public SimulationSystem {
public:
    CropGrowthSystem() = default;
    SIMULATION_SYSTEM_NAME(CropGrowthSystem, "CropGrowthSystem")

    void initialize(WorldData* world, EventBus* bus) override;
    void tick_active(const ChunkKey& chunk, float delta,
                     const TickContext* ctx = nullptr) override;
    void tick_sleeping(const ChunkKey& chunk, float delta,
                       const TickContext* ctx = nullptr) override;
    void shutdown() override;

    // 优先级 9：在 TreeGrowth(8) 之后运行。
    int priority() const override { return 9; }

    static constexpr int kMaxGrowthPerTick = 64;

private:
    bool try_grow_crop(EntityId entity_id,
                       const std::string& dimension_id,
                       int64_t current_tick,
                       const TickContext* ctx);

    bool check_growth_conditions(
        const CropSpeciesDef& species,
        const std::string& dimension_id,
        int root_x, int root_y, int root_z,
        CropGrowthStage current_stage,
        const TickContext* ctx) const;

    void advance_crop_stage(
        EntityId entity_id,
        const CropSpeciesDef& species,
        const std::string& dimension_id,
        int root_x, int root_y, int root_z,
        CropGrowthStage new_stage,
        int64_t current_tick);

    // 更新农田湿度（蒸发/降雨）。
    void update_farmland_moisture(
        const ChunkKey& chunk, int64_t current_tick,
        const TickContext* ctx);

    bool set_world_cell(
        const std::string& dimension_id,
        int world_x, int world_y, int world_z,
        TerrainMaterialId material, uint32_t flags);

    int growth_count_ = 0;
};
```

**生长逻辑**（`tick_active`）：
1. 先遍历 chunk 内 `FARMLAND` 实体，更新湿度（下雨 +0.05/tick，蒸发 -0.001/tick）
2. 再遍历 `CROP` 实体：
   - 查 `CropSpeciesDef`
   - 查下方 `FARMLAND` 实体的 `moisture` / `fertility`
   - 计算 `growth_modifier = fertility_factor * water_factor * season_factor`
     - `fertility_factor = (1 - sensitivity) + sensitivity * fertility`
     - `water_factor = (1 - sensitivity) + sensitivity * moisture`
     - `season_factor`：当前季节 == `grow_season` ? 1.0 : 0.3
   - 连作障碍：`consecutive_same_crop >= 3` → `growth_modifier *= 0.5`
   - `effective_ticks = base_ticks / growth_modifier`
   - 满足则推进阶段，更新地形材质 + `CropBlockEntityState`

### 2.5 命令服务器扩展（`gd_game_command_server.cpp`）

新增 4 个命令：

#### `cmd_till_farmland`
```
输入: { dimension, cell (上方为空气的 dirt/grass 格) }
校验: 目标格为 dirt/grass 材质，上方为空气
写入: set_terrain_cell(snt:farmland) + register_farmland_entity(moisture=0.5, fertility=0.7)
消耗: 锄子耐久 -1
emit: terrain_cell_synced + entity_created
```

#### `cmd_plant_crop`
```
输入: { dimension, cell (farmland 上方), item_id (种子) }
校验: 下方为 farmland 材质，目标格为空气，物品为 seed 类
查询: 由 item_key 反查 CropSpeciesDef（seed_item_key 匹配）
校验: 当前季节 == plant_season（或 plant_season==-1）
写入: set_terrain_cell(stage_material_keys[SEED]) + register_crop_entity(SEED)
消耗: inventory.remove(seed_item, 1)
更新: FarmlandBlockEntityState.last_crop_key / consecutive_same_crop
emit: terrain_cell_synced + entity_created
```

#### `cmd_harvest_crop`
```
输入: { dimension, cell (作物格) }
校验: 目标格为作物材质，对应 CROP 实体 growth_stage == MATURE
查询: CropSpeciesDef
产出: add_inventory_item(crop_item, random(min,max))
      add_inventory_item(byproduct, byproduct_count)  # 若有
分支:
  if repeat_harvest:
    重置 growth_stage = GROWING, last_growth_tick = now
    更新地形材质为 growing 阶段
  else:
    移除作物方块 + remove_entity(CROP)
    更新 FarmlandBlockEntityState.fertility -= 0.15
emit: terrain_cell_synced + inventory_changed
```

#### `cmd_fertilize`（骨粉催熟）
```
输入: { dimension, cell (作物格), item_id (骨粉) }
校验: 目标格为作物，CROP 实体存在，growth_stage != MATURE
写入: 推进 1 阶段（SEED→SPROUT→GROWING→MATURE），更新材质
消耗: inventory.remove(bone_meal, 1)
emit: terrain_cell_synced
```

### 2.6 玩家交互（`PlayerInteraction.gd`）

struct Gene {

struct Genome {
