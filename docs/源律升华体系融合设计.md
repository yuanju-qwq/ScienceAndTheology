# 一、总定位与设计结论

## 1.1 一句话定义

源律升华体系是一套以源质材料为世界底层、以八槽位器官网络为身体承载、以源质生理系统为能力闭环、以道路预设术式图为学习脉络、以个人源律术式编译为自由表达、以魔兽血脉与生态为内容来源、以符文和以太为工具表达的统一力量体系。

玩家不是收集一组互不相关的“器官装备”，也不是在八条职业线中选一条。玩家是在自己的身体中建立可运行、可失衡、可进化的源质生理网络。

## 1.2 本次重写的核心结论

~~~text
八元素：回答“源质在身体中能执行什么生理动作”
八槽位：回答“这些动作由身体的哪里承担”
八系统：回答“元素动作如何经由器官连成可运行的反应链”
个人源律术式：回答“如何把身体已经具备的动作、产物和回路编写成可编译的能力”
道路：回答“提供哪些预设术式图、核心节点和编译偏好，使同一条已成立反应链优先产出什么效果”
魔兽血脉与生态：回答“器官从哪里来、为何能共鸣、要付出什么代价”
符文、魔符、祭坛、以太：回答“工具、设施和工业如何以有限构筑调用、放大或稳定这份力量”
~~~

必须同时满足以下规则：

1. 八个源质生理系统不等于八个元素，也不等于八条道路。
2. 道路不等于一组锁死的器官或一排固定技能；道路提供预设术式图、道路核心节点和对系统反应的编译偏好。
3. 同一个系统在不同道路下可以编译出不同术式，例如呼吸交换系统在潮汐道路下偏水下恢复，在风暴道路下偏高空机动。
4. 单个器官只能提供局部元素动作；至少形成系统闭环后，才会开放可被个人术式调用的身体指令，并可被调度为当前主回路或协同回路。
5. 完成体是全身级阶段：八个槽位全部完成源质化后，八个器官还必须通过统合回路联合为一个整体，才能开始完成体成长。
6. 玩家与魔兽共享同一套源质身体规则。魔兽以天生血脉和生态自然成长，玩家以植入、培育、道路和仪式主动重构。
7. 魔兽不应只是器官掉落表。它们同时提供血脉、生态位、培养条件、共生对象、研究线索和高阶突破门槛。
8. 个人源律术式是独立于符文和魔符的自由编译体系，必须依赖升华器官网络；符文、魔符和武装只服务工具侧的有限构筑或外源供能设备。
9. 预发布阶段不维护旧接口。新数据模型一次到位，旧 source_law Godot 逻辑只作为迁移参考，不再新增功能。

## 1.3 设计目标

| 目标 | 设计结果 |
|---|---|
| 器官有组合深度 | 玩家通过核心、导流、效应、稳定等器官角色形成闭环，而不是只堆品质 |
| 道路有鲜明身份 | 每条道路提供可学习、可复制、可修改的预设术式图，并以核心节点和编译偏好维持独特表达 |
| 个人构筑有自由度 | 玩家自由连接身体指令、反应产物与回路服务；编译器以元素生理链而非预制技能模板约束结果 |
| 魔兽与世界相关 | 器官来源受血脉、群系、行为、食物链和培养条件约束 |
| 战斗、探索、工业同时受益 | 每个系统至少服务一项战斗、一项探索或生存、一项非战斗联动 |
| 风险可读且可处理 | 稳定、畸变、精神负荷和系统压力有明确来源、预警和处理手段 |
| 实现可控 | 系统组合由内容定义的配方驱动，不做不可维护的全自由排列组合 |
| 后续可扩展 | 首批只实装少量道路和系统，但数据接口从一开始支持完整八系统 |

## 1.4 不采用的方案

| 方案 | 优点 | 缺点 | 结论 |
|---|---|---|---|
| 八个系统直接等于八条职业线 | 容易理解 | 器官混搭弱，魔兽掉落很快固定化 | 不采用 |
| 所有器官任意组合自动产生未知技能 | 探索感强 | 组合数量爆炸，平衡和 UI 不可控 | 不采用 |
| 固定技能模板加器官/元素数值偏移 | 制作快速 | 元素、器官和系统会退化为词条，无法解释技能为何成立 | 不采用 |
| 每个器官绑定唯一道路 | 内容制作快速 | 道路与器官完全重叠，无法体现系统闭环 | 不采用 |
| 固定八槽位加内容定义的系统配方 | 可读、可测、可扩展 | 需要编写系统定义和预览 UI | 采用 |
| 身体指令加自由术式图编译 | 保留自由编程，同时可验证身体因果与风险 | 需要类型系统、编译诊断和术式编辑器 | 采用 |

# 二、统一分层模型

## 2.1 分层关系

~~~text
群系与魔兽生态
  -> 血脉、组织、催化材料、共生条件
  -> 玩家或魔兽的源质器官
  -> 源质生理系统
  -> 八器官统合与完成体
  -> 身体指令、反应产物与回路服务
  -> 道路预设术式图与编译偏好
  -> 玩家自由术式编译，或魔兽预设术式调用
  -> 符文、魔符、武装、祭坛、以太设备的有限工具表达
~~~

| 层级 | 核心问题 | 主要内容 | 不负责什么 |
|---|---|---|---|
| 源质与元素 | 力量能在身体中做什么 | 八元素对应的产生、传导、转化、释放、校准与回收动作 | 不决定玩家职业或独立术式 |
| 血脉与生态 | 力量为何存在 | 魔兽谱系、群系、食性、行为、组织来源 | 不直接给玩家数值 |
| 器官 | 身体如何承担元素动作 | 玩家植入或魔兽天生的八槽位、品质、成长、组织角色、元素反应贡献 | 不单独定义完整流派 |
| 源质生理系统 | 身体如何把元素动作连成反应 | 系统配方、反应链、闭环等级、身体指令、系统压力 | 不决定长期人格化方向 |
| 全身统合 | 八器官如何成为一具源质身体 | 统合桥接、全身级风险协调、完成体阶段 | 不替代局部系统 |
| 个人源律术式 | 如何把身体能力组织成行动 | 可编辑术式源图、编译、运行时回路申请、失败诊断 | 不凭空产生元素动作或绕过身体承载 |
| 道路 | 力量往哪里成长 | 主道路、预设术式图、核心节点、共鸣、领域和终局表达 | 不取代器官、系统或统合 |
| 工具结构 | 如何以有限构筑调用与放大 | 符文、魔符、武装、祭坛、以太机器 | 不成为个人术式编辑器或绕过身体承载 |

## 2.2 概念边界

必须避免以下混淆：

~~~text
稳定不是元素。
畸变不是元素。
灵能不是元素。
道路不是元素。
系统不是道路。
系统是已形成的身体功能闭环，不等于运行时的回路位置。
元素不是伤害颜色或泛用属性标签；每个元素都必须以具体生理动作进入器官和系统。
器官不是“带有元素”的装备，而是一个或多个元素动作在特定反应阶段的执行者。
系统必须验证元素反应链连续，不能只因元素标签相容就成立。
当前主回路是当前动作获得主要通量和道路表达的系统闭环，可随调度而变化。
协同回路是为当前主回路提供供能、控制、承载或环境支持的系统闭环。
统合回路是完成体后联结八个槽位的全身网络，不是一个可替换的局部系统。
器官不是技能。
系统闭环不是固定技能列表；它开放可被编译器验证和调用的身体指令。
个人源律术式是玩家保存、编辑和编译的自由图式，不需要符文或魔符作为语法节点。
道路预设术式图是可学习的源图，不是自动获得的最终技能；复制后可以修改。
符文、魔符与武装是工具侧的有限插槽构筑，不是人物个人术式的编译入口。
源质储备不是法力。
以太不是身体可吸收的源质。
~~~

## 2.3 玩家主循环

~~~text
探索群系、巢穴、遗迹
  -> 获取源质材料、组织样本、生态知识、符石
  -> 萃取、培养、调律、制作魔药
  -> 植入或成长源质器官
  -> 形成或强化源质生理系统
  -> 学习道路预设术式图，或编写、编译个人源律术式
  -> 以道路共鸣获得战斗、探索、工业联动能力
  -> 用符文、魔符、武装、祭坛和以太设备进行有限工具构筑或规模化
  -> 挑战更高等级生态、污染区、遗迹与 Boss
~~~

## 2.4 四种使用方式

| 使用方式 | 代表内容 | 玩家体验 |
|---|---|---|
| 内化 | 器官、系统、道路、魔药、能级 | 长期构筑、风险管理、身体进化 |
| 个人施术 | 术式源图、身体指令、当前主回路、协同回路 | 自由编程、即时战斗、移动、探索与救援 |
| 随身工具 | 符文、魔符、源律武装 | 有限插槽构筑、工具功能、外源供能 |
| 定点 | 祭坛、仪式、生态设施 | 基地经营、调律、战略能力 |
| 工业 | 以太、机器、自动化、以太武装 | 规模生产、能源、装备与后期效率 |

# 三、核心资源与力量形态

## 3.1 资源定义

| 名称 | 本质 | 是否可存储/交易 | 主要用途 |
|---|---|---|---|
| 源质材料 | 世界中的超凡物质 | 可以 | 萃取、魔药、符文、培育、工业转化 |
| 源质储备 | 玩家身体已内化的长期储量 | 不可以 | 器官成长、法力上限、突破资格 |
| 源质通量 | 当前器官网络安全传导能力 | 不可以 | 限制个人术式、工具术式、法力回复和过载 |
| 法力 | 可即时调用的安全输出缓冲 | 不可以 | 源律武装、个人术式、部分仪式 |
| 以太 | 工业稳定化后的源质形态 | 可以 | 机器、以太武装、遗迹设备 |
| 灵能 | 器官网络的信息读取能力 | 不作为物品 | 解析、预警、仪式预览、研究 |
| 符文 | 被封装的源律规律片段 | 可以 | 武装、祭坛、魔药校准、机器配方 |
| 魔符 | 由符文编译出的工具术式组件 | 可以，部分使用时消耗 | 有限工具术式构筑 |

## 3.2 转化边界

~~~text
源质材料 -> 提纯、萃取、魔药、器官培养 -> 源质储备
源质储备 -> 器官网络安全转换 -> 法力
源质材料 -> 工业稳定化 -> 以太
源质材料 -> 雕刻、蚀刻、融合 -> 符文
符文 -> 编译 -> 魔符或祭坛回路
~~~

以下转换禁止：

~~~text
源质储备 -> 以太：禁止，身体内化源质不可被机器抽取。
以太 -> 源质储备：禁止，工业形态不能直接被身体吸收。
普通人 -> 个人法力：禁止，必须完成启灵或使用外源供能装置。
源律武装 -> 自动赋予升华：禁止，武装是表达工具，不是身体承载替代品。
~~~

## 3.3 法力、通量与储备的关系

~~~text
源质储备：水库有多少水。
源质通量：管道每秒能安全通过多少水。
法力：水箱里已准备好、可立即取用的水。
~~~

建议派生关系：

~~~text
法力上限 = 基础启灵容量 + 心脏与血液贡献 + 有效元素反应链加成 + 装备/仪式修正 - 风险惩罚
法力回复 = 源质通量 * 转化效率 * 当前稳定度
个人术式可用效果预算 = 反应产物规模 * 系统闭环等级 * 元素反应完整度 * 对应通量份额 * 状态修正
个人术式表现 = 源图对可用效果预算的分配 + 道路核心或混搭联结的语义转换
~~~

## 3.4 凡人与升华者

| 状态 | 能做什么 | 不能做什么 |
|---|---|---|
| 凡人 | 采集、手工符文胚、操作普通机器、准备外源仪式 | 产生个人法力、稳定植入源质器官、主动施放个人源律术式 |
| 启灵者 | 建立首个系统闭环、使用基础法力、驱动基础武装 | 无限制混搭器官、运行多套高负荷系统 |
| 凝辉者及以上 | 同时维护多系统、形成道路共鸣、参与高阶仪式与遗迹 | 无代价忽略畸变、精神负荷或生态约束 |

# 四、源质元素、反应与连续性

## 4.1 基础八元素是生理动作

元素首先不是战斗颜色，而是源质在身体网络中允许执行的动作。每个器官必须把自己携带的元素落实到一个或多个反应阶段，系统则验证这些动作能否连续发生。元素允许哪些动作、动作允许出现在哪些阶段、哪些动作能连续或竞争，都由同一份元素生理规则内容定义。

| 元素 | 核心生理动作 | 常见反应阶段 | 常见器官表现 | 典型失衡 |
|---|---|---|---|---|
| 火 | 生热、升压、释能、加速转化 | 产生、转化、释放 | 炽核、熔血、炉胃囊 | 过热、烧灼、压力失控 |
| 水 | 溶解、携带、稀释、修复、回收 | 摄入、传导、回收 | 活化血、潮汐腺、黏膜 | 过度稀释、流失、低爆发 |
| 土 | 固结、承载、定型、沉积 | 转化、作用、校准 | 晶骨、甲壳、矿化皮 | 迟滞、硬化、结构疲劳 |
| 风 | 交换、牵引、传讯、排放 | 摄入、传导、释放 | 气囊、喷流膜、风压神经 | 缺氧、泄压、信号漂移 |
| 光 | 侦测、聚焦、辨识、净化信息 | 摄入、校准、过滤 | 辉光眼、观测晶、光导神经 | 感知过载、曝光、精神负荷 |
| 暗 | 贮藏、吸收、遮蔽、分解 | 摄入、转化、回收 | 暗膜、吸能囊、静默神经 | 侵蚀、沉降、饥渴反馈 |
| 秩序 | 限幅、节律、封界、校准 | 传导、校准、回收 | 秩序骨、节流神经、校准膜 | 迟滞、僵化、过度约束 |
| 混沌 | 变异、倍增、分裂、随机重构 | 转化、释放 | 畸变腺、活化菌胃、裂变囊 | 畸变、反噬、产物失控 |

同一元素可出现在不同器官中，但必须执行符合该元素的动作；同一器官也可以承载多个元素动作。因此不会退化为“火只能装心脏、土只能装骨骼”的八职业模型。

## 4.2 元素反应链

每个正式系统都不是“若干元素标签相加”，而是一条由器官实际执行的元素反应链：

~~~text
产生/摄入
  -> 传导
  -> 转化
  -> 作用/释放
  -> 校准/回收
~~~

不同系统会要求不同的阶段组合，但闭环至少要有一个来源或摄入点、一个传导或转化点、一个作用或释放点；高风险反应额外必须有校准、过滤或回收点。一个器官可以贡献多个相邻阶段，却不能独自填满同一系统的全部必需阶段。

~~~text
器官角色回答“谁来负责”。
元素动作回答“它到底做了什么”。
反应阶段回答“它在链条的哪一段做”。
系统闭环回答“这些动作能否把输入安全地变成结果并处理余量”。
~~~

例如炎炉重击不是“火属性攻击”：

~~~text
炽热心脏：火·生热升压·产生
  -> 熔血：水/火·携带热压·传导
  -> 炉胃囊：火/混沌·燃料转化·转化
  -> 净岩晶骨：土·定型承压·作用
  -> 耐热肺膜：风·排出余压·释放
  -> 秩序神经：秩序·限制爆发阈值·校准
~~~

缺少晶骨会让冲击反噬身体，缺少肺膜会让热压积聚，缺少秩序神经则只能以更高失控风险强行爆发。元素、器官和系统由此形成可见的因果链。

## 4.3 动作连续性与节点关系

不再只判断“火和水是否冲突”，而是判断两个元素动作在同一反应阶段是否能连续、分工或争夺控制权。

| 关系 | 例子 | 网络结果 |
|---|---|---|
| 连续 | 火产生热压 -> 水携带 -> 土承压 | 形成可解释的炎炉反应，效率高 |
| 分工 | 光侦测 -> 秩序校准 -> 水回收 | 形成稳定的辉光净化反应 |
| 并行 | 土定型与暗贮藏分别服务屏障和储备 | 可以共存，但不自动产生额外收益 |
| 阶段争夺 | 火与水都想作为同一循环的主产生端 | 需要切换反应主导权或增加节流 |
| 严重冲突 | 秩序限幅与混沌倍增同时控制同一转化节点 | 稳定显著下降，可能产生畸变或错误产物 |

因此火心和水肺不天然冲突：火负责产生、风肺负责排放、水血负责携带时，三者可以形成正常反应。冲突只在它们抢占同一节点、缺少缓冲或无法处理副产物时发生。

## 4.4 复合谱系是命名后的反应模式

| 名称 | 反应模式 | 规则 |
|---|---|---|
| 雷 | 风的传讯/释放 + 火的释能，或风的传讯 + 秩序的校准 | 是高速放电反应，不新增基础元素 |
| 生命 | 水的携带/修复 + 光的辨识/净化，常由秩序维持边界 | 是生物培养与恢复反应 |
| 冰 | 水的回收/凝结 + 秩序的限幅，或由土定型 | 是冷却、束缚与保存反应 |
| 金属 | 土的定型/承载 + 秩序的封界/校准 | 是结构与工业反应，不新增元素 |
| 腐化 | 暗的吸收/分解 + 混沌的变异/倍增 | 是高风险反应模式，不等于普通暗元素 |

复合名称只描述一段稳定、可复用的反应模式。内容不能以“雷”“生命”“冰”等名字绕过基础元素动作、器官贡献和系统闭环验证。

## 4.5 冲突缓冲与反应修复

| 反应问题 | 可用缓冲方式 |
|---|---|
| 火热压没有排出阶段 | 接入风肺、喷流囊或带风动作的皮肤接口 |
| 水携带过量而缺少定型 | 接入土骨、晶壳或秩序节流，避免稀释和流失 |
| 光侦测与暗贮藏争夺同一感知节点 | 用秩序神经分时调度，或拆分为感知与储备两个系统 |
| 秩序限幅与混沌倍增共用转化节点 | 只允许一方作为主导动作，另一方必须降为受控催化或由土/光提供结构与监测 |
| 暗分解产生污染副产物 | 接入水血运输、脏腑过滤和皮肤/肺部排出链 |

初期不需要制作复杂的全矩阵元素克制表。内容定义只需明确每个元素动作允许的阶段、可连续的下一动作、会竞争的节点以及可用的缓冲动作，evaluator 再据此给出缺口和风险。

## 4.6 源质纯度

| 纯度 | 典型用途 | 主要风险 |
|---|---|---|
| 污浊 | 应急工业、低阶合成 | 畸变、低兼容 |
| 粗源质 | 萃取与分离原料 | 效率低 |
| 属性源质 | 魔药、符文、普通培养 | 元素动作失配 |
| 纯净源质 | 高品质器官、调律 | 成本高 |
| 活化源质 | 组织培养、共生系统 | 失控生长 |
| 古代源质 | 遗迹、突破、传奇器官 | 信息污染、精神负荷 |
| 以太化源质 | 工业设备 | 不能身体吸收 |

# 五、八槽位器官网络

## 5.1 槽位不是系统

八槽位是玩家身体中可被源质化、替换或叠加的主要承载位置。它们不是八个独立职业，也不与八系统一一对应。

八槽位同时也是所有源质魔兽的统一功能模型。鸟类、蛇类、菌群或遗迹生体的实际解剖结构可以不同，但必须映射到相同的八个功能槽位；例如雷鸟的气囊映射为肺部，海蛇的感温膜映射为眼睛或神经，菌群的菌核映射为脏腑/源腺。

例如：

~~~text
源质循环系统让心脏产生、血液传导、神经或骨骼校准同一段源质反应。
呼吸交换系统让肺部摄入或排出、血液携带、皮肤或神经调节环境交换。
净化排异系统让眼睛辨识污染、血液转运、脏腑过滤、皮肤或肺部排出副产物。
~~~

## 5.2 八个源质器官槽位

| 槽位 | 主要身体职责 | 常见元素动作与阶段 | 常见器官角色 | 常见玩法收益 |
|---|---|---|---|---|
| 心脏 | 核心驱动、节律、主道路锚定 | 火/暗产生，水泵送，秩序节律；产生、转化 | 核心、泵送、爆发 | 源质储备、法力上限、当前主回路输出 |
| 骨骼 | 承压、支撑、筋束附着 | 土定型承压，秩序封界；转化、作用、校准 | 结构、稳定、效应 | 防御、近战、采矿、负重 |
| 血液 | 运输、恢复、污染传播 | 水溶解携带，火热媒，暗吸收；摄入、传导、回收 | 导流、恢复、净化 | 法力回复、生命回复、药效 |
| 肺部 | 气体与环境交换 | 风摄入、牵引、泄压，水气液交换；摄入、释放 | 接口、导流、适应 | 水下、毒区、高空、冲刺 |
| 眼睛 | 视觉与源律识别 | 光辨识聚焦，暗低光贮像；摄入、校准、过滤 | 感知、定位、校验 | 探测、弱点、遗迹解析 |
| 神经 | 反应、控制、信号传递 | 风传讯，秩序限幅，光聚焦；传导、校准 | 导流、节流、效应 | 冷却、闪避、施法响应、精神承载 |
| 皮肤 | 外部屏障、交换表面 | 土固结，水表面交换，暗遮蔽，风排放；作用、过滤、释放 | 接口、过滤、效应 | 护甲、抗性、隐匿、环境保护 |
| 脏腑/源腺 | 消化、排异、分泌、共生 | 火/混沌转化，水过滤回收，暗分解；摄入、转化、回收 | 转换、过滤、接口 | 材料吸收、净化、驯化、培养 |

第八槽正式命名为“脏腑/源腺”。它代表一组可替换的源质化内脏复合体，例如胃囊、净化肝、肾囊、潮汐腺、孢子腺或信息素腺。

## 5.3 为什么不新增独立肌肉槽

肌肉和筋束在玩法上主要服务骨肌承载系统。将其单独做为第九槽会把早期构筑切得过细，却仍然无法为消化、排异、分泌和共生提供承载位置。

因此：

~~~text
肌肉、筋束、腱膜：作为骨骼槽中的源质组织类型或附属组织。
胃、肝、肾、腺体：作为脏腑/源腺槽中的互斥主器官选择。
~~~

这既保留人体结构逻辑，也让第八槽成为对成长、生态和畸变都真正有价值的决策。

## 5.4 器官的八个数据维度

每个源质器官必须至少定义：

| 维度 | 说明 |
|---|---|
| 槽位 | 必须装入哪个身体位置 |
| 组织角色 | 核心、导流、效应、稳定、过滤、接口、感知等，可多选 |
| 元素反应贡献 | 每个元素对应的生理动作、反应阶段、容量与是否主导该阶段 |
| 源质纯度 | 纯度、活化状态、污染与可承受的反应负荷 |
| 血脉标签 | 来源谱系，例如爬行、鸟类、水生、菌群、遗迹生体 |
| 生态标签 | 适应群系、食性、温度、水压、昼夜、共生关系 |
| 系统标签 | 能参与哪些系统角色，例如“呼吸导流”“净化过滤” |
| 风险画像 | 稳定修正、畸变风险、精神负荷、系统压力类型 |

器官不能只保存一个 path_id。原生道路或参考路线可以作为内容标签，但不能决定器官能否进入系统。

## 5.5 器官角色

| 角色 | 作用 | 典型例子 |
|---|---|---|
| 核心 | 决定反应链的主产生/摄入或主转化节点、上限与能力方向 | 炽核兽心、潮汐腺、辉光眼 |
| 导流 | 决定资源流速、冷却、回复与连通 | 熔血、活化血、雷鸣神经 |
| 效应 | 决定系统直接产生什么动作或状态 | 岩蜥骨、海蛇皮、孢子腺 |
| 稳定 | 降低冲突、过载或畸变 | 净岩晶骨、秩序膜、净化肝 |
| 过滤 | 处理污染、毒素、无效源质 | 肾囊、净化血、滤毒皮 |
| 接口 | 把系统接到环境、魔兽或设备 | 海蛇肺、风暴气囊、信息素腺 |
| 感知 | 提供识别、定位、预警与解析 | 辉光眼、雷鸟感受器 |

组织角色不能替代元素反应贡献。例如“导流”只说明熔血可以承担传输职责；它还必须声明自己以水的携带、火的热媒或暗的吸收来传导什么。没有可连接的元素动作，导流角色不计入系统闭环。

## 5.6 器官的元素反应贡献

每个器官至少有一项元素反应贡献；高阶器官可以拥有多项，但每一项都必须说明具体的元素、动作、阶段和代价：

~~~text
炽热心脏：
  火 -> 生热升压 -> 产生 -> 高热负荷
  秩序 -> 维持节律 -> 校准 -> 爆发上限下降

熔血：
  水 -> 携带溶质 -> 传导 -> 失水风险
  火 -> 作为热媒 -> 传导 -> 烧灼风险

净岩晶骨：
  土 -> 定型承压与余热沉积 -> 作用/回收 -> 结构疲劳
  秩序 -> 限制形变 -> 校准 -> 动作迟滞
~~~

这使“火心 + 熔血 + 晶骨”不是三个带火、土、水标签的物品，而是一条能被系统 evaluator 逐段读取的产生、传导、作用与校准链。

## 5.7 品质与成长

| 品质 | 定位 |
|---|---|
| 劣化 | 污染、畸变或应急移植，带明显代价 |
| 普通 | 基础可用组织 |
| 优良 | 有明确系统优势 |
| 纯净 | 稳定且适合长期培养 |
| 古代 | 含遗迹或失落文明特性，需要解析 |
| 完美 | 终局目标，通常来自完整生态、Boss 或文明工程 |

成长不只增加面板数值。每个器官的成长节点应选择以下至少一种：

~~~text
提高通量。
降低系统压力。
增加系统角色权重。
解锁新的生态接口。
扩大道路共鸣。
改变可编译术式的可用指令、产物或风险形态。
~~~

# 六、八个源质生理系统

## 6.1 系统总表

| 系统 | 身体基础 | 必需元素反应链 | 主玩法 | 核心槽位 | 常见风险 |
|---|---|---|---|---|---|
| 源质循环 | 心脏、血液、神经 | 产生/摄入 -> 传导 -> 节流转化 -> 输出或回收 | 法力、爆发、恢复、节律 | 心脏 | 过热、流量断裂 |
| 骨肌承载 | 骨骼、神经、皮肤 | 驱动输入 -> 信号传导 -> 土性定型承压 -> 冲击释放/恢复 | 近战、采矿、护甲、移动 | 骨骼 | 结构疲劳、迟滞 |
| 呼吸交换 | 肺部、血液、皮肤 | 风/水摄入 -> 水性携带 -> 表面调节 -> 风性排出 | 水下、高空、毒区、冲刺 | 肺部 | 缺氧、压差、环境失配 |
| 消化炼化 | 脏腑、心脏、血液 | 材料摄入 -> 火/暗/混沌转化 -> 水性运输 -> 过滤回收或排渣 | 吸收、魔药、材料转化 | 脏腑/源腺 | 代谢过载、污染吸收 |
| 净化排异 | 脏腑、血液、皮肤、眼睛 | 光性辨识 -> 水性转运 -> 过滤分离 -> 风/皮肤排出 -> 秩序封界 | 解毒、抗畸变、状态抵抗 | 脏腑/源腺 | 过滤饱和、净化反噬 |
| 神经感知 | 眼睛、神经、心脏 | 光/暗感知 -> 风/秩序传讯 -> 能量响应 -> 反馈校准 | 锁定、反应、灵能、解析 | 神经或眼睛 | 精神负荷、感知噪声 |
| 外皮屏障 | 皮肤、骨骼、血液 | 环境输入 -> 土/暗表层定型 -> 水性修复或风性泄放 -> 秩序校准 | 护甲、抗性、隐匿、护盾 | 皮肤 | 屏障破裂、环境侵蚀 |
| 分泌共生 | 源腺、神经、血液、肺部 | 生态信号摄入 -> 腺体转化 -> 水性分配 -> 风/皮肤释放 -> 神经校准 | 驯化、召唤、形态适应、生态交互 | 脏腑/源腺 | 血脉排斥、共生失控 |

每个系统定义还必须声明反应产物、副产物与压力来源。例如炎炉循环的产物是可控热压，副产物是余热与燃烧残渣；潮汐交换的产物是可用环境载体，副产物是盐分、水压或低温负担。个人术式只能调用已经形成的产物与身体指令，风险从副产物和缺失阶段派生。

## 6.2 源质循环系统

源质循环系统是所有升华者的基础系统。它管理源质储备到法力的安全输出，也决定身体能否支持其他高负荷系统。

~~~text
最小元素反应链：心脏产生或转化 -> 血液传导 -> 神经、骨骼或脏腑节流并输出/回收
常见校准器：土性骨骼、秩序神经、秩序型脏腑
~~~

| 构筑方向 | 典型组合 | 结果 |
|---|---|---|
| 炎炉循环 | 炽热心脏 + 熔血 + 净岩晶骨 | 高爆发、高热、熔炼联动 |
| 潮汐循环 | 潮汐心核 + 活化血 + 海蛇肺 | 持续回复、水下耐久 |
| 沙铠循环 | 岩核 + 矿质血 + 秩序骨 | 护盾、承压、低波动 |
| 风暴循环 | 雷核 + 导电脉血 + 雷鸣神经 | 快速充能、短周期爆发 |
| 辉光循环 | 辉光心核 + 净化血 + 光眼 | 治疗、净化、稳定施法 |

循环系统通常开放“调动”“蓄积”“节流”“爆发释放”等身体指令，供个人术式调用；它不应替代骨肌系统的承压、呼吸系统的环境交换或神经系统的锁定与控制指令。

## 6.3 骨肌承载系统

骨肌承载系统负责力量传递、承压、姿态、冲击和工具使用。筋束和肌肉组织作为骨骼槽的组织类型，受神经控制，并由皮肤或血液提供外部保护与恢复。

~~~text
最小元素反应链：心脏或血液提供驱动 -> 风/秩序神经传令 -> 土性骨骼定型承压 -> 皮肤或血液吸收余量
~~~

| 构筑方向 | 典型组合 | 结果 |
|---|---|---|
| 沙铠承载 | 岩蜥骨 + 沙虫筋束 + 地脉皮肤 | 护甲、击退抗性、采矿 |
| 风暴承载 | 磁化骨 + 雷鸣神经 + 风暴肺 | 冲刺、空中调整、导电打击 |
| 炎炉承载 | 炎炉骨髓 + 熔血 + 耐热皮 | 重击、热武装承压、近战爆发 |
| 潮汐承载 | 柔韧骨膜 + 活化血 + 鳞皮 | 水中移动、控制抗性、持续作战 |

骨肌系统不能只提供“攻击力”。它还必须影响采矿强度、重型装备承载、移动姿态、击退、坠落和工程工具使用。

## 6.4 呼吸交换系统

呼吸交换系统负责从环境中取得可用源质载体，并处理水压、毒雾、高温、稀薄空气和风压等生存条件。

~~~text
最小元素反应链：肺部以风/水摄入环境载体 -> 血液以水性携带 -> 皮肤或神经调节交换 -> 肺部或皮肤排出无效载体
~~~

| 构筑方向 | 典型组合 | 结果 |
|---|---|---|
| 潮汐交换 | 海蛇肺 + 活化血 + 鳞皮 | 水下呼吸、水压适应、恢复 |
| 风暴交换 | 风暴气囊 + 雷鸣神经 + 导电脉血 | 滑翔、冲刺、高空耐受 |
| 炎炉交换 | 耐热肺膜 + 熔血 + 过滤皮 | 炎热区、蒸汽喷发、熔炉近作业 |
| 辉光交换 | 净化肺 + 光血 + 过滤皮 | 污染区净化、毒雾识别 |

呼吸系统是群系探索的重要门槛。它应让水域、火山、高原、污染区和遗迹内部都拥有不同的身体准备要求。

## 6.5 消化炼化系统

消化炼化系统把摄入的魔药、源质材料或特殊生物质转换成可被身体使用的增益。它不是“免费把物品变成经验”，而是决定魔药效率、临时强化、营养维持和低阶吸收能力。

~~~text
最小元素反应链：脏腑摄入材料 -> 火、暗或混沌动作转化 -> 心脏/血液驱动并运输产物 -> 脏腑、皮肤或肺部回收/排出副产物
~~~

| 构筑方向 | 典型组合 | 结果 |
|---|---|---|
| 炎炉炼化 | 炉胃囊 + 炽热心脏 + 熔血 | 提高燃料类与热源魔药效率，增加热负荷 |
| 潮汐炼化 | 潮汐腺 + 活化血 + 海蛇肺 | 水系药剂延长、恢复物质高效吸收 |
| 菌群炼化 | 共生菌胃 + 过滤血 + 净化肝 | 低纯源质利用、但有共生维护需求 |
| 古代炼化 | 遗迹消化核 + 秩序心 + 光眼 | 解析古代源质，精神负荷较高 |

消化炼化系统的价值在于让不同来源材料具有身体构筑意义，而不是把所有材料都压缩为统一经验值。

## 6.6 净化排异系统

净化排异系统负责处理毒素、污染、畸变、器官排斥和环境性负面状态。它是高风险混搭的安全阀，但不应完全消除风险。

~~~text
最小元素反应链：眼睛以光性辨识污染 -> 血液以水性转运 -> 过滤型脏腑分离 -> 皮肤或肺部以风性排出 -> 秩序动作封界
~~~

| 构筑方向 | 典型组合 | 结果 |
|---|---|---|
| 辉光净化 | 净化肝 + 光辉血 + 辉光眼 | 降畸变、提高仪式可见信息 |
| 沙铠排异 | 矿化肾囊 + 矿质血 + 岩皮 | 抗毒、承压、缓慢但稳定 |
| 潮汐排异 | 潮滤腺 + 活化血 + 海蛇皮 | 稀释污染、水域恢复 |
| 暗蚀过滤 | 暗膜肾囊 + 过滤皮 + 秩序眼 | 抗侵蚀，但有隐匿副作用 |

净化排异系统只能降低错误组合的代价，不能把严重冲突变成无代价最优解。

## 6.7 神经感知系统

神经感知系统控制反应、瞄准、施法响应、源律视觉和遗迹信息读取。它是风暴与辉光道路的重要基础，也能作为所有高操作构筑的协同回路。

~~~text
最小元素反应链：眼睛或感受器以光/暗摄入信息 -> 神经以风/秩序传讯 -> 心脏供能响应 -> 神经回传并校准阈值
~~~

| 构筑方向 | 典型组合 | 结果 |
|---|---|---|
| 风暴感知 | 雷鸟感受器 + 雷鸣神经 + 风暴心核 | 闪避、锁定、快速施法 |
| 辉光感知 | 辉光眼 + 光导神经 + 净化血 | 源质视野、遗迹解析、弱点识别 |
| 沙铠感知 | 地脉眼 + 稳定神经 + 岩核 | 矿脉探测、结构识别、工程预警 |
| 暗影感知 | 夜视膜 + 静默神经 + 暗血 | 潜行识别、低光感知、感知噪声风险 |

精神负荷主要来自此系统。高灵能不是纯数值加成，而是更完整地理解系统、仪式、生态和遗迹的能力。

## 6.8 外皮屏障系统

外皮屏障系统是身体与外界的第一层接触面，负责护甲、抗性、环境隔离、护盾、隐匿和表面源质交换。

~~~text
最小元素反应链：皮肤接收环境刺激 -> 土/暗动作形成表层 -> 骨骼承压或血液修复 -> 风性泄放或秩序动作校准边界
~~~

| 构筑方向 | 典型组合 | 结果 |
|---|---|---|
| 沙铠屏障 | 地脉皮肤 + 岩蜥骨 + 矿质血 | 岩盾、抗击退、抗坠落 |
| 潮汐屏障 | 海蛇鳞皮 + 活化血 + 潮汐肺 | 水压防护、水中抗性 |
| 炎炉屏障 | 熔甲皮 + 炎炉骨髓 + 熔血 | 火抗、热反伤、过热风险 |
| 辉光屏障 | 光幕皮 + 净化血 + 秩序骨 | 护盾、驱散、队友支援 |

屏障系统允许装备与身体共同参与防御：器官负责基础适应与共鸣，装备提供可替换的外部强度。

## 6.9 分泌共生系统

分泌共生系统是对传统内分泌和生殖功能的游戏化改写。它不涉及玩家繁殖玩法，而是负责信息素、孢子、腺体分泌、血脉适应、魔兽安抚、陪伴生物和生态接口。

~~~text
最小元素反应链：眼睛、肺部或皮肤摄入生态信号 -> 源腺以水/火/混沌动作转化 -> 血液运输 -> 肺部或皮肤释放 -> 神经以秩序动作校准
~~~

| 构筑方向 | 典型组合 | 结果 |
|---|---|---|
| 潮汐共生 | 潮汐腺 + 海蛇肺 + 活化血 | 安抚水生生物、水域共鸣、临时水生适应 |
| 菌群共生 | 孢子腺 + 共生菌胃 + 过滤血 | 种植、污染处理、维护成本 |
| 辉光共生 | 鹿角腺 + 光导神经 + 净化血 | 治疗脉冲、草木生态互动 |
| 风暴共生 | 鸣禽腺 + 雷鸣神经 + 风压肺 | 侦察鸟类、天气预警、声波信号 |

共生系统让“与魔兽或生物配合”不止等于击杀取材。它可驱动驯养、保护、迁徙、巢穴合作和特殊任务。

# 七、系统闭环、成长期与全身统合

## 7.1 系统配方不是任意排列

每个系统必须由内容定义的配方识别。配方使用槽位、组织角色、元素反应贡献、血脉标签、生态标签和风险条件，不使用“任意三个器官自动生成未知术式”的黑箱逻辑。系统只负责开放有明确生理含义的身体指令与反应产物；个人术式如何连接这些指令，由独立编译器负责。

~~~text
系统定义 =
所需角色
+ 元素反应链（阶段、允许动作、反应产物与副产物）
+ 闭环规则
+ 成长联结
+ 可开放的身体指令与可读反应产物
+ 风险与维护条件
~~~

角色匹配而反应链断裂时，系统只能显示“候选”或“失衡”，不能作为正式闭环。单个系统永远不是“完成体”。系统只描述身体的一项功能已经运行到什么程度；完成体只描述八个源质器官已经统合成一具完整的源质身体。

## 7.2 系统的三层成长

| 阶段 | 条件 | 获得内容 |
|---|---|---|
| 共鸣 | 两个满足系统角色的器官形成一段连续元素动作 | 被动属性、反应链预览、弱共鸣特效 |
| 闭环 | 核心 + 导流 + 效应三者连通，且反应链覆盖系统所需产生/摄入、传导/转化、作用/释放阶段 | 系统正式激活、开放身体指令与反应产物、允许分配通量 |
| 成长联结 | 闭环向其他已源质化器官或系统输出产物、接收副产物处理或共享校准节点 | 强化能力、跨系统被动、为全身统合提供一条有效连接 |

示例：

~~~text
炽热心脏（火·产生） + 熔血（水/火·传导） = 炎炉循环共鸣。
再接入净岩晶骨（土·承压作用） = 炎炉循环闭环。
接入耐热肺膜（风·泄压）或炉胃囊（火/混沌·燃料转化），
令循环接收炼化产物并处理余热
= 炎炉循环获得成长联结。

它仍然只是成长期中的一个系统，不是完成体。
~~~

## 7.3 成长期

成长期从启灵后的第二个源质器官开始，到八个槽位全部完成源质化为止。这个阶段的核心体验是逐个唤醒身体节点、建立局部闭环，并让不同系统逐渐互相供能、控制和保护。

| 身体阶段 | 源质化槽位数 | 主要体验 |
|---|---:|---|
| 启灵 | 1 | 锚定道路核心，建立第一条源质通路 |
| 早期成长 | 2-3 | 形成第一套系统共鸣或闭环 |
| 中期成长 | 4-6 | 当前主回路稳定运行，协同回路与成长联结出现 |
| 后期成长 | 7 | 补齐最后的身体短板，准备八器官统合 |
| 八器官齐备 | 8 | 所有槽位均源质化，但尚未自动成为完成体 |

八器官齐备只是完成体的前置条件，不是完成体本身。八个源质器官若没有形成统一回路，仍可能各自抢夺通量、产生排异或保持彼此孤立。

## 7.4 回路调度容量

系统可以被识别、保留和成长，但不是所有闭环都能同时获得高功率通量。回路调度容量只限制“当前主回路 + 协同回路”的同时运行数量，不会删除未被调度系统的被动效果与成长数据。任何被调度回路都必须先具有完整的元素反应链，不能把一个缺失泄压、过滤或校准节点的候选系统强行设为当前主回路。

| 身体阶段/能级 | 高功率回路容量 | 说明 |
|---|---:|---|
| 凡人 | 0 | 无源质系统 |
| 启灵 | 1 | 一个当前主回路；尚无协同回路 |
| 成长期 | 2 | 一个当前主回路与一个协同回路 |
| 八器官齐备 | 3 | 一个当前主回路与至多两个协同回路；可验证多个成长联结，但仍受统合前冲突限制 |
| 完成体 | 3 + 统合回路 | 不增加无限高功率回路；获得全身级能力与道路完成体资格 |
| 领域及以上 | 内容决定 | 可在领域或仪式条件下临时扩展 |

未进入当前主回路或协同回路的闭环可以保留低强度被动，但不占用全部通量，也不能承担需要高功率调度的个人术式节点。

## 7.5 当前主回路、协同回路与统合回路

系统没有永久的主、辅身份。系统是身体已经形成的功能闭环；回路是该闭环在当前动作、行为或战斗阶段中的调度角色。

| 运行层 | 选择规则 | 权限 | 限制 |
|---|---|---|
| 当前主回路 | 由待执行的个人术式、行为意图或战术预设从一个有效闭环中选出 | 使用主要通量，并承担术式的主身体指令与主反应产物；仅在图中接入道路核心时携带道路语义 | 同一时刻只能有一个；占用大部分资源，系统压力更高 |
| 协同回路 | 由待执行的个人术式、行为意图或行为模板选出，为当前主回路服务 | 为术式提供供能、控制、承载、环境接口、稳定或声明过的次级反应 | 数量受回路调度容量限制；不能无条件各自满功率执行术式节点 |
| 休眠闭环 | 已识别但未进入当前调度 | 保留器官兼容、低等级被动与成长资格 | 不提供高功率身体指令或术式节点 |
| 统合回路 | 仅在完成体且八器官统合有效时由全身网络生成 | 协调八器官、全身风险与完成体公共能力 | 不替代局部系统，不占用当前主回路或协同回路容量；完整度或桥接失效时离线 |

玩家不再永久指定某个系统为主，而是在编译个人术式时声明或由编译器推导其所需的当前主回路与协同回路。编译结果把这些要求固化为运行时申请；施放时不能临时绕过。将已经高功率运行的当前主回路改为另一闭环时，应受调度冷却、短暂通量衰减、休整或祭坛调律约束，避免每次遇敌都无成本换一具身体。魔兽 AI 使用同一规则，只是从物种预设术式图和环境行为中选择申请。

协同回路不能只给出泛用数值加成。它必须明确接走当前主回路的哪一段反应：例如骨肌承载接住炎炉循环的热压冲击，呼吸交换排出其余热，净化排异处理混沌炼化的污染副产物。

## 7.6 八器官齐备的判定

八器官齐备要求八个槽位都存在可运行的源质器官实例，而不是普通器官加上几个源质器官：

~~~text
心脏、骨骼、血液、肺部、眼睛、神经、皮肤、脏腑/源腺
全部完成源质化。
每个实例达到最低完整度和成长要求。
每个实例可被至少一个有效系统或统合桥接读取。
不存在未被压制的严重冲突或立即排异。
~~~

这一步完成后，玩家获得“统合资格”，可开始完成体的仪式与成长，不会自动跳过统合过程。

## 7.7 全身统合与完成体

八个器官必须形成单一连通的源质网络。为防止玩家只是把八个独立器官填满，统合至少需要四条覆盖所有槽位的桥接：

| 统合桥接 | 必须联合的器官 | 必须闭合的元素反应 | 作用 |
|---|---|---|---|
| 能量与炼化桥 | 心脏 + 血液 + 脏腑/源腺 | 产生/摄入 -> 传导 -> 转化 -> 回收 | 源质储备、法力、营养与过滤形成闭环 |
| 控制与反馈桥 | 心脏 + 神经 + 眼睛 | 感知 -> 传讯 -> 节律响应 -> 反馈校准 | 让道路意志、感知和施法控制回到同一节律 |
| 环境与屏障桥 | 血液 + 肺部 + 皮肤 | 环境摄入 -> 携带交换 -> 过滤/排出 -> 表层恢复 | 让外界交换、环境适应与排出能力一致 |
| 承载与行动桥 | 心脏/血液 + 骨骼 + 神经 + 皮肤 | 驱动 -> 控制传导 -> 定型承压 -> 冲击释放/恢复 | 让输出、姿态、承压和外部行动不再脱节 |

完成统合还要求：

1. 八个器官构成单一连通图，每个槽位至少进入一条统合桥接。
2. 四条桥接的元素反应首尾可追溯：产生、传导、转化、作用、排出与校准不能出现无法处理的孤立副产物。
3. 主道路核心参与能量桥或控制桥，不允许道路脱离身体。
4. 稳定度、畸变、精神负荷与系统压力达到仪式安全阈值。
5. 在对应生态、医疗设施或祭坛中完成“全身统合仪式”。
6. 玩家明确确认统合方向；仪式会锁定完成体的道路表达，但不禁止以后通过重大重铸改变。

完成统合仪式后，玩家进入“完成体·初统合”。完成体是一个新的长期阶段，而不是一次性四件套奖励：

| 完成体阶段 | 条件 | 获得内容 |
|---|---|---|
| 初统合 | 八器官齐备 + 全身统合仪式 | 统合回路、全身风险协调、完成体身体指令与道路完成体预设术式图 |
| 成熟完成体 | 八器官持续成长、完成跨系统维护与生态要求 | 更强的道路预设术式、领域资格、古代器官深度利用 |
| 领域完成体 | 高能级、遗迹或文明工程条件 | 对小范围环境、仪式或机器回路的稳定影响 |

## 7.8 系统评估顺序

~~~text
1. 验证每个器官实例是否有效、健康、已占用正确槽位。
2. 根据内容快照匹配可识别系统配方。
3. 对每个候选系统计算角色完整度、元素动作是否连续、反应产物/副产物、血脉关系和生态条件。
4. 生成元素反应报告：缺失阶段、竞争节点、可用缓冲动作和预计压力。
5. 计算成长联结与八器官齐备状态。
6. 计算全身统合桥接、全身元素反应连续性、主道路参与度与完成体资格。
7. 记录当前道路可提供的核心节点和反应偏好，但不修改通用反应产物；这些语义仅在术式图实际接入道路核心时由 compiler 应用。
8. 计算稳定、畸变、精神负荷与系统压力。
9. 输出不可变的身体指令、反应产物和可调度回路快照，供个人术式编译器或魔兽 AI 使用。
10. 得出当前主回路、协同回路、统合状态、失效原因和可展示的修复建议。
~~~

## 7.9 系统与完成体失效原则

| 失效类型 | 结果 |
|---|---|
| 缺核心或导流器官 | 对应系统不激活 |
| 缺失元素反应阶段 | 显示缺少产生、传导、转化、释放、过滤或校准的具体动作；候选系统不能闭环 |
| 副产物无去向 | 降低效率并累积热负荷、污染、压差或精神负荷，严重时强制降级 |
| 通量不足 | 已编译术式无法申请所需回路，或在内容明示允许时降级为低通量版本 |
| 生态条件缺失 | 成长联结或依赖该生态条件的道路核心/预设图暂停，基础闭环和通用图不直接销毁 |
| 稳定过低 | 增加失误、冷却、反噬或暂停 |
| 器官损伤/污染 | 系统性能下降，可能触发排异 |
| 严重冲突 | 强制要求稳定器，或禁止与指定冲突系统同时进入同一回路调度 |
| 完成体中移除任一源质器官 | 暂停统合回路、完成体身体指令与完成体预设术式图，退回八器官不齐备或成长期；不静默删除器官成长数据 |

# 八、道路与系统的配合

## 8.1 明确分工

~~~text
系统决定“元素动作如何通过身体变成可用反应”。
道路决定“提供哪些预设术式图、核心节点和编译偏好，让哪一段已成立反应优先产出什么效果”。
~~~

道路不是第九个系统，也不是器官库存分类。道路是玩家对已形成生理网络的长期引导方式：它偏好特定元素动作、反应阶段与产物，却不能凭空补出产生、传导、泄压或校准节点。

| 维度 | 系统 | 道路 |
|---|---|---|
| 来源 | 器官闭环 | 启灵仪式与主核心锚定 |
| 变化频率 | 植入、调律、休整后可调整 | 低频，改变需要重大仪式 |
| 主要效果 | 反应产物、身体指令、环境适应、风险 | 预设术式图、道路核心节点和特定反应产物的编译形态；不提供无条件效果预算倍率 |
| 是否能跨道路 | 可以 | 一个玩家通常只有一个主道路 |
| 与元素的关系 | 必须读取器官实际提供的元素动作与反应阶段 | 偏好已有动作的顺序、阈值、产物和副产物处理，不直接提供元素 |

## 8.2 道路共鸣规则

每条道路定义：

| 字段 | 说明 |
|---|---|
| 偏好元素动作 | 例如火的生热、风的泄压、光的辨识、秩序的限幅；不是单纯主/副元素标签 |
| 道路核心 | 启灵时锚定主道路的核心器官或仪式 |
| 优先系统 | 最容易承载道路语义的两个到三个系统，不让其他系统失去基础编译资格 |
| 共鸣条件 | 所需器官角色、反应阶段、元素动作、闭环等级和生态条件 |
| 产物偏转 | 道路如何改变已有反应的产物、阈值、释放形态或副产物处理 |
| 预设术式图 | 启灵、系统、标志与完成体四层源图；每张图声明所需身体指令、反应产物、主/协同回路和道路核心节点 |
| 道路核心节点 | 只有道路能提供的术式语义；保留它会把同一效果预算偏转为道路特有的形态、效率或稳定性取舍，不提供无条件倍率 |
| 冲突警告 | 容易争夺同一反应节点或无法处理副产物的组合 |

建议计算：

~~~text
可用效果预算 =
反应产物规模
* 反应链完整度
* 稳定、通量与环境修正

术式编译结果 =
术式源图对同一效果预算的分配
+ 道路核心节点或混搭联结带来的语义转换
~~~

道路不能把一个不成立的系统强行变成成立，也不能因为道路已锚定就暗中改变任何未使用道路核心节点的通用术式。例如辉光道路不能在没有“光性辨识 -> 神经传讯 -> 秩序校准”链条时直接获得完整遗迹解析；即使链条成立，也只能在术式图实际接入辉光核心节点时把已有感知反应偏转为解析、净化或支援产物。

## 8.3 道路共鸣层级

| 共鸣等级 | 条件 | 效果 |
|---|---|---|
| 无共鸣 | 系统反应链不包含道路偏好的元素动作或阶段，或术式不接入道路核心节点 | 按通用身体指令编译，保有完整基础效果预算 |
| 辅助共鸣 | 反应链包含一个道路偏好动作，且术式实际接入道路核心节点 | 在同一效果预算内获得小幅效率、稳定性或资源取舍，不获得普遍倍率 |
| 深度共鸣 | 优先系统形成闭环，完整覆盖道路要求的关键动作和副产物处理 | 解锁道路系统/标志预设术式图与可复制的核心节点，获得道路特有的语义转换 |
| 完成体共鸣 | 全身完成体 + 道路能级 + 全身反应连续 + 仪式/生态条件 | 解锁完成体预设术式图、专属形态、领域或终局表达 |

## 8.4 道路预设术式图与个人自由编译

每条道路都必须提供预设术式图。预设图是可阅读、可复制、可修改的术式源图，不是已经结算好数值的固定技能，也不是符文或魔符。它给玩家展示该道路如何把身体反应组织成行动，并提供自由改写时可保留、替换或绕开的结构起点。

| 预设图层级 | 解锁条件 | 作用 |
|---|---|---|
| 启灵图 | 道路锚定与最小身体闭环 | 教会最小可运行的输入、处理与输出，保证新道路有明确起点 |
| 系统图 | 对应优先系统形成闭环或成长联结 | 展示一个系统如何成为当前主回路，以及需要什么协同回路 |
| 标志图 | 道路深度共鸣 | 包含道路核心节点，展示该道路独有的产物偏转与副产物处理 |
| 完成体图 | 八器官统合和完成体共鸣 | 调用统合回路与全身身体指令，不能在成长期提前编译 |

道路预设图与玩家术式源图的关系如下：

~~~text
道路内容定义的预设术式图
  -> 玩家学习或发现
  -> 复制到个人术式库
  -> 自由增删节点、改接数据流、替换身体指令、调整回路申请
  -> 用当前身体快照编译
  -> 得到可施放的个人术式，或得到可读的编译失败报告
~~~

“自由”指节点组合、分支、循环、输入来源、输出方式和术式结构不受预制技能列表限制；它不允许绕过元素动作、反应阶段、产物类型、副产物去向、通量、回路容量或完成体前置。玩家把道路核心节点从派生图中删除后，术式作为通用个人术式编译，基础效果预算不受惩罚；保留核心节点则必须继续满足该道路声明的身体与反应条件，并接受该节点附带的专精取舍。

道路专精和混搭体系必须按同一效果预算设计：

| 构筑方式 | 获得什么 | 付出什么 | 不能获得什么 |
|---|---|---|---|
| 道路专精图 | 更少的中间节点、稳定的副产物处理、道路独有的输出语义 | 依赖道路核心、优先系统与特定环境；效果偏向窄场景 | 无条件全局伤害、治疗、护盾或资源倍率 |
| 通用个人图 | 任意已开放身体指令的自由组合 | 自己承担完整的连接、控制与副产物设计 | 不自动获得道路独有语义 |
| 混搭联结图 | 两个或更多系统共同生成的复合产物、跨环境适应和多目标解法 | 要求额外桥接、协同回路、稳定器与更严格的副产物处理 | 因为未使用道路图而被削减基础效果预算 |

混搭联结在术式中至少需要两个不同系统的有效身体指令、一个明确的跨系统产物连接，以及所有新增副产物的处理路径。它不是“多放几个元素”的数值奖励，而是让火热压与水性载体形成蒸汽推进、让潮汐修复与土性边界形成共生护壳、让辉光感知与风暴传讯形成导引放电等专精道路无法单独给出的复合语义。

例如沙铠道路的标志图可以从“压力输入 -> 土性定型 -> 沙铠核心节点（将承压产物转为外壳） -> 皮肤输出 -> 沉积回收”开始。该核心节点把预算优先分配给边界稳定、承压与持续时间，并不凭空放大总预算；玩家可以移除它后把同一压力链改为近战冲击、地形支撑或混搭护壳。无论哪种图，缺少皮肤边界、骨骼承压或回收路径时，编译器都必须拒绝或按内容定义编译成明确标注反噬的低功率版本。

## 8.5 一条道路如何使用多个系统

以炎炉道路为例：

~~~text
当前主回路：源质循环，执行“火热压产生 -> 熔血传导 -> 秩序限幅”的爆发反应。
协同回路一：消化炼化，将燃料与矿物转化为可输入循环的热源，并回收炉渣。
协同回路二：骨肌承载，以土性定型承接热压冲击，并把冲击释放为重击。
~~~

道路没有把这三套系统合并成一套。它们仍分别维护通量、风险和器官需求；只有术式图实际接入炎炉核心时，才会把这条已有反应链解释为高热、熔炼和近战输出的专精语义。这个定位也不是永久锁定的：执行矿脉重击时，骨肌承载可以成为当前主回路，源质循环退为“火热压供能”的协同回路；不接炎炉核心的通用或混搭图仍可使用同一组系统，但不会自动获得炎炉语义。

## 8.6 混搭原则

| 情况 | 结果 |
|---|---|
| 主道路 + 优先系统 + 道路核心节点 | 道路偏好动作在正确阶段出现，获得最少节点、最稳定的专精语义；效果预算不因此高于其他合法构筑 |
| 主道路 + 可连续反应的系统 | 可形成混合玩法；若完成混搭联结，可获得复合产物而非只吃部分道路共鸣 |
| 不使用道路核心节点的通用/混搭图 | 仍可调用所有已开放身体指令并使用完整基础效果预算；按自身桥接和副产物链决定强度 |
| 主道路 + 争夺节点的系统 | 可运行但需稳定器、调律、分时调度或降低活跃等级 |
| 两条道路都想做主道路 | 不允许同时拥有；第二条只能作为器官原生标签或混合系统来源 |

道路切换是重大成长选择。建议使用“道路重铸仪式”：先解除主道路锚定，再经过休整、净化、重新启灵和内容验证。不能在战斗或普通菜单中直接切换。

## 8.7 道路在完成体中的职责

完成体不是“道路的四件套”，而是八个器官已经统合后的全身状态。道路在这个阶段的职责是给同一具完整源质身体一个稳定的表达方向：

~~~text
八器官统合：解决“这具身体能否作为一个整体运行”。
道路完成体共鸣：解决“这个整体中哪条完整元素反应最终以何种力量形态向世界表达”。
~~~

因此，任何道路的形态、领域或终局能力都必须同时检查：

1. 八个槽位是否均已源质化。
2. 全身统合桥接是否有效。
3. 主道路是否参与统合核心。
4. 对应道路的优先系统是否已在成长期建立足够成长联结，并完整处理其反应产物和副产物。

代表性成长期反应链：

~~~text
炽热心脏（火·生热升压）
  -> 炉胃囊（火/混沌·燃料转化）
  -> 熔血（水/火·热媒传导）
  -> 净岩晶骨（土·承压定型）
  -> 耐热肺膜或秩序神经（风·泄压 / 秩序·限幅）
= 炎炉循环/炼化双系统成长联结
= 高输出、熔炼增益，需严格管理热负荷与畸变
~~~

## 9.8 后续道路

暗影、混沌、生命工程、古代文明等内容可以在后期扩展为道路，但必须先证明其拥有：

1. 明确的偏好元素动作、反应产物、副产物与风险。
2. 至少两个能完整利用这些动作的独特偏好系统。
3. 与现有道路不同的生态来源和玩法输出。
4. 不依赖“同一套反应链换颜色”的内容重复。

# 十、魔兽血脉、生态与同构源质身体

## 10.1 玩家与魔兽使用同一套力量体系

所有拥有源质能力的魔兽都使用与玩家相同的身体规则：

~~~text
八个功能槽位
  -> 源质器官
  -> 源质生理系统
  -> 成长期联结
  -> 八器官统合
  -> 完成体
  -> 道路预设术式图或天生血脉术式图
  -> 由同一源律术式编译器验证并执行
~~~

玩家不是“获得魔兽技能”，而是把能在魔兽身体中自然运行的源质器官与系统，通过培育、植入和调律重建到自己的身体中。

普通动物可以只有普通生理器官和无源质的休眠状态；魔兽、源质植物、遗迹生体和高阶畸变体必须拥有同构的源质身体状态。

## 10.2 功能槽位而非相同解剖

同一套八槽位是功能模型，不要求玩家和魔兽具有相同外形或器官名称。

| 统一功能槽位 | 人类表现 | 魔兽可能表现 |
|---|---|---|
| 心脏 | 心脏、源核 | 炽核、雷囊、晶核、潮汐泵 |
| 骨骼 | 骨骼与筋束 | 甲壳、晶架、骨板、根系支架 |
| 血液 | 血液、脉流 | 熔血、导电体液、菌液、树汁 |
| 肺部 | 肺、呼吸膜 | 气囊、鳃、喷流囊、换气菌褶 |
| 眼睛 | 视觉器官 | 复眼、热感膜、地脉感受器、观测晶 |
| 神经 | 神经束 | 电传导索、菌丝网络、灵能纤维、控制结 |
| 皮肤 | 表皮、膜、甲层 | 鳞片、甲壳、羽毛膜、菌幕、晶壳 |
| 脏腑/源腺 | 胃、肝、肾、腺体 | 胃囊、孢子腺、滤毒囊、熔炉囊、巢核 |

例如海蛇没有人类肺叶，但其海蛇肺或鳃囊仍占用“肺部”功能槽位；共生菌群没有人形神经，却通过菌丝控制结占用“神经”功能槽位。

## 10.3 共同规则与不同成长入口

| 维度 | 玩家 | 魔兽 |
|---|---|---|
| 初始状态 | 凡躯或少数已源质化器官 | 由物种、年龄、群系和血脉决定 |
| 器官来源 | 植入、培育、调律、遗迹重构 | 天生发育、捕食、蜕变、生态共生、进化 |
| 主道路 | 玩家通过启灵仪式锚定，可低频重铸 | 天生血脉道路，通常由物种定义，Boss 可变异或进化 |
| 回路调度 | 玩家为当前行为选择当前主回路与协同回路 | AI 根据行为、环境和生命阶段选择当前主回路与协同回路 |
| 完成体 | 八器官统合仪式后开始 | 成熟个体、领主、Boss 或生态核心自然/事件性统合 |
| 术式表达 | 玩家编辑、编译和施放个人源律术式 | AI 调用物种预设术式图；高智慧个体可拥有可演化的术式图 |
| 工具表达 | 符文、魔符、武装、祭坛与以太工具 | 巢穴回路、群体共鸣、遗迹接口或有限器官工具结构 |

差异只在成长入口与控制方式；系统匹配、元素反应连续性、血脉兼容、系统压力和完成体条件必须共用同一套计算器。魔兽不会拥有一套绕过器官的“天生元素技能”，其预设术式图同样只能调用由器官动作、系统反应链和当前回路实际提供的能力。

## 10.4 魔兽成长与完成体

魔兽的强度不应只由“物种等级”决定。其成长阶段应映射到与玩家相同的源质身体阶段：

| 魔兽阶段 | 身体状态 | 游戏表现 |
|---|---|---|
| 幼体/普通个体 | 少量源质器官或单一共鸣 | 基础攻击、群系适应、可取样 |
| 成长期个体 | 多个闭环与成长联结 | 明确的元素动作术式、群体地位、环境能力 |
| 成熟魔兽 | 八器官齐备或接近齐备 | 领地能力、稀有组织、可作为精英 |
| 完成体魔兽 | 八器官统合 | Boss、领主、生态核心或可驯养终局伙伴 |
| 畸变完成体 | 统合被污染或冲突劫持 | 高风险 Boss、污染源或特殊剧情对象 |

完成体魔兽和完成体玩家遵循同样的基本条件：八槽位都源质化、桥接连通、道路或血脉核心参与统合、并维持稳定的完整度。区别在于魔兽可以由生态事件、年龄、巢穴或吞食特定猎物满足条件，玩家则通过仪式和主动构筑满足条件。

## 10.5 系统驱动魔兽行为

魔兽 AI 的攻击、移动、环境适应和领地行为应从其系统状态与元素反应链派生，并从物种预设术式图中选择和编译动作，而不是为每个物种单独硬编码一套“魔法技能”。

| 系统 | 魔兽可读取的元素反应 | 行为例子 |
|---|---|---|
| 源质循环 | 炽核生热 -> 熔血传导 -> 风囊泄压；或雷囊释能 -> 神经限幅 | 炽核兽蓄热后喷发，雷鸟短时高功率放电 |
| 骨肌承载 | 心核驱动 -> 神经传令 -> 土性骨架定型 -> 冲击释放 | 岩蜥冲撞、沙虫钻地、甲兽承压 |
| 呼吸交换 | 肺/鳃摄入 -> 体液携带 -> 鳞膜调节 -> 气囊/鳃排出 | 海蛇深潜、雷鸟高空滑翔、火山兽耐烟 |
| 消化炼化 | 矿物/生物质摄入 -> 胃囊火/混沌转化 -> 体液运输 -> 滤囊回收 | 炽核兽吞食矿物充能，菌群分解污染 |
| 净化排异 | 感受器辨识 -> 菌液转运 -> 滤毒囊分离 -> 鳞幕/喷流排出 | 辉光鹿净化巢穴，污染兽排出腐化物 |
| 神经感知 | 感受器光/暗输入 -> 风/秩序传讯 -> 心核供能 -> 反馈校准 | 雷鸟锁定、沙虫感震、观测体扫描 |
| 外皮屏障 | 外界输入 -> 甲壳土性定型 -> 体液修复/风性泄放 -> 节律校准 | 岩蜥结晶护甲、海蛇水压鳞幕 |
| 分泌共生 | 生态信号输入 -> 腺体转化 -> 体液分配 -> 孢子/信息素释放 | 辉光鹿安抚草木、菌群召集孢子、海蛇呼唤鱼群 |

这使玩家看到魔兽使用的能力时，可以通过图鉴和灵能解析反推出其源质器官、系统弱点、可取样组织与成长方向。

## 10.6 血脉档案与身体模板

每个魔兽物种都应有血脉档案和初始身体模板：

| 字段 | 说明 |
|---|---|
| 物种 ID | 稳定内容标识 |
| 血脉标签 | 例如 reptile、aquatic、avian、fungal、relic_biomech |

## 10.8 从魔兽状态生成组织

组织产出必须从魔兽当前源质身体状态派生，而不是只查物种固定掉落表。

~~~text
物种模板决定“可能产出什么”。
个体的器官实例、元素反应贡献、成长、污染、完整度与完成体状态决定“这次实际产出什么品质和风险”。
~~~

| 获取方式 | 规则 |
|---|---|
| 击败获取 | 从被击败个体的有效源质器官、组织角色和完整度生成战利品 |
| 非致命取样 | 不破坏个体统合，得到低量但可持续的样本 |
| 驯养/共生 | 按生态周期提供分泌物、蜕皮、血样或培养条件 |
| 培育复制 | 从样本复现低阶组织；不能无代价复制完成体器官 |
| 遗迹重构 | 生成工程化组织，风险取决于古代身体模板和解析质量 |

完成体魔兽的器官不是“必掉的更大数值物品”。玩家应得到高度受限的核心样本、统合材料、道路线索或生态契约，避免通过刷 Boss 直接复制一具完整完成体。

## 10.9 血脉兼容与生态协作

不要以“同物种才可组合”限制玩家。推荐采用标签规则：

~~~text
同血脉：稳定加成明显。
相邻生态：可组成常规系统，例如海蛇肺与潮汐鱼腺。
跨生态但元素动作可连续：可组成混合系统，增加调律需求。
血脉敌对或生活条件冲突：提高排异、维护或系统压力。
~~~

示例：

~~~text
岩蜥骨 + 沙虫筋束：同荒原生态，适合沙铠承载。
海蛇肺（风/水摄入） + 雷鸟神经（风/秩序传讯）：可做潮汐/风暴混合，但需要秩序节流处理高频信号。
炽核兽心（火产生） + 辉光鹿净化腺（水/秩序过滤）：可做高风险炎炉净化，但必须增加风性泄压，不能无代价消除过热。
~~~

## 10.10 生态不能只是素材矿点

每个关键群系至少应提供：

1. 一种源质材料。
2. 一种运行同构八槽位模型的魔兽或源质生体。
3. 一种生态交互或环境挑战。
4. 一条研究、仪式或工业线索。
5. 一种可持续方式，避免最优策略永远是无差别清图。
6. 一个能让魔兽成长、统合或失控的世界条件，使生态状态直接影响敌人与资源。

# 十一、成长、植入与能级

## 11.1 能级定义

能级描述源质力量的高度；身体阶段描述八器官源质化与统合的完整程度。二者相关，但不能互相替代。

| 能级 | 名称 | 主要解锁 |
|---|---|---|
| 0 | 凡人 | 外源设备、材料准备、基础研究 |
| 1 | 启灵 | 主道路锚定、首个可用当前主回路、个人法力 |
| 2 | 凝辉（成长期） | 多器官成长、首条协同回路、道路深度共鸣 |
| 3 | 心光（完成体起始） | 仅在八器官统合后可进入，解锁统合回路与道路完成体预设术式图 |
| 4 | 领域 | 小范围生态、仪式或机器回路影响 |
| 5 | 文明级 | Boss、文明火种、终局工程与高阶遗迹 |

心光不是“某个核心器官升到足够高等级”即可获得。它必须以八器官源质化和全身统合作为前置。

初版重点只实现凡人、启灵和凝辉的数据与内容，但从 V0.1 起就必须保存八器官统合与完成体所需状态。

## 11.2 身体阶段

玩家与魔兽共享以下身体阶段；玩家通过植入和仪式推进，魔兽通过年龄、生态、蜕变和事件推进。

| 身体阶段 | 源质化槽位 | 玩家表现 | 魔兽表现 |
|---|---:|---|---|
| 凡躯/普通个体 | 0 | 无个人法力 | 无源质能力或只有微弱生态特性 |
| 启灵 | 1 | 锚定道路核心，建立第一通路 | 幼体或低阶魔兽激活天生核心 |
| 成长期 | 2-7 | 逐步形成闭环、身体指令与成长联结 | 成年化、获得系统指令、预设术式图与领地行为 |
| 八器官齐备 | 8 | 获得统合资格，仍有冲突与准备工作 | 成熟精英、领主候选或生态关键个体 |
| 完成体·初统合 | 8 且全身连通 | 统合回路与道路完成体起始能力 | Boss、领主、生态核心或终局伙伴 |
| 完成体·成熟/领域 | 8 且持续成长 | 道路形态、领域与文明级资格 | 区域性天候、巢穴、群落或遗迹影响 |

成长期是整个中段成长过程。它从第二个器官源质化开始，在第八个器官完成源质化时结束；任何局部系统闭环都只能代表成长期进展。

## 11.3 玩家启灵流程

~~~text
选择或发现道路线索
  -> 准备启灵魔药、核心组织、稳定剂和仪式材料
  -> 在祭坛或医疗设施中确认主道路
  -> 植入道路核心并形成第一个系统闭环
  -> 生成源质储备、通量和法力
  -> 开放首个系统身体指令、领取道路启灵预设术式图并进行首次编译
~~~

主道路必须由玩家明确确认，不应因为一次误食或临时器官替换而自动改变。

## 11.4 植入与自然成长生命周期

| 阶段 | 玩家 | 魔兽 |
|---|---|---|
| 获得组织 | 采样、交易、击败、培育 | 天生发育、吞食、蜕变、生态共生 |
| 解析 | 判断角色、血脉、系统资格与兼容 | 由物种模板和生态状态确定 |
| 成长 | 用材料、使用、研究和调律提高组织等级 | 由年龄、食物、领地、群落和环境提高 |
| 回路调度 | 玩家配置当前主回路、协同回路与通量预设 | AI 按行为与环境自动调度相同的回路结构 |
| 统合 | 在祭坛或医疗设施完成仪式 | 在巢穴、遗迹、天候或事件中自然/强制统合 |
| 风险处理 | 净化、移除、重构、休整 | 躲避、迁徙、沉眠、畸变、领地失控 |

## 11.5 槽位源质化与成长期节奏

源质槽位不是生理器官凭空长出，而是普通身体部位被稳定地源质化。玩家必须依次解决身体短板，而不能只用四个强器官宣告构筑完成。

| 成长期节点 | 源质化槽位 | 体验与目标 |
|---|---|---|
| 启灵核心 | 1 | 建立道路锚定与第一通路 |
| 初期成长 | 2-3 | 形成一条正式系统闭环 |
| 中期成长 | 4-6 | 当前主回路与协同回路形成成长联结 |
| 后期成长 | 7 | 补齐最缺失的环境、过滤、感知或承载职责 |
| 八器官齐备 | 8 | 验证四条统合桥接，准备全身统合仪式 |
| 初统合 | 8 且连通 | 开始完成体成长，不再只是追加单个器官 |

## 11.6 全身统合仪式

全身统合仪式是玩家进入完成体的唯一标准入口。它不能被普通机器或单次消耗品跳过。

~~~text
验证八个源质器官与最低完整度
  -> 评估四条统合桥接、全身反应连续性、节点竞争、血脉关系和系统压力
  -> 准备道路核心、统合稳定剂、生态催化物和高阶符文
  -> 在对应祭坛或医疗设施中确认
  -> 执行有恢复风险的全身统合仪式
  -> 写入完成体·初统合状态，开放全身身体指令并解锁道路完成体预设术式图
~~~

如果统合失败，不应随机永久损毁玩家；应按预览生成可恢复的后果，例如高畸变、器官完整度下降、统合冷却、特定桥接断裂或暂时排异。

## 11.7 能力来源分工

| 能力来源 | 应该提供什么 |
|---|---|
| 系统身体指令 | 有生理来源的输入、转化、控制、承压、泄压、输出和回收操作 |
| 个人源律术式 | 玩家自由编辑并编译的行动能力；调用系统指令、反应产物和当前主/协同回路 |
| 统合回路 | 八器官协调、全身风险调度和完成体公共身体指令 |
| 道路预设术式图 | 道路独有的学习脉络、核心节点和完成体图；可复制为个人术式再改写 |
| 源律武装 | 符文/魔符驱动的有限工具术式，不是个人术式编辑器 |
| 祭坛仪式 | 定点、慢速、强战略效果 |
| 以太设备 | 工业化、持续性、规模化能力 |
| 装备 | 外部强度、工具功能、可替换模块 |

这样可以避免所有能力都塞入一个“固定器官技能列表”，也避免把人物自由术式错误塞进符文和魔符的工具插槽。

## 11.8 示例成长路线

### 沙铠启灵到成长期

~~~text
岩蜥组织样本 + 土源质 + 秩序稳定剂
  -> 岩核心脏
  -> 净岩晶骨
  -> 沙铠承载系统闭环
  -> 获得沙铠启灵术式图，编译岩盾、采矿或承压变体
  -> 后续补齐血液、肺部、眼睛、神经、皮肤与脏腑
  -> 八器官齐备后才可进行山岳铠躯统合
~~~

### 潮汐启灵到成长期

~~~text
海蛇肺样本 + 潮汐腺 + 水源质 + 活化培养液
  -> 海蛇肺
  -> 活化血
  -> 潮汐交换系统闭环
  -> 获得潮汐启灵术式图，编译水下适应、恢复或净化变体
  -> 后续补齐心脏、骨骼、眼睛、神经、皮肤与脏腑
  -> 八器官齐备后才可进行潮生躯体统合
~~~

### 风暴启灵到成长期

~~~text
雷鸟神经样本 + 风暴气囊 + 风源质 + 秩序节流符
  -> 雷鸣神经
  -> 风暴肺
  -> 风暴感知系统闭环
  -> 获得风暴启灵术式图，编译冲刺、锁定或电弧变体
  -> 后续补齐心脏、骨骼、血液、眼睛、皮肤与脏腑
  -> 八器官齐备后才可进行雷云躯体统合
~~~

# 十二、稳定、畸变与系统压力

## 12.1 四个核心状态

| 状态 | 本质 | 主要来源 | 玩家影响 |
|---|---|---|---|
| 稳定度 | 网络安全程度 | 品质、反应连续性、调律、血脉兼容 | 术式编译/施放成功、恢复、系统效率 |
| 畸变度 | 污染与错误转化积累 | 污浊源质、反应冲突、失败仪式 | 异常、器官损伤、畸变事件 |
| 精神负荷 | 信息处理与控制压力 | 神经感知、古代解析、复杂术式 | 解析失败、感知噪声、施法迟滞 |
| 系统压力 | 某个系统的即时负担 | 过热、缺氧、过滤饱和、共生失衡 | 对应系统降级、反噬或停机 |

稳定、畸变和精神负荷是全局值。系统压力是按系统或压力类型维护的局部状态，不应在 HUD 上堆成八条长期资源条。

## 12.2 稳定度

稳定度主要由以下因素计算：

~~~text
器官品质与成长
+ 系统角色完整度
+ 元素动作与反应阶段连续性
+ 产物、副产物与缓冲路径完整度
+ 血脉与生态兼容
+ 调律、秩序符文与稳定器
- 污染
- 严重冲突
- 当前系统压力
~~~

稳定度低时的表现应逐步升级：

| 区间 | 表现 |
|---|---|
| 81-100 | 正常运行 |
| 61-80 | 轻微效率下降，显示提示 |
| 41-60 | 已编译术式增加冷却或成本 |
| 21-40 | 明显失误、系统能力降级 |
| 0-20 | 强制暂停高负荷系统、可能触发排异或畸变事件 |

## 12.3 畸变度

畸变不是随机惩罚条，而是高收益方案的长期成本。

| 来源 | 例子 |
|---|---|
| 污染材料 | 直接使用污浊源质或畸变组织 |
| 反应节点冲突 | 火热压与水稀释同时争夺同一产生节点，或秩序限幅与混沌倍增同时控制同一转化节点 |
| 副产物淤积 | 热压、污染、盐分或错误信号没有泄放、过滤、回收或校准去向 |
| 血脉排斥 | 强敌对血脉未经调律直接拼接 |
| 过载行为 | 反复强行维持高压力系统 |
| 仪式失败 | 在不满足预览条件时启动高风险仪式 |
| 遗迹污染 | 解析未知古代组织或文明核心 |

畸变处理方式：

~~~text
净化排异系统：降低积累速度与清除低阶污染。
调律药剂与仪式：修正允许的元素动作阶段、分时调度节点或补足短期缓冲。
休整与生态维护：恢复系统压力，不能瞬间抹除全部代价。
移除器官：可中止污染源，但有恢复期和资源损失。
~~~

## 12.4 精神负荷

精神负荷主要服务神经感知、辉光道路、遗迹和复杂武装。它不应被所有玩家都当作常驻惩罚。

| 来源 | 后果 | 解决方式 |
|---|---|---|
| 高频锁定、解析 | 感知噪声、目标误判 | 降低感知系统功率、休整 |
| 古代遗迹信息 | 幻象、仪式预览不完整 | 辉光净化、研究设备、团队协作 |
| 混沌或暗系混搭 | 错误信号、冲动释放 | 秩序稳定器、降低通量 |
| 同时调度过多回路 | 控制迟滞 | 保留一个当前主回路，只接入本动作需要的协同回路 |

## 12.5 系统压力类型

| 压力 | 对应系统 | 典型来源 |
|---|---|---|
| 热负荷 | 循环、炼化、屏障 | 炎炉输出、热武装、火山环境 |
| 结构疲劳 | 骨肌、屏障 | 重击、重型装备、坠落 |
| 交换压力 | 呼吸 | 水压、毒雾、高空、真空遗迹 |
| 代谢过载 | 炼化 | 高阶魔药、污染吸收、快速培养 |
| 过滤饱和 | 排异 | 持续毒素、污染区、净化队友 |
| 感知噪声 | 神经感知 | 高频解析、雷暴、古代信号 |
| 屏障破裂 | 外皮 | 连续受击、元素侵蚀 |
| 共生失衡 | 分泌共生 | 生态条件缺失、血脉不匹配 |
| 副产物淤积 | 任意高负荷反应 | 余热、压差、污染、盐分或错误信号缺少处理路径 |

## 12.6 低频诊断日志

不确定的系统匹配、数值异常和内容错误必须优先用低频日志定位，不在每 tick 输出高频日志。

建议记录：

~~~text
系统首次形成、降级、失效或恢复。
元素反应链首次连续、断裂、节点竞争或副产物无去向。
畸变阶段变化。
主道路锚定、重铸或验证失败。
内容定义中的未知标签、循环配方或非法槽位。
器官植入、移除、调律和持久化失败。
~~~

不记录：

~~~text
每 tick 的通量。
每帧的法力。
每次 UI 刷新。
正常状态下的重复系统评估。
~~~

# 十三、魔药、培育与调律

## 13.1 魔药类型

| 类型 | 用途 |
|---|---|
| 启灵魔药 | 锚定主道路并帮助建立首个闭环 |
| 植入魔药 | 降低特定器官移植的排异风险 |
| 培育催化剂 | 决定器官品质、血脉保留、组织角色与可形成的元素反应贡献 |
| 调律药剂 | 调整允许的元素动作阶段、血脉或系统角色间的兼容 |
| 净化药剂 | 降低污染和低阶畸变 |
| 晋升魔药 | 满足下一能级的身体与源质门槛 |
| 应急抑制剂 | 短时压低系统压力，带后续恢复代价 |

## 13.2 器官培育原则

~~~text
完整器官不是普通材料的直接合成品。
高阶器官必须继承来源血脉、生态条件和成长过程。
同一组织在不同培养条件下可成为不同系统角色，也可在受控范围内获得不同的元素反应贡献。
~~~

例如海蛇来源的腺体：

| 培育条件 | 结果 |
|---|---|
| 水源质 + 活化培养液 | 潮汐腺，水性携带/回收，偏呼吸与共生 |
| 水源质 + 光符文校准 | 净潮腺，水性转运 + 光性辨识，偏净化排异 |
| 水源质 + 风压培养 | 喷流腺，水性蓄压 + 风性释放，偏水下机动 |

## 13.3 标准培育流程

~~~text
组织样本
  -> 血脉与污染检测
  -> 源质萃取与纯度选择
  -> 培养液、符文和生态条件配置
  -> 培育舱或祭坛完成组织成长
  -> 生成器官定义允许的实例
  -> 玩家确认植入与调律
~~~

## 13.4 植入不是吞食

旧式“直接吞噬器官”可保留为极高风险的野外应急行为，但不作为标准成长路径。

标准流程必须是：

~~~text
取得样本 -> 处理与培育 -> 预览系统变化 -> 确认植入 -> 恢复与调律
~~~

这样玩家能够理解器官会改变哪些元素反应阶段、系统、道路共鸣、风险和生态关系。

## 13.5 调律与重构

| 操作 | 效果 | 限制 |
|---|---|---|
| 调律 | 调整允许的反应阶段、动作阈值或分时顺序，降低短期节点竞争 | 不改变器官血脉本质，不能凭空新增不支持的元素动作 |
| 净化 | 清理污染与低阶畸变 | 不能免费修复高阶严重冲突 |
| 重构 | 改变器官组织角色或元素反应贡献 | 需要配方、研究和恢复期，必须重新验证所有受影响反应链 |
| 移除 | 取消一个器官实例 | 系统可能失效，需休整 |
| 道路重铸 | 改变主道路 | 重大仪式，不能作为普通换装 |

# 十四、个人源律术式与工具术式

## 14.1 个人术式与工具术式的边界

人物的源律术式与符文、魔符、武装必须是两条独立系统：

| 系统 | 承载位置 | 自由度 | 核心用途 | 不承担什么 |
|---|---|---|---|---|
| 个人源律术式 | 玩家自身的术式库与源质身体 | 自由图式编程；可自行增删节点、改线、分支和组合系统 | 把身体已具备的元素动作、反应产物和回路服务编译为行动 | 不制造缺失的器官、元素动作或副产物处理链 |
| 符文/魔符工具术式 | 武装、工具、祭坛、机器的有限插槽 | 受槽位、设备接口、供能、配方和物品等级限制 | 让外部工具稳定执行固定或有限组合的功能 | 不保存或编辑人物个人术式源图 |

人物可以徒手或借助身体本身施放个人源律术式。武装可以作为瞄准、接触、承载或外源供能工具，但不能成为个人术式的语法容器；一把装满魔符的武器只拥有自己的有限工具功能，不会替玩家绕过个人术式编译。

## 14.2 个人源律术式的源图

个人源律术式是一张有类型端口的源图。它不是固定技能模板，也不是把元素和器官填进若干加成槽，而是把已由身体 evaluator 证明存在的能力连接成一段可执行程序。

| 节点类别 | 来源 | 作用 |
|---|---|---|
| 输入节点 | 反应产物、环境载体、目标信息、已储存法力 | 声明术式从何处取得可用资源或信息 |
| 身体指令节点 | 已闭环系统或统合回路 | 执行产生、携带、转化、聚焦、承压、泄压、过滤、回收等真实生理操作 |
| 道路核心节点 | 当前道路的已解锁内容 | 对既有产物进行道路语义偏转；删除后术式可继续作为通用图，基础效果预算不受惩罚，但失去道路标志语义 |
| 控制节点 | 神经感知、灵能、术式自身的条件/分支/顺序结构 | 选择目标、控制时序、分支与反馈；不能把控制信号当作能量来源 |
| 协同服务节点 | 已申请的协同回路 | 声明承压、泄压、过滤、环境接口、稳定或副产物接收服务 |
| 输出节点 | 身体接触点、环境、目标或合法外部接口 | 将已有产物释放为冲击、护壳、位移、修复、揭示、共生信号等结果 |

图式的自由度来自节点连接和控制流，而不是无限无类型节点。玩家可以让炎炉循环为骨肌冲击供能、让潮汐交换处理炎炉余热、让辉光感知控制风暴放电的目标选择；但每一次跨系统连接都必须在图中写出输入、产物、协同回路和副产物去向。

道路预设术式图是个人源图的第一批教材和原型。玩家复制后可任意改写，保存的始终是源图而非某次身体配置下的固定数值。器官更换、系统断裂、道路重铸或统合状态变化时，源图仍保留，但已编译结果会失效并要求重新编译。

## 14.3 编译、缓存与施放

~~~text
个人术式源图
  + 当前不可变身体能力快照
  + 当前道路的核心节点与反应偏好
  + 内容定义快照
  -> 静态类型与控制流检查
  -> 验证每个身体指令、反应产物和副产物处理链
  -> 推导当前主回路、协同回路、通量、法力、冷却与风险
  -> 生成已编译术式或可读失败报告
  -> 施放时只复核动态状态、回路可用性、目标和环境
~~~

编译器必须至少给出以下失败原因：缺失身体指令、元素动作阶段不连续、产物类型不匹配、协同回路未闭环、副产物无去向、道路核心节点条件不满足、统合前调用完成体节点、通量不足、控制流越界或内容版本不兼容。只有内容显式声明存在低功率路径时，编译器才可生成带反噬或效率惩罚的降级版本；默认行为是拒绝编译，而不是暗中把术式改成另一个技能。

为避免每帧重新编译，个人术式在编辑、器官植入/移除、调律、道路重铸、系统状态变更、统合状态变更或内容版本变更后失效。重新编译产生可缓存的值对象；施放时只进行低成本的动态检查和回路调度申请。诊断日志只记录编译首次成功、失效、失败原因变化和运行时拒绝，不记录每次节点执行。

## 14.4 符文定位

符文是可储存、可组合、可工业化生产的工具侧源律规律片段。它们用于设备、武装、祭坛、培育与机器，不是人物个人源律术式的节点或编译材料。

~~~text
源质材料 -> 提纯、雕刻、蚀刻、融合 -> 符文
符文 -> 魔符、祭坛、培育校准、机器配方、遗迹解锁
~~~

| 等级 | 名称 | 强度定位 |
|---|---|---|
| 1 | 普通 | 前期工具、基础术式 |
| 2 | 精良 | 稳定构筑、系统辅助 |
| 3 | 卓越 | 高阶仪式、全身完成体强化 |
| 4 | 传奇 | 遗迹、领域、文明级内容 |

基础数据仍可沿用八元素乘四等级，但每个符文必须声明自己调用、限制或放大的元素生理动作与反应阶段；实际内容无需在第一版一次实装全部组合。

## 14.5 魔符类型

| 类型 | 作用 |
|---|---|
| 形态魔符 | 决定弹射、自施、范围、射线、触碰等送达方式 |
| 效果魔符 | 把已有反应产物转为燃烧、治疗、护盾、位移、揭示等外部效果；不能创建缺失元素动作 |
| 增幅魔符 | 决定范围、速度、效率、持续、穿透、节能或变数 |
| 系统魔符 | 只在工具装配时满足指定系统反应链、产物与副产物处理条件才可启用，提供身体与武装联动 |

## 14.6 源律武装定位

源律武装是有限工具术式载体，不是独立力量源，也不是人物个人术式源图的存储或编辑器。

~~~text
普通人 + 源律武装 = 可装配、维护或使用外源供能的工具功能。
升华者 + 源律武装 = 用个人法力与系统反应产物驱动已装配的有限工具术式；个人源律术式仍由身体直接编译。
以太武装 + 以太核心 = 使用工业能源运行高阶功能，但仍受设备过载约束。
~~~

## 14.7 系统与武装共鸣

武装只能调用系统已经完成的产物，并把它转为可配置的送达形态：

| 系统 | 可调用反应产物 | 武装共鸣例子 |
|---|---|---|
| 源质循环 | 稳定通量、可控热压、循环缓冲 | 降低法力波动、提高爆发窗口、影响过热 |
| 骨肌承载 | 定型承压、冲击释放 | 提高重武器、采矿武装和近战导具承载 |
| 呼吸交换 | 可用环境载体、喷流或压差 | 提高喷射、滑翔、水下或毒区武装稳定性 |
| 消化炼化 | 精炼药效、可用热源、处理后的材料 | 提高药剂弹、热源工具、消耗品利用 |
| 净化排异 | 已分离污染、封界缓冲 | 降低污染术式、诅咒与遗迹武装反噬 |
| 神经感知 | 校准信号、锁定信息 | 改善锁定、追踪、施法速度和解析 |
| 外皮屏障 | 表层护盾、承压边界 | 改善护盾、铠装和近距离术式安全性 |
| 分泌共生 | 受控分泌物、生态信号 | 允许与共生生物、诱饵、侦察器协作 |

## 14.8 道路对同一武装的不同解释

同一个“范围水系武装”在不同道路下可以不同：

| 道路 | 必需反应 | 效果形态 |
|---|---|---|
| 潮汐 | 风/水摄入 -> 水性携带/回收 | 治疗潮、净化水域 |
| 风暴 | 风性传讯/喷流 -> 水性载体 -> 火性释能或秩序限幅 | 高速水刃、蒸汽推进 |
| 辉光 | 光性辨识 -> 水性转运 -> 秩序过滤封界 | 驱散污染、显影敌人 |

这保证武装构筑仍有可配置性，但不会脱离身体网络，也不会与人物个人术式图混为同一套自由编程系统。

## 14.9 魔符消耗规则

是否保留一次性魔符应由内容类型决定：

| 类型 | 建议 |
|---|---|
| 基础形态与效果 | 可重复安装，降低新玩家成本 |
| 高阶增幅与遗迹魔符 | 允许消耗式，形成长期资源决策 |
| 仪式专用魔符 | 通常消耗 |
| 系统魔符 | 高价值，受系统与道路条件限制 |

不建议把所有魔符都做成安装即销毁，否则玩家会因试错成本过高而不愿探索系统组合。

# 十五、祭坛、仪式与基地回路

## 15.1 祭坛定位

祭坛是大型、定点、慢速的源质回路。它服务于身体无法独自完成的调律、研究、基地和世界效果。

| 仪式类别 | 作用 |
|---|---|
| 道路锚定 | 启灵、道路重铸、道路深度共鸣 |
| 系统调律 | 预览、稳定、调度当前主回路、调整反应阈值或降低节点竞争 |
| 器官培育 | 提供特殊生态或古代源质条件 |
| 净化排异 | 清理污染、处理高风险移植 |
| 机器祝福 | 加速、冷却、增产、校准 |
| 世界仪式 | 传送、群系天气、遗迹门、生态修复 |
| 灵能解析 | 显示遗迹、蓝图和隐藏系统信息 |

## 15.2 祭坛执行流程

~~~text
放入符文、组织、稳定剂或以太组件
  -> 实时显示可满足的仪式与缺失条件
  -> 显示道路、系统、器官、元素反应链与风险变化预览
  -> 玩家在场确认
  -> 服务器验证资源、世界状态、内容定义和权限
  -> 执行有时长的仪式
  -> 原子提交结果并发布值事件
~~~

## 15.3 仪式示例

| 仪式 | 条件 | 效果 |
|---|---|---|
| 秩序调律 | 秩序符文 + 稳定剂 + 当前主回路 | 暂时降低节点竞争或提高反应阈值，不能永久抹除畸变 |
| 潮汐锚定 | 水/光符文 + 海蛇组织 + 水域 | 潮汐道路深度共鸣或水域适应 |
| 炎炉节流 | 火/秩序符文 + 净岩组织 | 把部分热压导入土性沉积回收，降低热负荷并限制爆发上限 |
| 血脉桥接 | 两种兼容样本 + 生态催化物 | 允许特定混合系统成长联结，并可作为全身统合桥接材料 |
| 辉光解析 | 光/秩序符文 + 观测组织 | 提升遗迹预览与隐藏条件显示 |
| 群落修复 | 水/光/土符文 + 共生组织 | 修复受损生态，获得长期材料收益 |

## 15.4 祭坛与自动化的边界

材料供应、台座摆放和预检测可以自动化；玩家确认、道路变化、器官植入、晋升和高风险仪式不能完全自动化。

# 十六、以太科技、工业与自动化

## 16.1 以太定位

以太是从源质材料工业稳定化得到的机器能源与工程介质。它不等于电力，也不等于法力。

~~~text
普通工业能源：驱动一般机器。
源质材料：可被处理的超凡物质。
以太：源质材料的工业稳定化产物。
法力：人体安全输出缓冲。
~~~

## 16.2 工业机器链

| 机器 | 功能 |
|---|---|
| 源质萃取器 | 从矿物、植物、魔兽材料提取粗源质 |
| 源质离心机 | 分离元素组分、污染和活化组分，为器官反应贡献提供培养原料 |
| 组织分析仪 | 读取血脉、角色、元素反应贡献、生态标签和污染 |
| 器官培育舱 | 在受控条件下培育基础到高阶器官 |
| 魔药反应釜 | 制作启灵、植入、调律、净化和晋升药剂 |
| 符文蚀刻机 | 工业生产符文 |
| 符文融合器 | 提高符文等级或构造复合符文 |
| 符文转印机 | 把符文转为魔符或器官校准组件 |
| 以太转化炉 | 把源质材料转为以太 |
| 以太锻炉 | 制造以太武装、稳定器和高级设备 |
| 灵能解析仪 | 解析遗迹蓝图与古代组织 |
| 生态培养器 | 维持共生菌群、植物和非致命取样生态 |

## 16.3 系统对工业的影响

身体系统不应成为“免费替代机器”的方式。它们提供的是少量个人增益、操作资格或特殊效率。

| 系统 | 合理工业联动 |
|---|---|
| 炎炉循环 | 主持高热仪式、降低个人操作反噬 |
| 沙铠承载 | 使用重型工程装备、维护承压设备 |
| 潮汐交换 | 水域、冷却和液体机器环境作业 |
| 辉光感知 | 识别配方异常、遗迹设备诊断 |
| 净化排异 | 在污染材料处理中降低玩家风险 |
| 分泌共生 | 维持生态培养、驯养物种与生物处理线 |

机器的生产速度、能源、配方和库存仍由工业系统拥有。

## 16.4 以太武装与过载

| 模块 | 作用 |
|---|---|
| 以太核心 | 提供工业能量等级 |
| 源质接口 | 决定可调用元素或系统共鸣 |
| 符文槽 | 定义术式规律 |
| 魔符槽 | 定义主动释放形态 |
| 稳定槽 | 降低过载、污染或反噬 |
| 输出槽 | 增加伤害、采矿或设备效率 |
| 辅助槽 | 提供移动、探测、保护等功能 |

过载必须由设备温度、以太负载和玩家系统稳定共同决定。强大的身体可以降低某些反噬，但不能无限消除设备损耗。

## 16.5 自动化边界

| 可自动化 | 必须保留玩家参与 |
|---|---|
| 萃取、离心、材料运输、基础符文生产 | 启灵、道路锚定、道路重铸 |
| 基础魔药、培养液、低阶组织培育 | 器官植入与高阶重构 |
| 仪式材料摆放、预检、配方物流 | 高风险仪式确认 |
| 以太生产、武装零件制造 | 遗迹解析决策、Boss 组织获取 |
| 共生生态的基础维护 | 血脉桥接和全身统合/完成体解锁 |

AE2 或同类物流系统应帮助材料管理，但不能吞掉身体构筑的关键决定。

# 十七、世界、遗迹与灵能研究

## 17.1 群系设计

| 群系 | 主要资源 | 生物与系统线索 | 主要道路 |
|---|---|---|---|
| 晶岩荒原 | 土源质、晶化骨、稳定剂 | 岩蜥、沙虫、承压与采矿 | 沙铠 |
| 风暴高原 | 风源质、磁晶、导电组织 | 雷鸟、高空交换、感知 | 风暴 |
| 潮汐海岸 | 水源质、活化液、海生组织 | 海蛇、呼吸与共生 | 潮汐 |
| 炎炉裂谷 | 火源质、熔血、热核 | 炽核兽、炼化与循环 | 炎炉 |
| 辉光森林 | 光源质、净化花粉、活化植物 | 辉光鹿、净化与感知 | 辉光 |
| 污染沼泽 | 污浊源质、菌群、过滤组织 | 共生菌群、排异与高风险炼化 | 混合内容 |
| 古代遗迹 | 古代源质、蓝图、观测组织 | 观测体、神经感知与研究 | 全道路后期 |

## 17.2 遗迹定位

遗迹不是单纯战利品房间，而是旧文明的高阶源律工程。它应提供：

1. 古代器官与系统定义的线索。
2. 全身完成体的道路表达、统合仪式或领域结构。
3. 以太设备与工业蓝图。
4. 灵能负荷、污染和守卫构成的风险。
5. 对生态失衡、文明历史和源质本质的解释。

## 17.3 灵能等级

| 等级 | 玩家可读取的信息 |
|---|---|
| 0 | 只能看到器官基础描述 |
| 1 | 看见系统是否可形成与基础身体指令 |
| 2 | 看见缺失角色、基础反应阶段连续性、术式端口类型和压力类型 |
| 3 | 看见道路核心语义、混搭联结候选、血脉冲突与仪式大致结果 |
| 4 | 看见全身统合与完成体条件、遗迹蓝图关联、术式编译详细副作用 |
| 5 | 看见隐藏配方、文明级路线、高阶生态关系与古代术式图结构 |

## 17.4 研究主线

~~~text
发现组织或遗迹
  -> 灵能或机器解析
  -> 解锁血脉、系统、身体指令、道路预设术式图、混搭联结、仪式或工业定义
  -> 获取配方与生态条件
  -> 实际培育、植入或建造
~~~

研究不能直接给玩家永久面板强度。它应主要解锁理解、配方、预览和新的可构筑空间。

1. 为什么该系统形成了。

## 18.9 系统评估与调度

系统评估必须在内容快照上运行，并与 ECS 线程模型一致：

~~~text
主线程捕获玩家与魔兽的 SourceLawBodyState 及不可变内容快照
  -> worker 先匹配器官角色与 elemental_contributions，再计算元素反应链、系统、统合、通量、风险和派生结果
  -> barrier 后按稳定实体标识应用结果
  -> 生成 SourceLawBodyCapabilitySnapshot 并使相关已编译术式缓存失效
  -> 提交事件、存档脏标记、任务事件和受控的展示值
~~~

worker 任务不得保存 World、registry、UI、脚本 VM 或网络对象引用。系统输出应是值类型，便于测试和确定性排序。body evaluator 必须先生成 ElementalReactionReport，再识别有效系统、身体指令、可用反应产物和统合状态；它不编译某张个人术式图。个人术式 compiler 使用这份不可变能力快照，在编辑、缓存失效或显式重新编译时生成 `CompiledSourceLawSpell`；施放 runtime 只申请已声明的当前主/协同回路并检查瞬时资源、目标和环境。玩家与魔兽不能各自拥有一个规则分支，行为差异只能出现在源图来源、回路调度策略、AI 行为和内容模板层。

## 18.10 事务边界

以下操作必须由服务器以事务形式完成：

| 操作 | 原子性要求 |
|---|---|
| 植入器官 | 扣除材料、验证槽位与元素反应贡献、写入器官、重新评估反应链和系统、递增 body_revision 并使受影响术式缓存失效 |
| 移除器官 | 验证恢复期、移除实例、降级反应链/系统、产生副产物、递增 body_revision 并使受影响术式缓存失效 |
| 调律/净化 | 扣除材料、更新状态或允许的反应贡献调律、验证畸变、反应连续性和风险、递增 body_revision 并使受影响术式缓存失效 |
| 道路锚定/重铸 | 验证仪式、更新主道路、重新计算反应偏转与共鸣、使包含道路核心节点的术式缓存失效 |
| 编辑个人术式源图 | 验证节点 ID、端口连接、控制流上限与内容引用，写入新的 source_revision；允许保存暂时不满足当前身体的源图，但必须标记为未编译 |
| 编译个人术式 | 在不可变身体/内容快照上生成 CompileReport 与可丢弃缓存；失败只更新诊断，不替换玩家源图 |
| 当前主回路调度 | 验证调度冷却/休整、更新当前主回路与协同回路的通量分配 |
| 仪式完成 | 验证世界、材料、玩家状态并一次提交结果 |
| 魔兽自然成长/统合 | 验证物种模板、生态条件、年龄/行为事件并更新共享身体状态 |

## 18.12 内容验证与测试

内容加载时必须验证：

1. 器官槽位合法、角色非空，每项 elemental_contribution 的元素、动作、阶段和容量均有效，且通过对应 ElementalPhysiologyRule。
2. 每个系统配方有至少一个核心、一个导流和一个效应路径，并引用存在的 ElementalReactionDefinition。
3. 每条正式反应链至少覆盖来源/摄入、传导/转化、作用/释放三个阶段；需要处理副产物时，必须存在过滤、排出、校准或回收步骤。
4. 同一反应节点的竞争动作、不可连续动作和无去向副产物都有可展示的失败原因或内容定义的缓冲路径。
5. 系统成长联结不依赖不可达的生态条件。
6. 全身统合定义覆盖八个槽位和四条必需桥接，包含全身元素反应连续性，且不会把局部闭环判为完成体。
7. 每条道路都声明启灵、系统、标志和完成体预设术式图，图中的道路核心节点、反应偏转和完成体前置存在，且不会修改缺失动作或为未使用核心节点的通用图添加隐藏倍率。
8. 每个混搭联结至少引用两个不同系统的有效身体指令、一个可追溯跨系统产物连接和完整副产物处理；不得只因多个元素标签同时出现就成立。
9. 每张预设术式图、魔兽天生术式图和玩家术式源图都通过节点唯一性、端口类型、控制流边界、引用合法性、回路申请和完成体门槛验证。
10. 每个 ToolSpellAssemblyDefinition 的槽位数、符文/魔符标签、供能和外部接口均有效，且没有接受个人术式源图或道路核心节点作为装配项的路径。
11. 每个魔兽身体模板都能映射八个功能槽位，初始反应链与天生术式图可由候选器官实际满足，并复用共享系统和术式定义。
12. 血脉关系不形成无意循环或全局互斥死锁。
13. 任何可执行术式都能说明其系统反应、统合、道路核心或混搭联结前置条件。
14. 存档 payload 的 schema、版本、大小与未知字段处理明确。

测试至少覆盖：

~~~text
八槽位序列化与非法槽位拒绝。
系统共鸣、闭环、成长联结与失效原因。
器官元素反应贡献、阶段连续、节点竞争、副产物处理与可读修复建议。
八器官齐备、四条统合桥接、完成体进入与完成体中断。
道路共鸣与混合系统。
道路预设图复制、删除道路核心后的通用编译，以及道路核心不提高基础效果预算。
两系统以上的混搭联结、复合产物、副产物处理与同预算比较。
个人术式源图的端口类型、控制流、缓存失效、失败报告与运行时回路申请。
工具术式槽位上限、符文/魔符限制以及与个人术式源图完全隔离。
元素动作竞争、严重反应冲突及稳定器缓冲。
血脉兼容与生态条件。
植入、移除、调律和道路重铸事务。
玩家与魔兽在同一身体模板下获得相同系统评估结果。
玩家个人术式与魔兽天生术式图在同一能力快照下获得同样的编译规则。
魔兽自然成长、领主统合和畸变完成体事件。
worker 快照评估的确定性。
低频日志只在状态转换时触发。
~~~

## 18.13 旧接口清理

新实现完成后应删除或停止编译以下旧概念的生产路径：

# 十九、UI、反馈与诊断

## 19.1 HUD

HUD 只显示即时战斗和风险信息：

~~~text
生命。
法力。
当前主回路图标、协同压力与调度冷却警告。
当前道路标志。
当前已装备个人术式的冷却、编译状态与当前主回路申请警告。
畸变或精神负荷的阶段性警告。
~~~

不在 HUD 上长驻显示八个器官、所有系统和全部资源条。

## 19.2 升华者面板

面板推荐分为六个固定页签：

| 页签 | 内容 |
|---|---|
| 身体 | 八槽位、器官品质、成长、污染、角色标签、元素反应贡献 |
| 系统 | 反应链、共鸣/闭环/成长联结、当前主回路、协同回路、通量、产物/副产物、失效原因 |
| 统合 | 八器官齐备状态、四条统合桥接、完成体阶段与阻塞原因 |
| 术式 | 个人术式库、可用身体指令、源图编辑、编译报告、已装备术式与混搭联结 |
| 道路 | 主道路、预设术式图、道路核心节点、共鸣语义、重铸条件 |
| 风险与研究 | 稳定、畸变、精神负荷、压力、血脉与生态知识 |

系统页必须能清楚显示：

~~~text
这个系统为何成立。
哪些器官充当核心、导流、效应与稳定。
每个器官在产生/摄入、传导、转化、作用/释放、校准/回收中的元素动作。
该系统的反应产物、副产物、当前泄压/过滤/回收去向。
道路偏转了哪一个已有动作、阶段或产物。
该闭环开放了哪些身体指令、输入/输出端口和副产物服务。
当前缺少什么才能形成成长联结。
当前为何被调度为当前主回路，哪些协同回路正在提供什么支持。
为何当前无法激活或被降级。
~~~

术式页必须把“源图可自由编辑”与“身体规则不可绕过”同时表现出来：

~~~text
术式源图的输入、身体指令、道路核心、控制流、协同服务与输出节点。
每条连线的端口类型、实际提供它的器官/系统、当前主回路与协同回路申请。
可用效果预算来自哪一段反应，以及道路核心或混搭联结如何分配该预算。
可直接定位的编译错误：缺失节点、类型不匹配、反应断裂、副产物无去向、回路不足、完成体未解锁。
道路预设图的原始版本、个人复制版本和差异；删除道路核心后仍可使用的通用图状态。
~~~

编辑器应允许玩家保留暂时无法编译的源图，并以清晰诊断而不是删除内容的方式提示缺失身体条件。工具侧魔符编辑必须使用独立的有限插槽界面，不能进入个人术式页。

统合页必须单独显示：

~~~text
八个槽位中哪些尚未源质化。
能量与炼化、控制与反馈、环境与屏障、承载与行动四条桥接的状态。
哪些元素反应阶段、血脉、完整度或压力阻止统合。
全身反应中尚未处理的产物、副产物或竞争节点。
完成全身统合仪式后会解锁或暂停什么。
~~~

## 19.3 器官槽位表现

八槽位应以身体示意而不是八个同样的背包格展示。每个槽位显示：

| 信息 | 目的 |
|---|---|
| 当前器官名称与品质 | 快速判断构筑 |
| 系统角色标记 | 看出它服务哪些闭环 |
| 元素反应标记 | 看出它以什么元素动作服务哪个反应阶段、与谁可连续或竞争 |
| 血脉标记 | 看出来源、生态兼容和维护条件 |
| 污染/完整度 | 看出维护需求 |
| 可形成系统提示 | 降低组合门槛 |

不在 UI 中把复杂公式直接丢给玩家；详细数值可在展开说明中查看。

## 19.5 魔兽与生态图鉴

图鉴应记录已发现的：


## 19.6 诊断面板与日志

开发期提供受控诊断视图，显示任意玩家或魔兽最近一次器官元素贡献、反应链、系统、统合、身体能力快照、术式编译报告、内容版本、失效条件和低频状态转换。正式玩家 UI 不显示内部 ID、worker 细节或日志。

# 二十二、平衡规则与风险控制

## 22.1 三重限制

所有强能力至少受以下三类限制中的两类约束：

| 类型 | 例子 |
|---|---|
| 资源 | 法力、通量、以太、消耗品 |
| 时间 | 冷却、蓄力、恢复期、仪式时长 |
| 风险 | 稳定、畸变、精神负荷、系统压力、生态条件 |

## 22.2 不让当前主回路包打天下

| 规则 | 目的 |
|---|---|
| 一个当前主回路占用大部分通量 | 防止同时满功率运行所有系统 |
| 协同回路只提供声明过的供能、控制、承载、环境或稳定支持 | 防止协同回路伪装成第二个无代价主输出 |
| 当前主回路可调度但不能无成本瞬切 | 保留行为切换的战术价值与身体负担 |
| 道路核心只在术式图实际接入时偏转已成立反应链中的动作与产物 | 防止道路凭空补齐元素、器官或系统能力，也防止未使用道路图的混搭术式被暗中削弱 |
| 道路专精图、通用图和混搭联结图共享同一效果预算 | 防止道路变成必选职业倍率；差异只能来自语义、稳定性、节点效率、覆盖面与风险分布 |
| 每个高收益反应都要声明副产物与处理链 | 防止“火、水、光”等元素只成为无代价伤害或治疗标签 |
| 全身统合与完成体需要生态或仪式条件 | 让世界探索与基地建设有意义 |
| 高品质器官带维护和稀缺条件 | 防止纯品质堆叠 |
| 混搭联结需要额外桥接、协同回路或稳定器 | 让复合产物有明确代价，但不把“不玩道路”视为惩罚 |
| 工业不能直接提供身体法力 | 保留升华与工业的边界 |
| 玩家与魔兽共用身体 evaluator 与术式 compiler | 防止玩家规则和怪物术式逐步背离 |

## 22.3 初期强度定位

~~~text
凡人：能准备材料和使用外源设备，但没有个人法力。
启灵：略强于早期工具与装备，具有清晰特长与明显限制。
凝辉：形成一条当前主回路与至少一条协同回路，可应对中期群系与精英。
以太武装：强但需要工业和维护，不替代身体构筑。
心光及以上：后期遗迹、Boss 和领域内容。
~~~

## 22.4 复杂度控制

| 风险 | 控制方式 |
|---|---|
| 玩家看不懂组合 | 系统只由内容定义配方形成，术式编辑器显示可用身体指令、端口、缺失角色与失败原因 |
| 器官、道路、元素混淆 | 固定使用“元素做什么动作、器官在哪个阶段承担、系统如何闭环、术式如何连接、道路或混搭如何偏转产物”的术语 |
| 组合太多难平衡 | 限制同时高功率调度的回路数量，用效果预算、类型端口、混搭联结与内容定义约束自由图式 |
| 工业吞掉成长 | 保留植入、道路、晋升、仪式和遗迹的人为确认 |
| 魔兽变成刷材料 | 提供取样、驯养、共生和生态恢复收益 |
| 魔兽与玩家规则漂移 | 用同一 SourceLawBodyState、系统定义和统合 evaluator |
| 风险全靠随机 | 所有排异、冲突和压力都提供可预览原因与处理手段 |

## 22.5 验收问题

每新增一个器官、系统、道路、身体指令或术式图时，必须回答：

1. 它服务哪个槽位、哪些系统角色？
2. 它以哪个元素执行什么动作，处在产生/摄入、传导、转化、作用/释放还是校准/回收阶段？
3. 它为何来自这个魔兽或生态？
4. 它和现有内容形成了什么新反应链与新闭环，而不是数值更大？
5. 反应产物、副产物、风险、维护或反制分别是什么？
6. 它给战斗、探索、工业中至少哪两项带来什么差异？
7. UI 如何解释它的形成、失效和升级条件？
8. 它在玩家与对应魔兽身体上是否使用同一条反应、系统与统合规则？
9. 它开放或使用哪些身体指令、端口、当前主回路、协同回路与副产物服务？
10. 它是道路语义、通用术式还是混搭联结？是否和其他合法构筑使用同一效果预算？
11. 是否需要新增日志、内容验证或测试用例？

# 二十三、最终基座

## 23.1 体系总纲

~~~text
源质材料是世界的超凡物质，可以采集、加工、存储和交易。
源质储备是身体内化的长期储量，不能被机器抽取。
源质通量是器官网络安全传导力量的能力。
法力是通量转化出的即时施法缓冲。
元素是源质能在身体中执行的生理动作，不是伤害颜色。
血脉和生态说明源质器官为何存在、如何相容、如何维护。
器官是身体中执行元素动作的承载节点，每项贡献都要声明反应阶段与代价。
八个源质生理系统是器官把产生、传导、转化、作用、释放、校准与回收连成的功能闭环。
闭环开放身体指令与反应产物，不直接发放固定技能。
八个器官全部源质化并通过统合桥接联合后，身体才进入完成体。
个人源律术式是玩家自由连接身体指令、反应产物、控制流和回路服务后编译出的能力；它不需要符文或魔符。
道路提供可复制、可修改的预设术式图和道路核心语义。道路核心只转换同一效果预算的表现、稳定性与风险，不能把道路专精变成通用倍率职业。
通用图和混搭联结图可不使用道路核心，仍使用完整基础效果预算；混搭的价值来自跨系统复合产物与额外桥接要求。
玩家与魔兽共享同一套八槽位、系统、统合和完成体规则；不同之处只在成长入口与控制方式。
稳定、畸变、精神负荷和系统压力是身体网络的状态。
符文是工具侧可封装的源律规律。
魔符与源律武装是有限插槽的工具术式，不是人物个人术式的编译入口。
祭坛是定点的大型源质回路。
以太是源质材料的工业稳定化形态，不可直接被身体吸收。
遗迹是旧文明保存的高阶源律工程。
~~~

## 23.2 最终玩法流程

~~~text
发现群系、魔兽、矿脉与遗迹
  -> 获得源质、组织样本、血脉知识与符石
  -> 通过培育、魔药、调律获得源质器官
  -> 让器官在八槽位中承担元素动作并形成可验证的反应链
  -> 由反应链形成源质生理系统与成长联结
  -> 获得身体指令、反应产物与回路服务
  -> 学习道路预设源图或自由编写通用/混搭个人术式图
  -> 用当前身体编译术式；道路核心和混搭联结只改变同一效果预算的语义与风险分布
  -> 八个器官全部源质化后完成全身统合，进入完成体
  -> 解锁统合身体指令和道路完成体预设图
  -> 用法力直接施放个人术式；另以符文、魔符和武装装配有限工具功能
  -> 用祭坛与以太工业规模化、稳定化或扩展
  -> 处理生态、污染、畸变和遗迹风险
  -> 进入更高能级、更复杂系统与文明级内容
~~~

## 23.3 最终开发原则

