# AE2 独立游戏实现分层图说明文档

![AE2 独立游戏实现分层图](./a_clean_infographic_diagram_image_no_humans_show.png)

## 1. 表现层 (UI & 交互)
- **用户界面 (GDScript)**: 终端 UI、滑杆、物品槽等界面。
- **交互逻辑 (GDScript)**: 拖拽、点击、快捷操作等用户交互处理。
- **游戏场景**: UI 显示的 3D/2D 场景和动态效果。

> 技术栈: GDScript + Godot UI，负责界面呈现和事件处理。

## 2. 脚本层 (绑定层)
- **C++ 和 GDScript 绑定 (GDNative / GDExtension)**:
  - 类注册与绑定
  - 方法映射
  - 信号通信
  - 数据类型转换
  - 内存管理桥接

> 技术栈: GDExtension (C++)，用于核心逻辑与 UI 的交互接口。

## 3. 核心系统层 (AE2 体系)
- **网络系统**: 网络拓扑管理、节点发现、网络合并/拆分、信道管理、网络状态同步。
- **存储系统**: 存储组件、存储单元、存储总线、容量计算、存储优化。
- **物品系统**: 物品注册、元数据管理、分类、序列化、比较。
- **终端系统**: 终端接口、ME 终端、Pattern 终端、液体终端、合成终端。
- **合成系统**: 配方管理、自动合成、任务队列、资源探测、多优先级管理。
- **通道系统**: 通道类型、数量管理、优先级、阻塞、优化。

> 核心数据结构与算法: 图结构、路径查找、缓存系统、事件系统。
> 技术栈: C++17/20，高性能核心逻辑实现。

## 4. 引擎适配接口层 (C++ 接口)
- **世界接口**: 块查询、实体管理、世界查询
- **渲染接口**: 模型渲染、特效系统、材质管理
- **音频接口**: 音效播放、音乐管理、音频空间
- **输入接口**: 输入状态、按键映射、控制器支持
- **网络接口**: 数据同步、多用户通信、延迟补偿

> 技术栈: C++ 抽象接口，负责与 Godot 引擎进行通信。

## 5. 游戏引擎层 (Godot)
- **引擎核心**: 场景树、渲染引擎、物理引擎、音频系统、动画系统、资源管理、多线程、脚本系统

> 技术栈: Godot Engine 4.x

## 6. 平台层 (操作系统)
- Windows
- macOS
- Linux
- Steam Deck

> 技术栈: 原生平台 API，系统调用、硬件访问、文件系统、网络通信
