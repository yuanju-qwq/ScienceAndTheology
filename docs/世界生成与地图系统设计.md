# 世界生成与地图系统设计

## 1. 概述

"科学与神学"的世界采用**双层互通**设计：Surface（地表）和 Underground（地下）通过 Connector 相连。世界数据在 C++ 核心层管理（`WorldData` / `ChunkData` / `TerrainData`），Godot GDScript 层负责渲染和交互。

核心架构原则：**Godot 不是世界本身**，TileMapLayer 只是 `ChunkData` 的可视化代理。

## 2. 图层系统

### 2.1 当前图层（2 层）

| 图层 ID | 名称 | 描述 |
|---------|------|------|
| `surface` | 地表 | 开放世界，有草地、树木、水域、建筑 |
| `underground` | 地下 | 洞穴系统，石壁、矿脉、遗迹 |

### 2.2 坐标系统

统一全局坐标系：`layer_id(StringName) + cell_position(Vector2i)`。

Chunk 坐标 = `{layer_id, chunk_x, chunk_y}`，Chunk 大小 = 32×32 cells。

在 GDScript 中通过 `MapLayerController` 管理图层切换。

### 2.3 多层扩展 [未实现]

> 当前只支持 `surface` 和 `underground`。架构预留了更多层：
> - Ruins（遗迹层）
> - Facility / Factory Depth（工厂深度层）
> - Deep Mine（深层矿井）
> - 建筑内部等
>
> C++ 端 `TerrainGenerator::generate_chunk()` 对未知层有 `else { fill with stone }` 回退。
> GDScript 端 `MapLayerController` 当前只识别 `surface`/`underground`。

## 4. 地形生成流水线（TerrainGenerator）

### 4.1 六阶段生成

`TerrainGenerator` 调用 6 个 Pass，每个 Pass 使用独立种子的 `NoiseGenerator`，天然线程安全。

```
generate_chunk(layer_id, cx, cy)
├── 1. pass_base_terrain()    ✅ 已实现
├── 2. pass_biome()           🔲 Stub
├── 3. pass_ore()             ✅ 已实现
├── 4. pass_structure()       🔲 Stub
├── 5. pass_object()          🔲 Stub
└── 6. pass_gameplay()        ✅ 已实现
```

### 4.2 Pass 1: 基础地形（已实现）

**Surface 层**：
- 使用 elevation noise（`scale=0.02, octaves=4`）区分地表高度
- 顶部行：DIRT（可通行），进一步用 detail noise 添加 SAND 斑块
- 下部行：STONE

**Underground 层**：
- 使用 cave noise（`scale=0.04, octaves=4`）生成洞穴
- 洞穴内：AIR（可通行），边缘区域保持 STONE

### 4.7 Pass 6: Gameplay 元素（已实现）

使用 placement noise（`scale=0.15, octaves=1`）确定 Connector 位置：

| Connector 类型 | 触发层 | 激活方式 | 方向 |
|---|---|---|---|
| Cave Entrance | Surface | 交互（E 键） | 双向 |

## 5. 噪声生成器（NoiseGenerator）

### 5.1 当前实现

自定义 Perlin 分形噪声（~140 行 C++）：
- 256 字节排列表（Fisher-Yates 洗牌、确定性种子）
- Perlin 改善衰减曲线：`6t⁵ − 15t⁴ + 10t³`
- 分形八度叠加（可配置 octaves / persistence）
- 返回值范围 [-1, 1]

### 5.2 保留原因

- 纯 C++ 栈分配，**零堆内存分配**
- **天然线程安全**：每个 `generate_chunk()` 调用在栈上构造独立实例
- 虚函数开销为零（inline 方法）
- 与 Godot `FastNoiseLite`（Object/RefCounted 子类）相比，在 `WorkerThreadPool` 工作线程中使用 Godot Object 存在线程安全风险

> 将来若需要 Cellular / DomainWarp 等高级噪声，可引入独立 C++ 头文件 [FastNoiseLite](https://github.com/Auburn/FastNoiseLite)（非 Godot 绑定版本），直接嵌入核心层，避免 Godot Object 依赖。

## 6. 异步 Chunk 生成

### 6.1 架构

使用 Godot 内置 `WorkerThreadPool::get_singleton()` 单例：

```
GDScript __process__ → GDWorldData.process_async_results()
GDScript → GDWorldData.request_chunk_async()
  └── WorkerThreadPool::add_native_task()
      └── 工作线程 → _async_chunk_callback()
          ├── TerrainGenerator::generate_chunk()  # 栈上 NoiseGenerator
          └── mutex → async_results_.push(result)
```

### 6.2 GDWorldData 异步接口

```cpp
class GDWorldData : public godot::Resource {
    // 非阻塞：提交 chunk 生成任务
    void request_chunk_async(layer_id, chunk_x, chunk_y);

    // 每帧调用：消费已完成的结果，emit chunk_ready 信号
    int64_t process_async_results();

    int64_t get_async_pending_count() const;  // 排队中 + 运行中
    int64_t get_worker_thread_count() const;  // CPU 核心数
    int64_t get_async_completed_count() const;
    int64_t get_max_async_queue_size() const; // 默认 256
};
```

### 6.3 容量控制

`max_async_queue_size`（默认 256）：当 `get_async_pending_count() >= max` 时，`request_chunk_async()` 静默丢弃新请求，防止无界队列增长。

### 6.4 线程安全保证

- `TerrainGenerator::generate_chunk()` 只读 `world_seed_`（const），其余栈上分配
- `NoiseGenerator` 在 `generate_chunk()` 内构造独立 `perm_` 数组
- 结果通过 `std::mutex` 保护的 `std::queue` 回传主线程
- `emit_signal("chunk_ready")` 仅在主线程的 `process_async_results()` 中调用

## 7. GDScript 地图系统（原型层）

### 7.1 MapLayerController（图层管理）

```gdscript
class MapLayerController extends Node2D:
    signal layer_changed(old_layer, new_layer)
    var current_layer: StringName  # "surface" / "underground"

    func change_layer(target_layer)  # 切换图层
    func toggle_layer()              # Tab 键切换
    func set_layer_view_context(layer_alphas, status_suffix)  # 双层视角
```

两个 `TileMapLayer` 子节点（`SurfaceLayer` / `UndergroundLayer`），切换时设置：
- `visible`：当前层 true，另一层由 `LayerViewContext` 控制
- **碰撞/导航**：非活动层禁用
- **process_mode**：非活动层 DISABLED

### 7.2 Connector 系统

```gdscript
class MapConnector extends Resource:
    var connector_id: int
    var from_layer, to_layer: StringName
    var from_cell, to_cell: Vector2i
    var one_way: bool         # 单向通行
    var locked: bool          # 锁定（需机关解锁）
    var connector_type: StringName   # "cave_entrance" / "rift" / "ruin_gate"
    var activation_mode: int  # INTERACT / AUTO_ON_ENTER
```

**ConnectorManager**：管理所有 `MapConnector` 资源，提供空间查询：
```gdscript
func get_connector_at(layer_id, cell_position) → MapConnector
func get_target_for(layer_id, cell_position) → {connector, layer, cell}
```

### 7.3 Mechanism 系统

```gdscript
class MapMechanism extends Resource:
    var mechanism_id: StringName
    var layer_id, cell_position
    var flag_name: StringName    # 关联世界旗标
    var activation_mode: int     # INTERACT / AUTO_ON_ENTER
    var one_shot: bool           # 一次性触发
    var effects: Array[Dictionary]  # {type, path, connector_id, ...}
```

**MechanismManager**：世界旗标系统 + 机关效果：

支持的效果类型：
| 效果 | 说明 |
|------|------|
| `node_visible` | 控制节点显示/隐藏（如封印门→打开门） |
| `connector_locked` | 锁定/解锁 Connector（如祭坛解锁遗迹大门） |

原型机关：
- `surface_sun_altar_001`：交互后打开 Underground 遗迹大门（`underground_gate_open` 旗标）
- `underground_pump_001`：交互后升起 Surface 断桥（`surface_bridge_raised` 旗标）

### 7.4 PlayerController（角色控制）

```gdscript
class PlayerController extends CharacterBody2D:
    signal connector_used(connector_id, from_layer, to_layer)
    signal mechanism_activated(mechanism_id, layer_id)
    var move_speed: float = 96.0

    # 移动：WASD / 方向键
    # 交互：E 键 → _try_use_connector / _try_activate_mechanism
    # 自动触发：移动进入 Connector/Mechanism 所在格
```

图层过渡流程：
1. `_play_transition_fade_in()` — 淡入遮罩
2. `layer_controller.change_layer(target_layer)` — 切换图层
3. `global_position = target_tile_layer.map_to_local(target_cell)` — 传送
4. `_play_transition_fade_out()` — 淡出遮罩

### 8.1 LayerViewRegion

```gdscript
    var region_id: StringName

    var center_cell, radius: int

    var revealed_layer: StringName       # 要透视的图层

    var secondary_alpha: float = 0.42    # 透视层透明度
```

### 8.2 LayerViewController

每个 `_process` tick 检测玩家是否进入视域区域，调用 `MapLayerController.set_layer_view_context()` 设置图层透明度：

**LOCAL_PEEK**：在 Connector 附近部分透视另一层。
- 玩家在地表 Cave Entrance 附近（2 格半径）→ Underground 层半透明显示
- 玩家在地下 Rift 附近（2 格半径）→ Surface 层半透明显示

**DUAL_LAYER**：双图层同屏。
- 原型 Rift 区域（`(-3, 1)`，2 格半径）→ Surface 和 Underground 同时可见（88%/52% 透明度）

线程安全：无状态、可重入。

## 18. GDScript 用法示例

### 异步 Chunk 加载

```gdscript
var world = GDWorldData.new()
world.seed = randi()
world.max_async_queue_size = 256

func _process(delta):
    # 消费已完成的异步生成结果（emit chunk_ready 信号）
    var completed = world.process_async_results()

# 请求异步生成
world.request_chunk_async("surface", 0, 0)
world.request_chunk_async("underground", 0, 0)

# 同步生成（立即返回）
var terrain = world.get_or_generate_chunk("surface", 0, 0)
# terrain = {size_x: 32, size_y: 32, materials: PackedByteArray, flags: PackedInt32Array}
```
