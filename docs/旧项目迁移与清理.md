# 旧项目迁移与清理

> 当前清理基线，更新于 2026-07-16。本文只记录旧 Godot/GDExtension 树的保留理由、迁移范围和删除门槛；当前运行时能力以 [项目架构与运行时基线.md](项目架构与运行时.md) 为准，玩法与联机状态分别以 [P7 玩法迁移设计](p7_玩法迁移设计.md) 和 [游戏网络协议设计](游戏网络协议设计.md) 为准。

## 1. 范围和原则

顶层 CMake 只构建 `snt_engine/` 与 `game/`。旧 `src/`、GDScript、Godot 场景和 GDExtension 不参与当前构建或运行包，只能作为迁移来源。

- 行数只用于表示遗留规模，不代表有效代码量、复杂度或工期。
- `src/bindings/` 是 Godot 包装层，不是迁移目标；底层功能替代后直接删除。
- 旧 `tests/core/` 不在当前 CTest 链。删除实现前，将仍有价值的断言迁入 `snt_engine/tests/` 或 `game/tests/`。
- 项目尚未发布，不保留旧 API、旧存档或 Godot 兼容层。每个旧功能必须被标记为 `保留`、`替换` 或 `废弃` 后才可删除对应来源。

## 2. 遗留清单

| 范围 | 规模 | 当前状态 | 处理方式 |
| --- | ---: | --- | --- |
| `src/core/` | 279 文件，42,413 行 | 部分数据与运行时能力已进入新主链，大量玩法域未迁入 | 逐域替换或废弃 |
| `src/bindings/` | 122 文件，19,553 行 | 旧 Godot GDExtension 包装层 | 不迁移包装代码 |
| `scripts/` 和示例 mod | 124 个 `.gd`，31,616 行 | 目前只有脚本框架、炉子内容和基础 UI 原型 | 按玩法/UI 域重建 |
| Godot 入口 | 配置、场景、shader、locale | 只保留为迁移参考 | 替代方案验收后统一删除 |

旧 `src/server/` 的 15 个 `SNT1` socket/protocol 文件已删除：TCP/UDP transport、login、command queue 和 opaque delta producer 分别由 `snt_network`、`snt_game_network`、`GameServerCommandSink` 和 `SNTG` v5 的 server-originated provider boundary 替代；client-trusted `POS_UPDATE`、LAN discovery 和可选密码不进入新版范围。旧 `src/bindings/multiplayer/` 与 `tests/core/m3_server_network.cpp` / `m5_multi_planet.cpp` 仍是未构建的 Godot 遗留引用，随其所属树的后续删除门槛处理。

`src/` 现存受 Git 跟踪的文件共有 412 个，约 2.57 MiB；`scripts/` 共有 252 个文件，约 1.30 MiB，其中包括 125 个会随脚本删除的 `.uid` 文件。

## 3. 已确认的新归属

| 旧模块 | 当前归属 |
| --- | --- |
| `src/core/world_gen/` | `game/worldgen/`（`snt_game_worldgen`） |
| `src/core/save/` | `game/world/save/`（`snt_game_world_save`） |
| `src/core/mobile_structure/` | `game/world/mobile/`（`snt_game_world_mobile`） |
| `WorldData`、通用 terrain、raw region framing | `snt_voxel_data` / `snt_voxel_storage` 加 game-owned sidecar |
| greedy meshing、体素碰撞、射线、运行时调度、渲染 | 当前自研引擎链 |
| 脚本内容、炉子机器、任务运行态、任务存档与 chunk-anchored 机器存档生命周期 | `game/` 的 P7 主线 |

这些归属不代表旧玩法等价迁移完成。炉子、pit kiln、charcoal pit、bloomery 和 anvil 的配方已迁入当前 P7 内容包；多输入槽、手动激活、cover/ignition/structure/tool tag 激活契约、bloomery 3x3x2/17-cell 体素结构校验和 v12 chunk-sidecar 机器存档也已具备。`MachineInteractionService` 只接受主线程已验证的值 context，结构校验在缺失 chunk 时 fail-closed；尚未接入认证玩家、库存扣除、方块编辑或收集。完整工业/魔法、自动化、宇宙和旧 UI 仍须按新版边界重建或明确废弃。

## 5. 删除门槛

| 对象 | 删除前必须满足 |
| --- | --- |
| `src/` | 对应功能已替换或正式废弃，关键行为测试已迁入当前 CTest；`src/server/` 已按此门槛删除 |
| `scripts/` | 所有保留玩法已进入 `game/`；关联 `.uid` 一并删除 |
| Godot 入口、场景、shader、locale、示例 mod | 新启动、打包、UI、渲染、国际化和 mod 策略均已验收或废弃 |
| `tests/core/` | 有价值的断言已迁入当前测试目标 |
| `resource/terrain/` | 不删除；当前 `game/CMakeLists.txt` 仍复制它 |
| 其他 `resource/` | 先标记为运行包资产、源素材或废弃物，不能按目录批量删除 |
| 未跟踪 `bin/`、`.godot/` | 停止使用旧 Godot 工作流且当前运行包验证完成后再清理 |

最终删除前必须在干净构建目录完成配置、客户端/服务端构建和当前 CTest，并确认构建、运行包、测试与工具不再引用 `src/`、`scripts/`、`res://`、`godot-cpp`、`.gdextension` 或 Godot 场景路径。

## 6. 取舍和下一步

| 路线 | 优点 | 缺点 |
| --- | --- | --- |
| 全功能等价迁移 | 最大限度保留旧玩法和内容资产 | 需要重建大量 Godot UI、GDExtension 边界和旧测试 |
| 按新版范围裁剪 | 最快收敛到无 Godot 的主链，符合不维护兼容的策略 | 需要明确记录被放弃的玩法 |

当前采用“按新版范围裁剪”：先为每个功能域记录唯一新归属和 `保留`、`替换` 或 `废弃` 决策，再按优先级迁移。P7.4 的机器运行态已以 chunk-sidecar anchor、稳定 Guid、v12 多输入/激活状态恢复与未锚定拒绝保存完成，P7.2.4 已补充 cover/ignition/structure/tool tag 的 activation contract 与 bloomery 体素结构校验；玩家任务进度已以 revision-gated autosave 避免无变更定时重写。P7.5 已把 `SNTG` v5 承载的 snapshot/delta 值 codec、per-peer budget、provider 调度、客户端 drain、同 dimension 玩家 AOI、presentation-only player blob、per-observer delta baseline 与客户端 `GameRemotePlayerWorld` 建立在无头网络边界内；服务端已完成 movement/physics，客户端只提交 intent 并消费权威位置。新版玩家产品规则现由[玩家状态与交互产品设计](玩家状态与交互产品设计.md)冻结：`SNTP` 状态、断线保存/实体销毁、重连恢复和 dirty-gated checkpoint 已完成；床、死亡、不可破坏 sidecar 墓碑与重生 resolver 已作为服务端后端接入。方块编辑入口、战斗死亡 producer、墓碑 UI、机器 collect/activation、任务书、Steam 成就 adapter、chunk/machine source 与 remote avatar rendering 仍待实现；所有真实交互不得信任 client context。
