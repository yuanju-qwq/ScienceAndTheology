# 旧项目迁移与清理

> 当前清理基线，更新于 2026-07-15。本文只记录旧 Godot/GDExtension 树的保留理由、迁移范围和删除门槛；当前运行时能力以 [项目架构与运行时基线.md](项目架构与运行时.md) 为准，玩法与联机状态分别以 [P7 玩法迁移设计](p7_玩法迁移设计.md) 和 [游戏网络协议设计](游戏网络协议设计.md) 为准。

## 1. 范围和原则

顶层 CMake 只构建 `snt_engine/` 与 `game/`。旧 `src/`、GDScript、Godot 场景和 GDExtension 不参与当前构建或运行包，只能作为迁移来源。

- 行数只用于表示遗留规模，不代表有效代码量、复杂度或工期。
- `src/bindings/` 是 Godot 包装层，不是迁移目标；底层功能替代后直接删除。
- 旧 `tests/core/` 不在当前 CTest 链。删除实现前，将仍有价值的断言迁入 `snt_engine/tests/` 或 `game/tests/`。
- 项目尚未发布，不保留旧 API、旧存档或 Godot 兼容层。每个旧功能必须被标记为 `保留`、`替换` 或 `废弃` 后才可删除对应来源。

## 2. 遗留清单

| 范围 | 规模 | 当前状态 | 处理方式 |
| --- | ---: | --- | --- |
| `src/core/` | 279 文件，42,413 行 | 部分数据与运行时能力已进入新主链，大量玩法域未迁入 | 逐域替换或废弃 |
| `src/bindings/` | 122 文件，19,553 行 | 旧 Godot GDExtension 包装层 | 不迁移包装代码 |
| `src/server/` | 15 文件，1,941 行 | 直连 transport、登录、首条权威任务命令和任务生命周期已有新路径；AOI、snapshot/delta 与完整玩法联机仍缺 | 完成保留的服务端玩法后删除 |
| `scripts/` 和示例 mod | 124 个 `.gd`，31,616 行 | 目前只有脚本框架、炉子内容和基础 UI 原型 | 按玩法/UI 域重建 |
| Godot 入口 | 配置、场景、shader、locale | 只保留为迁移参考 | 替代方案验收后统一删除 |

`src/` 受 Git 跟踪的文件共有 428 个，约 2.64 MiB；`scripts/` 共有 252 个文件，约 1.30 MiB，其中包括 125 个会随脚本删除的 `.uid` 文件。

## 3. 已确认的新归属

| 旧模块 | 当前归属 |
| --- | --- |
| `src/core/world_gen/` | `game/worldgen/`（`snt_game_worldgen`） |
| `src/core/save/` | `game/world/save/`（`snt_game_world_save`） |
| `src/core/mobile_structure/` | `game/world/mobile/`（`snt_game_world_mobile`） |
| `WorldData`、通用 terrain、raw region framing | `snt_voxel_data` / `snt_voxel_storage` 加 game-owned sidecar |
| greedy meshing、体素碰撞、射线、运行时调度、渲染 | 当前自研引擎链 |
| 脚本内容、炉子机器、任务运行态与任务存档生命周期 | `game/` 的 P7 主线 |

这些归属不代表旧玩法等价迁移完成。尤其 bloomery、pit kiln、charcoal pit、anvil、完整工业/魔法、自动化、宇宙和旧 UI 仍须按新版边界重建或明确废弃。

## 4. 需要产品决策的功能域

| 旧域 | 主要内容 | 当前动作 |
| --- | --- | --- |
| `simulation` | 生态、季节、作物、区块物理、tick、状态同步 | 按新 ECS/scheduler 重新划分所有权 |
| `universe` | Sector、跨区加载、星体 LOD、重力、跨区实体、可靠性 | 以 [统一宇宙 World 架构设计](unified_universe_world_design.md) 为目标，尚未形成等价新路径 |
| `ae2`、`sfm`、`network` | 自动化、物流、电力、流体、信号 | 先决定是否进入新版范围，再设计数据边界 |
| `source_law`、`magic` | 源律、器官、药剂、符文、仪式 | 以 [源律升华体系融合设计](源律升华体系融合设计.md) 为产品规格，尚未完成迁移 |
| `multiblock`、`player`、旧 UI | 结构、玩家状态、装备、交互界面 | 逐项指定替换目标或废弃 |

## 5. 删除门槛

| 对象 | 删除前必须满足 |
| --- | --- |
| `src/` | 对应功能已替换或正式废弃，关键行为测试已迁入当前 CTest |
| `scripts/` | 所有保留玩法已进入 `game/`；关联 `.uid` 一并删除 |
| Godot 入口、场景、shader、locale、示例 mod | 新启动、打包、UI、渲染、国际化和 mod 策略均已验收或废弃 |
| `tests/core/` | 有价值的断言已迁入当前测试目标 |
| `resource/terrain/` | 不删除；当前 `game/CMakeLists.txt` 仍复制它 |
| 其他 `resource/` | 先标记为运行包资产、源素材或废弃物，不能按目录批量删除 |
| 未跟踪 `bin/`、`.godot/` | 停止使用旧 Godot 工作流且当前运行包验证完成后再清理 |

最终删除前必须在干净构建目录完成配置、客户端/服务端构建和当前 CTest，并确认构建、运行包、测试与工具不再引用 `src/`、`scripts/`、`res://`、`godot-cpp`、`.gdextension` 或 Godot 场景路径。

## 6. 取舍和下一步

| 路线 | 优点 | 缺点 |
| --- | --- | --- |
| 全功能等价迁移 | 最大限度保留旧玩法和内容资产 | 需要重建大量 Godot UI、GDExtension 边界和旧测试 |
| 按新版范围裁剪 | 最快收敛到无 Godot 的主链，符合不维护兼容的策略 | 需要明确记录被放弃的玩法 |

当前采用“按新版范围裁剪”：先为每个功能域记录唯一新归属和 `保留`、`替换` 或 `废弃` 决策，再按优先级迁移。近期优先完成机器运行态持久化、真实任务事件/reward、Steam adapter、AOI、snapshot/delta，再安排工业、源律、魔法、自动化和宇宙系统。
