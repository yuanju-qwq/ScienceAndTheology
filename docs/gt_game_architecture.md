# GregTech 独立游戏实现分层图说明文档

## 1. 表现层 (UI & 交互)
- **用户界面 (GDScript)**: 机器面板、滑杆、物品槽、能源统计面板。
- **交互逻辑 (GDScript)**: 拖拽、点击、快捷操作、机器状态切换。
- **游戏场景**: 工业多块结构显示、管道流动效果、粒子动画。

> 技术栈: GDScript + Godot UI，负责界面呈现和事件处理。

## 2. 脚本层 (绑定层)
- **C++ 和 GDScript 绑定 (GDExtension)**:
  - 类注册与方法绑定
  - 信号通信
  - 数据类型转换
  - 内存管理桥接

> 技术栈: GDExtension (C++)，用于核心逻辑与 UI 的交互接口。

## 3. 核心系统层 (GregTech 工业逻辑)
- **机器系统**: 单机机器、多块结构、效率计算、状态管理。
- **能量系统**: 电压类型、电流、能量分配、损耗计算。
- **物品/流体系统**: 管道流、存储单元、运输优化、优先级调度。
- **配方系统**: 高复杂配方、自动加工、批量处理、多优先级任务队列。
- **任务/事件系统**: 事件触发、进程调度、异常处理、状态同步。

> 核心数据结构与算法: 图结构、路径查找、缓存系统、事件系统。
> 技术栈: C++17/20，高性能核心逻辑实现。

## 4. 引擎适配接口层 (C++ 接口)
- **世界接口**: 块查询、实体管理、环境数据
- **渲染接口**: 模型渲染、特效系统、材质管理
- **音频接口**: 音效播放、音乐管理、音频空间
- **输入接口**: 按键、控制器、触摸输入
- **网络接口**: 多用户通信、数据同步、延迟补偿

> 技术栈: C++ 抽象接口，负责与 Godot 引擎通信。

## 5. 游戏引擎层 (Godot)
- **引擎核心**: 场景树、渲染引擎、物理引擎、音频系统、动画系统、资源管理、多线程、脚本系统

> 技术栈: Godot Engine 4.x

## 6. 平台层 (操作系统)
- Windows
- macOS
- Linux
- Steam Deck

> 技术栈: 原生平台 API，系统调用、硬件访问、文件系统、网络通信


### 建议的实现顺序

1. C++ 能量系统 （最小可行）— GTValues 的电压/电流常量 + 能量节点图
2. 物品/材料系统 — Material × Form 的数据模型
3. 配方系统 — RecipeMap + RecipeBuilder
4. ProcessingLogic — 配方匹配 + 超频计算
5. 机器基类 — 单方块机器生命周期
6. GDExtension 绑定 — 暴露核心 API 给 GDScript
7. Godot UI — 机器面板、能量仪表盘
