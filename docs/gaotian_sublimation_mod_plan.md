```text

Above the High Sky: Sublimation Age
泰拉升华
以太与源质
SublimationCraft

```

如果是 Minecraft Mod，中文名可以叫：

```text

```

英文内部 ID：

```text

```

## 1.2 公开发布时的命名建议

如果只是私人整合包，可以保留比较明显的原作指向。


# 二、设计目标

本 Mod 的目标不是简单添加几个魔法技能，而是把《高天之上》里的核心体系转化成一套可以长期游玩的游戏系统。

核心目标如下：

1. 玩家可以从普通人逐步成长为升华者。
2. 升华体系要有明确路线、能级、器官、素材、风险。
3. 世界中存在源质生态、魔兽、遗迹、旧文明科技。
4. 战斗、探索、科技、炼金、装备都围绕“源质 / 以太 / 灵能”展开。
5. 系统要能和科技 Mod 兼容，尤其适合接入 GT / AE2 风格科技包。
6. 初版不要做得过重，要先保证可玩，再逐步扩展。

# 三、核心玩法概述

## 3.1 玩家主循环

```text
探索世界
  ↓
发现源质生态 / 魔兽 / 遗迹
  ↓
采集素材、击败魔兽
  ↓
分析素材属性
  ↓
提取源质
  ↓
炼制魔药
  ↓
构建或强化升华器官
  ↓
提升能级
  ↓
解锁新区域、新装备、新机器
  ↓
挑战更高层级敌人和遗迹
```

## 3.2 最核心闭环

第一版最重要的是先做出这个闭环：

```text
魔兽素材 → 源质 → 魔药 → 升华 → 技能 → 更强魔兽
```

中后期再接入：

```text
遗迹 → 蓝图 → 以太机器 → 以太武装 → 高能级晋升
```

# 四、核心设定转化

## 4.1 三大基础资源

| 资源 | 定位 | 游戏用途 |
|---|---|---|
| 源质 | 生命侧超凡能量 | 魔药、升华器官、能力释放 |
| 以太 | 工业侧高阶能量 | 机器、武装、遗迹科技 |
| 灵能 | 信息侧精神力量 | 预知、解析、探测、蓝图研究 |

## 4.2 三者关系

```text
源质：来自生物、魔兽、生态、器官
以太：来自遗迹、机器、以太转化、旧文明设备
灵能：来自玩家精神、特殊天赋、遗迹共鸣、预知碎片
```

可以设计成：

```text
源质 → 以太：可以转化，效率中等
以太 → 源质：可以转化，但风险高
灵能 → 信息：用于解析、预知、研究
```

示例数值：

```text
1000 源质 → 600 以太
1000 以太 → 300 源质 + 畸变风险
100 灵能负荷 → 解析一个遗迹蓝图
```

# 五、升华者系统

## 5.1 能级设计

玩家初始为普通人，通过魔药和器官成长进入升华者体系。

| 能级 | 名称 | 游戏定位 |
|---|---|---|
| 0 | 凡人 | 普通玩家，无超凡能力 |
| 1 | 第一能级：启灵 | 解锁基础源质能力 |
| 2 | 第二能级：凝辉 | 器官系统成熟，能力强化 |
| 3 | 第三能级：心光 | 获得专属核心能力 |
| 4 | 第四能级：领域 | 影响周围环境 |
| 5 | 第五能级：文明级 / 极限级 | 后期 Boss / 终局内容 |

初版建议只做到：

```text
凡人 → 第一能级 → 第二能级
```

第三能级及以上作为中后期版本内容。

## 5.2 玩家升华数据

玩家需要一个独立数据面板。

```text
能级
主升华路线
源质容量
源质恢复速度
源质纯度
器官稳定度
精神负荷
元素亲和
畸变程度
灵能等级
```

示例数据结构：

```java
public class PlayerSublimationData {
    private int level;                 // 0-5
    private String pathId;             // sand_armor, tide, storm...
    private int sourceCurrent;
    private int sourceMax;
    private float sourceRegen;
    private float stability;
    private float mutation;
    private int mentalLoad;
    private Map<String, Integer> affinities;
    private List<OrganData> organs;
}
```

# 六、升华器官系统

## 6.1 器官槽位

玩家不是简单穿装备，而是在体内构建升华器官。

| 槽位 | 作用 |
|---|---|
| 心脏 | 决定源质容量和核心能力 |
| 骨骼 | 决定防御、力量、稳定度 |
| 血液 | 决定恢复、抗性、源质流动 |
| 肺部 | 决定环境适应和移动能力 |
| 眼睛 | 决定探测、预知、视觉能力 |
| 神经 | 决定反应、速度、技能冷却 |
| 皮肤 | 决定护甲、抗性、外部防护 |

初版不要全部开放，可以先做：

```text
心脏
骨骼
皮肤
眼睛
神经
```

## 6.2 器官品质

```text
劣质
普通
优良
纯净
完美
古代
```

数据结构：

```java
public class OrganData {
    private String organId;
    private String slot;
    private String element;
    private int quality;
    private int growth;
    private int stability;
    private int mutationRisk;
}
```

## 6.3 器官成长

器官可以通过战斗、源质灌注、特殊素材成长。

```text
器官等级 1：基础效果
器官等级 2：数值增强
器官等级 3：解锁被动
器官等级 4：解锁主动技能
器官等级 5：升阶材料需求
```

例如沙铠岩核：

```text
等级 1：+2 护甲
等级 2：+4 护甲
等级 3：受到伤害时有概率生成岩盾
等级 4：解锁主动技能“晶岩护壁”
等级 5：可进化为“地脉岩核”
```

## 6.4 器官兼容性

同路线器官兼容度高，不同路线混搭有风险。

```text
沙铠心脏 + 沙铠骨骼：稳定度 +20%
沙铠心脏 + 风暴神经：稳定度 -10%，但获得特殊冲撞效果
炎炉心脏 + 潮汐肺：稳定度 -30%，高冲突
辉光眼 + 任意道路：降低畸变风险
```

这个系统可以鼓励玩家构筑 BD，而不是无脑堆最强器官。

# 七、魔药系统

## 7.1 魔药定位

魔药不是普通药水，而是升华关键道具。

| 类型 | 作用 |
|---|---|
| 启灵魔药 | 进入第一能级 |
| 强化魔药 | 强化器官 |
| 晋升魔药 | 突破到下一能级 |

## 7.2 魔药制作流程

```text
采集魔兽素材
  ↓
源质萃取
  ↓
属性分离
  ↓
加入稳定剂
  ↓
魔药反应釜合成
  ↓
检测兼容率
  ↓
服用 / 注射 / 灌注
```

机器流程：

```text
魔兽素材 → 源质萃取器 → 粗源质
粗源质 → 源质离心机 → 属性源质
属性源质 + 稳定剂 + 器官样本 → 魔药反应釜 → 魔药
```

## 7.3 魔药配方示例

### 沙铠启灵魔药

```text
岩蜥鳞片 x4
晶化骨粉 x2
土属性源质 x1000
稳定剂 x1
普通药剂瓶 x1
```

效果：

```text
解锁沙铠道路
源质容量 +100
获得技能：岩盾
获得器官：沙铠岩核
```

风险：

```text
兼容率低于 60%：获得“石化迟缓”负面效果
兼容率低于 40%：源质暴走
```

### 风暴启灵魔药

```text
雷鸟羽毛 x4
磁晶碎片 x2
雷属性源质 x1000
稳定剂 x1
普通药剂瓶 x1
```

效果：

```text
解锁风暴道路
源质容量 +80
移动速度 +5%
获得技能：雷光步
```

风险：

```text
低稳定度时随机触电
高精神负荷时技能失控
```

### 潮汐启灵魔药

```text
海蛇鳞片 x4
潮汐腺体 x1
水属性源质 x1000
稳定剂 x1
普通药剂瓶 x1
```

效果：

```text
水下呼吸
游泳速度提升
获得技能：潮汐恢复
```

风险：

```text
离水太久时源质恢复降低
```

# 八、源质系统

## 8.1 源质属性

建议先设计 8 种源质属性：

| 属性 | 方向 |
|---|---|
| 土 | 防御、稳定、护盾 |
| 水 | 恢复、流动、适应 |
| 火 | 爆发、燃烧、熔炼 |
| 风 | 速度、闪避、位移 |
| 雷 | 爆发、连锁、充能 |
| 光 | 净化、治疗、探测 |
| 暗 | 潜行、诅咒、侵蚀 |
| 生命 | 再生、成长、器官培育 |

初版建议只开放：

```text
土
水
火
雷
光
生命
```

## 8.2 源质获取

| 来源 | 产物 |
|---|---|
| 魔兽掉落物 | 粗源质 |
| 源质矿石 | 低纯度源质 |
| 特殊植物 | 生命源质 |
| 遗迹设备 | 以太化源质 |
| Boss 核心 | 高纯度源质 |
| 玩家器官成长 | 少量自循环源质 |

## 8.3 源质纯度

```text
污浊源质：便宜，但畸变风险高
粗源质：普通素材
纯净源质：高级魔药材料
活化源质：器官培育材料
古代源质：高级道路突破材料
```

## 8.4 源质污染

污染来源：

```text
服用低品质魔药
植入冲突器官
长时间待在污染群系
使用破损以太装备
被畸变魔兽攻击
强行突破能级
```

污染后果：

```text
源质恢复下降
随机负面效果
器官稳定度降低
精神负荷增加
最终畸变成怪物
```

# 九、畸变系统

## 9.1 畸变值

```text
0-20：正常
21-40：轻微污染
41-60：出现畸变症状
61-80：严重畸变
81-100：源质失控
```

## 9.2 畸变症状

| 畸变值 | 效果 |
|---|---|
| 20+ | 偶尔视野扭曲 |
| 40+ | 源质恢复不稳定 |
| 60+ | 随机触发器官副作用 |
| 80+ | 受到伤害时可能源质暴走 |
| 100 | 死亡并生成畸变体，或者进入特殊濒死状态 |

## 9.3 畸变体机制

如果玩家畸变死亡，可以在死亡点生成一个“玩家畸变体”。

```text
玩家畸变体继承部分玩家能力
击败后可回收部分器官素材
高难度整合包中可作为惩罚机制
```

这个机制很有特色，但初版可以先不做。

# 十、灵能系统

## 10.1 灵能定位

灵能不是普通魔法，而是信息、预知、解析能力。

灵能主要服务于：

```text
危险提示
矿物探测
遗迹解析
蓝图研究
战斗预判
科技解锁
```

## 10.2 灵能等级

| 等级 | 名称 | 能力 |
|---|---|---|
| 0 | 无灵能 | 无特殊效果 |
| 1 | 微弱感知 | 危险提示 |
| 2 | 清晰感知 | 矿物 / 敌人探测 |
| 3 | 预判 | 显示攻击轨迹 |
| 4 | 解析 | 破解遗迹机关 |
| 5 | 命运分支 | 高级预知和蓝图获取 |

初版建议只做：

```text
危险提示
矿物探测
遗迹解析
```

## 10.3 灵能负荷

灵能不能无限使用。

```text
使用灵能 → 增加精神负荷
精神负荷过高 → 视野模糊、错误提示、吸引怪物
休息、药剂、辉光道路能力 → 降低精神负荷
```

## 10.4 灵能技能示例

### 危险预感

效果：

```text
附近有高等级敌人时，屏幕边缘出现红色提示。
```

实现：

```text
周期性扫描玩家周围实体
如果敌人等级高于玩家能级
触发 HUD 提示
```

### 源质视野

效果：

```text
短时间显示附近源质矿、魔兽、遗迹节点。
```

实现：

```text
给目标方块或实体添加轮廓
持续 5-10 秒
消耗灵能负荷
```

### 遗迹解析

效果：

```text
玩家可以读取遗迹方块的数据，解锁蓝图。
```

实现：

```text
右键遗迹核心
检测灵能等级
消耗精神负荷
概率获得蓝图碎片
```

# 十一、以太科技系统

## 11.1 以太定位

以太是机器和武装使用的能源，可以作为科技线的核心。

不要让它直接替代 RF / EU，而是作为特殊能源。

```text
RF / EU：普通工业能源
以太：高阶超凡工业能源
源质：生命超凡能源
```

如果接入 GT，可以设计为：

```text
普通机器用 EU
源质机器消耗源质
以太机器同时消耗 EU + 以太
```

# 十二、以太武装系统

## 12.1 装备分类

| 装备 | 作用 |
|---|---|

| 以太火炮 | 远程爆发武器 |

| 机动背包 | 冲刺、悬浮、短飞行 |

## 12.2 武装模块槽

装备可以模块化升级。

```text
核心槽：决定装备等级
源质槽：决定属性
稳定槽：降低过载风险
输出槽：增加伤害
辅助槽：增加特殊效果
```

例子：

```text
以太长剑 + 火源质核心 = 炽焰斩
以太长剑 + 雷源质核心 = 雷鸣斩
以太长剑 + 土源质核心 = 重岩斩
以太长剑 + 光源质核心 = 净化斩
```

## 12.3 过载机制

以太武装不能无脑使用。

```text
连续使用 → 以太温度上升
温度过高 → 武器失效
稳定度过低 → 反噬玩家
源质属性冲突 → 产生畸变
```

# 十八、和 GT / AE2 整合思路

如果要把它塞进 GT 科技包，建议不要让本 Mod 完全独立，而是让它成为 GT 的一条特殊分支。

## 18.1 和 GT 的整合

| GT 系统 | 本 Mod 对接 |
|---|---|
| 电压等级 | 机器阶段 |
| 化学反应釜 | 魔药反应釜高级配方 |
| 离心机 | 源质属性分离 |
| 真空冷冻机 | 稳定源质结晶 |
| 装配线 | 高级以太武装 |
| 多方块 | 文明火种终端、以太反应炉 |

## 18.2 和 AE2 的整合

AE2 可以管理材料，但不应轻易自动化所有升华。

可以设计：

```text
普通源质材料：可 AE2 存储
魔药：可 AE2 合成
器官培育：需要专用机器
晋升仪式：不可完全自动化，需要玩家参与
灵能解析：需要玩家本人在场
```

这样可以避免系统被 AE2 完全扁平化。

## 18.3 自动化边界

| 内容 | 是否可自动化 |
|---|---|
| 源质萃取 | 可以 |
| 魔药基础合成 | 可以 |
| 稳定剂生产 | 可以 |
| 器官培养 | 半自动 |
| 玩家晋升 | 不可完全自动 |
| 灵能解析 | 玩家参与 |
| Boss 材料 | 不可量产或难量产 |

# 二十六、平衡设计

## 26.1 玩家强度控制

不能让玩家第一能级就碾压原版。

建议：

```text
第一能级：略强于铁套 / 钻石剑阶段
第二能级：强于钻石套，但仍怕 Boss
第三能级：接近后期装备
以太武装：强，但需要能源和维护
```

## 26.2 技能消耗

技能要有三重限制：

```text
源质消耗
冷却时间
精神负荷或稳定度风险
```

例如：

```text
岩盾：
消耗 30 源质
冷却 8 秒
持续 6 秒

雷光步：
消耗 20 源质
冷却 5 秒
精神负荷 +2

潮汐恢复：
消耗 50 源质
冷却 15 秒
恢复 4 点生命
```

## 26.3 魔药成功率

成功率由这些因素决定：

```text
素材品质
源质纯度
稳定剂等级
玩家路线兼容
当前畸变值
机器等级
灵能辅助
```

公式可以简单写成：

```text
最终成功率 =
基础成功率
+ 素材品质加成
+ 稳定剂加成
+ 机器等级加成
- 畸变惩罚
- 路线冲突惩罚
```

# 二十七、风险与解决方案

## 27.1 风险一：系统太大，做不完

解决：

```text
第一版只做 3 条路线
只做到第一能级
只做 3 个怪物
只做 2 台机器
不要一开始做遗迹和灵能全套
```

## 27.2 风险二：设定复杂，玩家看不懂

解决：

```text
加一本指南书
JEI 显示完整流程
升华者面板显示下一步目标
魔药 tooltip 显示成功率和风险
```

## 27.3 风险三：和科技 Mod 冲突

解决：

```text
能源系统可配置
配方可 CraftTweaker 修改
源质不要直接等价 RF/EU
晋升需要玩家参与，避免全自动破坏平衡
```

## 27.4 风险四：战斗技能太强

解决：

```text
技能消耗源质
技能有冷却
源质恢复受器官和环境限制
高级技能增加精神负荷
过度使用导致畸变
```

## 27.5 风险五：版权问题

解决方式：

```text
不要直接使用小说专有角色、剧情、地名、完整原文设定。
可以做“受启发的源质升华体系”。
路线、器官、怪物、地名尽量原创化。
如果是私人整合包可以更接近原作。
如果公开发布，建议改名和重构设定。
```

# 三十、最终推荐方案

最推荐的整体路线是：

```text
第一阶段：升华者职业 + 魔药 + 源质
第二阶段：魔兽生态 + 器官成长
第三阶段：机器链 + 以太科技
第四阶段：遗迹探索 + 灵能解析
第五阶段：以太武装 + 高能级 Boss
第六阶段：GT / AE2 整合包适配
```

不要一开始就做“大而全”。

先把这个核心闭环做出来：

```text
魔兽素材 → 源质 → 魔药 → 升华 → 技能 → 更强魔兽
```

等这个闭环好玩了，再接：

```text
遗迹 → 蓝图 → 以太机器 → 以太武装 → 高能级晋升
```

这样这个 Mod 才不会变成一堆设定，而是一个真正能玩的系统。
