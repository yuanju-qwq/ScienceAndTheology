# 模拟与事件系统设计

## 1. 概述

模拟层是连接 C++ 核心游戏逻辑与 Godot 渲染层的桥梁。核心组件：`TickSystem`（统一调度器）、`EventBus`（发布/订阅事件）、`MachineSystem`（机器子系统）、`StateSync`（状态增量同步），以及 `GDTickSystem`（Godot 绑定 + EventBus→Signal 桥接）。

命名空间 `science_and_theology`，位于 `src/core/simulation/`（12 个文件）。

## 2. TickSystem（统一模拟调度器）

### 2.1 核心参数

| 参数 | 值 | 说明 |
|------|-----|------|
| 模拟频率 | 20 TPS | 每秒 20 ticks |
| 活跃 Chunk 半径 | 4（默认） | Manhattan 距离 |
| 休眠 tick 间隔 | 20 ticks | 每 1 秒 tick 一次休眠 Chunk |

### 2.2 Tick 划分

```cpp
class TickSystem {
    WorldData* world_data_;
    std::unique_ptr<EventBus> event_bus_;
    std::unique_ptr<ErrorHandler> error_handler_;
    std::unique_ptr<StateSyncServer> state_sync_;
    std::vector<std::unique_ptr<SimulationSystem>> subsystems_;
};
```

### 2.3 每帧流程

```
tick(delta)
├── 1. rebuild_chunk_sets()
│     ├── 计算 player_cx, player_cy Manhattan 半径内的 Chunk → active_chunks_
│     └── 其余已加载 Chunk → sleeping_chunks_
├── 2. 按优先级排序 subsystems (0=Power → 1=Machine → 2=Fluid → 3=Logistics → 4=Region)
│     ├── for each subsystem: tick_active()  — 遍历 active_chunks_
│     └── for each subsystem: tick_sleeping() — 仅 should_tick_sleeping(cx,cy) 的 Chunk
├── 3. event_bus_->process_queue()  — 双缓冲排出延迟事件
└── 4. state_sync_->set_tick_counter(tick_counter_++)
```

### 2.4 休眠 Chunk 调度

`should_tick_sleeping(cx, cy)` 使用空间哈希将 Chunk 分散到不同帧：
```
(sleep_frame = (abs(cx) * 997 + abs(cy) * 1013) % kSleepTickInterval)
tick 当 tick_counter % kSleepTickInterval == sleep_frame
```

## 3. EventBus（事件总线）

### 3.1 设计

线程安全的 pub/sub 系统，支持两种分发模式：

```cpp
class EventBus {
    using Handler = std::function<void(const GameEvent&)>;
    using HandlerId = uint64_t;

    HandlerId subscribe(GameEventType type, Handler handler);
    void unsubscribe(HandlerId id);

    void emit(const GameEvent& event);      // 立即同步分发（调用线程上执行）
    void enqueue(const GameEvent& event);   // 延迟排队（线程安全）
    void process_queue();                   // 排出延迟队列（双缓冲）
    size_t queue_size() const;
};
```

### 3.2 延迟队列双缓冲

```
enqueue → deferred_queue_ (mutex 保护)
process_queue():
  1. swap(deferred_queue_, processing_queue_)
  2. 遍历 processing_queue_ → emit(event) → 各 handler 执行
  3. processing_queue_.clear()
  # 在 handler 执行期间，新的 enqueue 进入新的 deferred_queue_，不会死锁
```

### 3.3 GameEvent 类型体系

```cpp
enum class GameEventType : uint32_t {

    CHUNK_LOADED       = 200,
    CHUNK_UNLOADED     = 201,

    // 实体事件 (300-302)

    // Connector 事件 (400-402)
    CONNECTOR_ACTIVATED = 400,
    CONNECTOR_LOCKED    = 401,
```

## 4. EventBus → Godot Signal 桥接（已实现）

### 4.1 架构

`GDTickSystem` 在 `set_world_data()` 后自动调用 `subscribe_to_event_bus()`，将 8 种核心 EventBus 事件桥接到 Godot Signal：

```
EventBus (C++ core)
  └── subscribe(GameEventType, lambda)
      └── lambda → emit_signal("signal_name", args...)
          └── GDScript: tick_sys.signal_name.connect(_handler)
```

### 4.2 已注册的 Godot Signal

| Signal | 参数 | 触发条件 |
|--------|------|----------|
| `machine_state_changed` | machine_id: int, old_state: int, new_state: int | Machine::tick() 状态变更 |
| `machine_recipe_completed` | machine_id: int, recipe_name: String | Machine::try_complete() 成功 |
| `machine_error` | machine_id: int, error_code: String, message: String | Machine 发生错误 |
| `power_overload` | node_id: int, demand: int, capacity: int | PowerNetwork 检测到过载 |
| `chunk_generated` | layer: String, cx: int, cy: int | TerrainGenerator 生成完成 |
| `chunk_state_changed` | layer: String, cx: int, cy: int, old_state: int, new_state: int | ChunkData 状态变更 |
| `entity_created` | entity_id: int, type_name: String, cx: int, cy: int | 新实体注册 |
| `entity_destroyed` | entity_id: int, layer: String | 实体销毁 |

### 4.3 订阅生命周期

```cpp
void GDTickSystem::set_world_data(gd_world) {
    unsubscribe_from_event_bus();     // 解绑旧的
    tick_system_ = new TickSystem(ptr);
    subscribe_to_event_bus();         // 绑定新的
}

GDTickSystem::~GDTickSystem() {
    unsubscribe_from_event_bus();     // 析构时自动解绑
}
```

### 4.4 GDScript 用法

```gdscript
@onready var tick_sys: GDTickSystem = $GDTickSystem

func _ready():
    tick_sys.machine_error.connect(_on_machine_error)
    tick_sys.chunk_generated.connect(_on_chunk_generated)
    tick_sys.power_overload.connect(_on_power_overload)
    tick_sys.entity_created.connect(_on_entity_created)

func _on_machine_error(machine_id: int, error_code: String, message: String):
    show_popup("Machine %d: %s" % [machine_id, message])

func _on_chunk_generated(layer: String, cx: int, cy: int):
    refresh_chunk_view(layer, cx, cy)

func _on_power_overload(node_id: int, demand: int, capacity: int):
    play_explosion_effect(get_node_position(node_id))
```

### 5.1 机器注册表

```cpp
    void unregister_machine(MachineId id);

```

### 5.2 回调注入

`MachineSystem` 在注册 Machine 时注入三个回调：

```cpp
machine->set_state_changed_callback([this](MachineId id, MachineState from, MachineState to) {
    event_bus_->emit(GameEvent::machine_state_changed(id.id, (int)from, (int)to));
});

machine->set_recipe_completed_callback([this](MachineId id, const char* name) {
    event_bus_->emit(GameEvent::machine_recipe_completed(id.id, name));
});

machine->set_error_callback([this](MachineId id, const char* code, const char* msg) {
    event_bus_->emit(GameEvent::machine_error(id.id, code, msg));
});
```

### 5.3 活跃 Chunk Tick

遍历 `active_chunks_` → 获取 ChunkData.machines → 对每个 Machine 调用 `tick()`。Machine 内部状态机处理配方匹配、超频、进度推进。

## 7. StateSync（状态增量同步）

### 7.1 架构

```
C++ Sim Core (TickSystem)
    └── StateSyncServer
        ├── mark_dirty(chunk)  // 标记脏 Chunk
        ├── compute_delta(keys) // 计算增量
        └── create_snapshot(key) // 完整快照

GDScript Rendering (Proxy)
    └── StateSyncClient
        ├── apply_delta(delta) // 应用增量 → WorldData
        └── render proxies
```

### 7.2 SyncFlags（脏标志位）

```cpp
enum class SyncFlags : uint32_t {
    TERRAIN          = 0x01,
    MACHINE_STATE    = 0x02,
    MACHINE_INVENTORY = 0x04,
    POWER_GRID       = 0x08,
    FLUID_NET        = 0x10,
    ITEM_NET         = 0x20,
    CONNECTOR        = 0x40,
    ENTITY           = 0x80,
};
```

### 7.3 StateDelta

```cpp
struct StateDelta {
    SyncFlags flags;
    int64_t timestamp;
    std::vector<ChunkKey> chunks_modified;
    std::vector<EntityId> entities_created;
    std::vector<EntityId> entities_destroyed;
    std::vector<std::pair<EntityId, int>> machine_state_changes;
    // ... 可扩展
};
```

### 7.4 GDScript 接口

```gdscript
# 每帧获取脏 Chunk 列表
var dirty = tick_sys.get_dirty_chunks()
# dirty = [{layer: "surface", cx: 0, cy: 0}, ...]

# 计算增量

# delta = {flags: int, chunks_modified: Array, entities_created: Array, ...}

# 创建完整快照
var snapshot = tick_sys.create_snapshot("surface", 0, 0)
```

## 8. GDTickSystem（Godot 绑定）

### 8.1 类结构

```cpp
class GDTickSystem : public godot::Node {
    GDCLASS(GDTickSystem, godot::Node)

    std::unique_ptr<TickSystem> tick_system_;
    godot::Resource* gd_world_data_;
    std::vector<EventBus::HandlerId> event_subscriptions_;

    // 绑定方法
    void set_world_data(gd_world);
    void register_machine_system();
    void tick(delta);
    void set_player_chunk(layer, cx, cy);

    // 状态查询
    int64_t get_active_radius();
    void set_active_radius(radius);
    int64_t get_tick_count();
    int64_t get_active_chunk_count();
    godot::Array get_active_chunks();

    // 事件（已弃用）
    godot::Array poll_events();

    // 错误
    godot::Array get_machine_errors();
    void clear_machine_error(machine_id);

    // StateSync
    godot::Array get_dirty_chunks();
    godot::Dictionary compute_delta(chunk_keys);
    godot::Dictionary create_snapshot(layer, cx, cy);
};
```

### 8.2 GDScript 典型用法

```gdscript
# 初始化
@onready var tick_sys = GDTickSystem.new()
var world = GDWorldData.new()
world.seed = 12345

func _ready():
    tick_sys.set_world_data(world)
    tick_sys.register_machine_system()
    tick_sys.set_player_chunk("surface", 0, 0)
    tick_sys.machine_error.connect(_on_error)
    # ... 其他信号

func _process(delta):
    tick_sys.tick(delta)
    tick_sys.set_player_chunk(player.layer, player.cx, player.cy)

    # StateSync
    for chunk in tick_sys.get_dirty_chunks():
        var delta = tick_sys.compute_delta([chunk])
        update_chunk_view(delta)

    # 或使用 process_async_results
    world.process_async_results()

# 错误 UI
func _on_error(machine_id, code, msg):
    $ErrorPanel.show_error(machine_id, msg)
```

## 9. Region Graph [未实现]

> 以下来自原设计文档。

`RegionGraph` 类用于管理空间区域划分，计划支持：
- **电网区域**（Power Grid Regions）— 每个变压器隔离的子网络为一个区域
- **流体区域**（Fluid Regions）— 按管道连通性划分
- **连通区域**（Connected Regions）— BFS 可达的子图
- **污染扩散**（Pollution Spread）— 以区域为单位的扩散模型
- **温度系统**（Temperature System）— 区域级温度计算

`SimulationSystem` 优先级 4 预留给 `RegionSystem`。
