# 玩家道具系统设计

## 1. 概述

为"科学与神学"2D工厂自动化+探索游戏设计完整的玩家道具系统，支持工具使用（挖掘/砍伐）、武器战斗、装备穿戴、物品栏管理。系统跨 C++ 核心层、GDExtension 绑定层、GDScript 表现层三层架构实现。

### 1.1 交互模型

| 操作 | 功能 |
|------|------|
| WASD | 角色移动 |
| 鼠标位置 | 自动决定角色朝向（player 旋转面向鼠标） |
| 鼠标左键 | 使用手持工具（挖掘/攻击/砍伐） |
| 鼠标右键 | 上下文交互（打开机器界面/工作台/拾取物品） |
| 数字键 1-9 | 切换快捷栏当前槽位 |
| Tab | 打开/关闭背包界面 |
| E (保留) | 保留原 Connector/Mechanism 交互通道 |

角色朝向始终跟随鼠标，移动完全独立（WASD）。左键触发工具使用，右键根据上下文打开 GUI 或拾取物品。

#### tool_def.hpp — 工具类型与属性定义

```cpp
enum class ToolType : uint8_t {
    NONE = 0, PICKAXE, AXE, SHOVEL, SWORD
};

struct ToolStats {
    ToolType type;           // 工具类型
    uint8_t tier;            // 材质等级 0=木 1=石 2=铁 3=钻
    float speed_multiplier;  // 速度倍率
    uint8_t mining_level;    // 可挖掘的材质等级
    float attack_damage;     // 攻击伤害
    float attack_cooldown;   // 攻击冷却(秒)
    int32_t max_durability;  // 最大耐久
    uint8_t range;           // 作用范围(默认1格)

    static ToolStats default_for(ToolType type, uint8_t tier);
};
```

#### inventory.hpp/.cpp — 物品栏系统

```cpp
struct InventorySlot {
    ItemId item_id = kInvalidItemId;
    int32_t count = 0;
    int32_t durability = -1;  // -1=不适用耐久, ≥0=当前耐久
};

class Inventory {
    // 默认网格: 9列 × 4行 = 36格
    // 行0 = 快捷栏, 行1-3 = 主背包
    int32_t width() / height() / slot_count();
    const InventorySlot& get_slot(index);
    bool set_slot(index, item_id, count, durability);
    int32_t add_item(item_id, count, durability);  // 返回未添加数量
    bool remove_from_slot(index, count);
    void swap_slots(a, b);
    bool split_stack(src_index, dst_index);        // 半拆分堆叠
    int32_t count_item(item_id);
    int32_t find_item(item_id);
    bool has_enough(item_id, count);
    void clear();
};
```

核心逻辑：
- **add_item**：优先叠加已有同种物品堆，再填充空格子，返回溢出数量
- **split_stack**：从源格子取一半数量到目标空格子，耐久复制
- **最大堆叠**：默认64，可由 max_stack 属性调整

### 5.3 PlayerInteraction.gd（新建 — 核心）

```gdscript
class_name PlayerInteraction
extends Node

# 信号
signal tool_used(tool_id: int, target_cell: Vector2i)
signal machine_opened(machine_id: int)
signal crafting_opened()

# 变量
@export var interaction_range: int = 4  # 最大交互距离（格）
var _tool_cooldown: Timer
var _mining_progress: float = 0.0
var _current_mining_cell: Vector2i = Vector2i.ZERO
```

#### 左键 — 工具使用流程

```
_input(event):
  左键按下 → _try_use_tool()
    1. 获取鼠标指向的 cell 坐标
    2. 获取手持工具 (from GDPlayerEquipment.get_equipped(MAIN_HAND))
    3. 获取工具属性 (from ToolRegistry)
    4. 检查距离 (player_cell → target_cell ≤ tool.range)
    5. 查询 GDWorldData.get_terrain_cell(material, flags, is_mineable)
    6. 判断交互类型:
       - is_mineable && tool.type == PICKAXE → 开始挖掘
       - is_mineable && tool.type == AXE → 开始砍伐
       - is_mineable && tool.type == SHOVEL → 开始挖掘(软质)
       - tool.type == SWORD → 检查目标是否有实体，攻击
       - 目标是空气 && 手持方块 → 放置方块
       - 空手 → 无效果（可扩展拳击）
    7. 启动冷却 Timer + 进度条 + 动画
```

#### 挖掘进度流程

```
挖掘启动:
  基础时间 = terrain_hardness[material]  # 不同材质不同硬度
  实际时间 = 基础时间 / tool.speed_multiplier
  Timer.wait_time = 实际时间
  Timer.start()

每帧 Update:
  _mining_progress = (实际时间 - Timer.time_left) / 实际时间
  更新进度条显示

Timer timeout:
  old_material = cell.material
  GDWorldData.set_terrain_cell(layer, cx, cy, x, y, MAT_AIR)
  EventBus emit TERRAIN_CHANGED → TerrainVisualUpdater 更新 TileMapLayer
  生成掉落物 → GDWorldData 创建 ItemDrop entity
  扣除工具耐久 (inventory slot durability -= 1)
  如果耐久归零 → 工具破损消失
```

#### 右键 — 上下文交互

```
_input(event):
  右键按下 → _try_context_interact()
    1. 获取鼠标指向的 cell
    2. 检查该 cell 附近是否有:
       - Machine entity → emit machine_opened(machine_id)
       - Workbench entity → emit crafting_opened()
       - ItemDrop entity → 拾取物品
       - Connector (已由 E 键处理)
    3. 无匹配 → 无效果
```

- 序列化字段：`{entity_id, item_id, count, cell_x, cell_y, world_x, world_y, spawn_time_ms}`
- 加载时恢复为 ItemDrop 场景实例

## 8. 与现有系统的集成点

### 8.1 材料系统

- 工具配方通过现有 `CraftingGrid` 注册（木镐 = `[木板, 木板, 木板]` + `[空, 木棍, 空]` + `[空, 木棍, 空]`）
- 工具物品使用已分配的 `kNonMaterialItemBase + 36~51` ItemId
- `ItemRegistry` 可通过 `get_item_display_name()` 查询工具名称

### 8.2 存档系统

- 玩家物品栏 → 独立 `.inv` 文件序列化（与 chunk 文件分离）
- 装备槽 → 与物品栏共用同一个存档结构
- ItemDropData → 扩展 `ChunkSerializer` 序列化

### 8.3 TickSystem

- 掉落物需要 tick：5分钟生命周期 + 同类合并检测 + 玩家拾取范围检测
- 可作为新的 TickSystem 子系统注册

### 8.4 AE2 存储网络

- 玩家物品栏是独立维度，不直接加入 ME 网络
- 未来通过终端（AE Terminal）实现物品栏 ↔ ME 网络的双向传输
- 工具耐久在物品栏中跟踪，放入 ME 网络时保留耐久信息
