# 玩家道具系统设计

## 1. 概述

玩家道具系统服务于当前 **3D 方块/体素工厂探索游戏**：玩家在第一人称/近第一人称视角中移动、选择方块、挖掘、放置、打开机器 UI，并通过背包、快捷栏和装备系统管理工具与材料。

系统跨三层：

- C++ 核心层：物品栏、装备槽、工具属性、地形/方块修改 API、事件。
- GDExtension 绑定层：`GDPlayerInventory`、`GDPlayerEquipment`、`GDWorldData`、`GDGameCommandServer`。
- GDScript 表现层：`PlayerController.gd`、快捷栏/背包/合成/熔炉 UI、3D 世界对象渲染。

## 2. 当前交互模型

| 操作 | 功能 |
|---|---|
| 鼠标移动 | 控制玩家 yaw 和相机 pitch |
| WASD / 方向键 | 水平移动 |
| Shift | 冲刺 |
| Space | 跳跃；当前也复用为交互入口之一 |
| 鼠标左键 | 对准方块后提交 `mine_block` |
| 鼠标右键 / E | 打开对象或放置手持对象 |
| 数字键 1-9 | 切换快捷栏槽位并装备到主手 |
| 鼠标滚轮 | 切换快捷栏 |
| Tab / I / Escape | 鼠标捕获、背包/界面切换 |
| C | 合成界面 |
| B | 百科界面 |

`PlayerController.gd` 通过 `Camera3D` 向前做物理射线，命中方块后计算：

- `cell`：被命中的方块坐标。
- `chunk/local`：提交给当前兼容 `GDWorldData` API 的 chunk 坐标。
- `place_cell`：沿命中法线外推得到的放置位置。
- `expected_material`：提交命令时用于服务端校验，避免客户端选中状态过期。

## 3. 架构总览

```text
┌─ GDScript 表现层 ────────────────────────────────────────┐
│ PlayerController3D, HotbarUI, InventoryUI, CraftingUI    │
│ FurnaceUI, WorldObjectRenderer, target/selection UI      │
├─ 命令与同步层 ───────────────────────────────────────────┤
│ GameCommandServer.gd -> GDGameCommandServer              │
│ mine_block / place_object / craft_recipe / furnace_*     │
├─ GDExtension 绑定层 ─────────────────────────────────────┤
│ GDPlayerInventory, GDPlayerEquipment, GDWorldData        │
├─ C++ 核心层 ─────────────────────────────────────────────┤
│ player/inventory, player/equipment, player/tool_def      │
│ TerrainData set/get cell, EventBus, ItemRegistry         │
└─────────────────────────────────────────────────────────┘
```

当前仍有一部分世界对象状态在 GDScript manager 中：`WorkbenchManager`、`FurnaceManager`。梯子已迁移为地形方块（`snt:ladder`，`TF_CLIMBABLE`），不再由独立 manager 管理。新增服务端状态应优先下沉到 C++ block entity，不要继续扩大 GDScript manager 的权威职责。

## 4. C++ 核心层

### 4.1 工具定义

```cpp
enum class ToolType : uint8_t {
    NONE = 0,
    PICKAXE,
    AXE,
    SHOVEL,
    SWORD
};

struct ToolStats {
    ToolType type;
    uint8_t tier;
    float speed_multiplier;
    uint8_t mining_level;
    float attack_damage;
    float attack_cooldown;
    int32_t max_durability;
    uint8_t range;
};
```

工具规则：

- 镐：石头、矿石、机器外壳等硬质方块。
- 斧：木头、树叶、部分魔法植物方块。
- 铲：泥土、沙子、黏土、砾石等软质方块。
- 剑：实体战斗，后续可对部分生物/魔法目标触发特殊效果。

### 4.2 物品栏

```cpp
struct InventorySlot {
    ItemId item_id = kInvalidItemId;
    int32_t count = 0;
    int32_t secondary_id = -1;
};
```

规则：

- `secondary_id == -1` 的普通物品按 `item_id` 堆叠。
- `secondary_id != -1` 的工具/魔法书/附魔装备不堆叠。
- 背包尺寸当前为 `9 x 4`，前 9 格作为快捷栏。
- `GDPlayerInventory` 暴露 `get_slot`、`set_slot`、`add_item`、`remove_from_slot`、`swap_slots`、`split_stack` 等方法。

### 4.3 装备槽

```cpp
enum class EquipmentSlot : uint8_t {
    MAIN_HAND = 0,
    OFF_HAND,
    HEAD,
    CHEST,
    LEGS,
    FEET,
    COUNT
};
```

当前快捷栏切换会把选中槽物品装备到 `MAIN_HAND`，命令服务器用主手物品校验工具 tag、挖掘等级和掉落规则。

## 5. 当前 GDScript 表现层

### 5.1 PlayerController.gd

当前 `PlayerController` 已经是 `CharacterBody3D`，负责：

- 鼠标捕获与相机旋转。
- WASD/跳跃/冲刺移动。
- 初始化背包与装备。
- 连接快捷栏、背包、合成、熔炉、百科 UI。
- 维护当前射线目标和 `SelectionBox`。
- 左键提交 `mine_block`。
- 右键依次尝试打开熔炉、使用连接点/机关、放置世界对象。
- 处理 `inventory_synced` 后刷新 UI。

关键射线逻辑：

```gdscript
var from := camera.global_position
var to := from + (-camera.global_transform.basis.z * REACH)
var query := PhysicsRayQueryParameters3D.create(from, to)
var hit := get_world_3d().direct_space_state.intersect_ray(query)
```

命中后用 `ChunkRendererBridge.world_position_to_cell()` 转为当前兼容方块坐标。

### 5.2 挖掘流程

```text
左键
├── Camera3D 射线命中方块
├── 读取 material / chunk / local / cell
├── submit_command({
│     type = "mine_block",
│     dimension = &"overworld",
│     chunk = Vector3i(cx, cy, cz),
│     local = Vector3i(lx, ly, lz),
│     cell = Vector3i(x, y, z),
│     expected_material = material
│   })
├── GDGameCommandServer 校验工具、目标材质、掉落和背包
├── 服务端写 GDWorldData.set_terrain_cell(..., air)
└── emit terrain_cell_synced / inventory_synced
```

短期挖掘是即时生效。后续需要增加：

- 挖掘进度和硬度计时。
- 方块裂纹/粒子/音效。
- 工具耐久消耗。
- 服务端按距离和视线重复校验。
- 对 3D `VoxelCoord(x,y,z)` 的原生支持。

### 5.3 放置与交互流程

右键/E 当前优先级：

1. 如果目标附近有熔炉，打开 `FurnaceUI`。
2. 如果当前位置有连接点或机关，执行交互。
3. 如果主手是工作台、熔炉或梯子，提交 `place_object`。

```text
place_object
├── object_type: workbench / furnace / ladder
├── layer: 当前兼容 layer
├── cell: place_cell
└── item_id: 主手物品
```

服务端校验背包数量和占位冲突，写入对应 manager，并同步 UI。

## 6. UI 层

### 6.1 HotbarUI

快捷栏目标：

- 9 格横排，底部居中。
- 当前选中槽高亮。
- 显示物品图标、数量、耐久条。
- 数字键、滚轮、点击均可切换。
- 切换后同步主手装备。

### 6.2 InventoryUI

背包目标：

- `9 x 4` 网格。
- 前 9 格与快捷栏一致。
- 支持拖拽交换、拆分堆叠、Shift 快速移动。
- 装备槽和后续饰品/魔法书槽可以接入同一物品选择逻辑。

### 6.3 目标与提示 UI

当前已有准星、目标名 `TargetLabel`、`SelectionBox`、连接点提示。后续可增加：

- 方块硬度/挖掘进度条。
- 工具不足提示。
- 方块信息、矿脉信息、机器端口提示。
- 可交互面的图标提示。

## 7. 世界对象与掉落

### 7.1 世界对象

当前可放置对象：

| 对象 | 物品 | 当前状态 |
|---|---|---|
| 工作台 | `ITEM_WORKBENCH` | GDScript manager + 3D box model |
| 熔炉 | `ITEM_FURNACE` | GDScript manager + FurnaceUI + 3D box model |
| 梯子 | `ITEM_LADDER` | 地形方块（`snt:ladder`，`TF_CLIMBABLE`）+ 攀爬移动 |

目标是迁移为 C++ block entity：

```cpp
struct BlockEntityPlacement {
    EntityId id;
    std::string block_key;
    int x, y, z;
    uint8_t facing;
};
```

### 7.2 掉落物

当前 `mine_block` 可以直接把掉落加入玩家背包。后续需要实体化掉落：

- 3D `Area3D` 拾取范围。
- `MeshInstance3D` 或 billboard icon 表现。
- 同类合并、生命周期、磁吸拾取。
- 服务端保存掉落实体，客户端只渲染同步结果。

## 8. 与 AE2 / 工业 / 魔法的兼容

### 8.1 AE2 Secondary Data

`secondary_id` 是通用二次数据字段：

- 工具耐久。
- 魔法书法术配置引用。
- 附魔/祝福状态。
- 染色、品质、稀有度。

AE2 存储网络按 `(item_id, secondary_id)` 精确区分物品；配方匹配可通过 `drop_secondary()` 做模糊匹配。

### 8.2 工业系统

工具和机器物品应使用稳定 item key，不在 GDScript 中散落魔法数字。放置机器时要进入命令服务器，由服务端检查：

- 背包是否有对应物品。
- 放置方块是否为空。
- 目标面是否允许附着。
- 多方块结构是否占位冲突。
- 是否满足扳手/工具/权限要求。

### 8.3 魔法系统

魔法书可复用装备槽或快捷栏触发。地形改造、范围采矿、传送和机器祝福必须走服务端命令，不能让客户端法术脚本直接写世界。
