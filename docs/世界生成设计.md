# 2D 工厂游戏新世界架构设计文档

## 文档目标

本文档定义：

* 新世界生成系统
* Chunk Streaming 架构
* 数据驱动世界系统
* C++ Core 与 Godot 分层
* Simulation 与 Rendering 解耦

目标：

构建一个适用于：

* 大地图
* 自动化
* 电力网络
* 流体系统
* 物流系统
* 无限世界

的 2D 工厂游戏底层架构。

---

# 一、核心设计目标

---

# 1. 数据驱动世界

世界本体：

```text
WorldData
```

Godot 节点：

```text
仅作为渲染代理
```

禁止：

```text
Node = 世界实体
```

---

# 2. Chunk Streaming

世界按 Chunk 分块：

```text
16x16
32x32
64x64
```

支持：

* 动态加载
* 动态卸载
* 异步生成
* 无限地图

---

# 3. Simulation 与 Rendering 解耦

Simulation：

```text
C++ Core
```

Rendering：

```text
Godot
```

Godot 不负责：

* 电网
* Tick
* 物流
* 世界数据

Godot 只负责：

* 显示
* 输入
* UI
* 摄像机

---

# 4. 面向大型工厂游戏扩展

架构必须支持：

* 数十万 Tile
* 数万 Machine
* 大规模物流
* Region Graph
* 多线程生成
* Save/Load
* Replay
* Multiplayer

# 三、世界数据架构

---

# WorldData

世界唯一数据入口。

```cpp
class WorldData {
public:
    ChunkData& get_chunk(int x, int y);

    EntityId create_entity();

    void destroy_entity(EntityId id);

    MachineData* get_machine(EntityId id);

private:
    std::unordered_map<ChunkCoord, ChunkData> chunks;
};
```

---

# ChunkData

Chunk 是：

```text
世界数据分片
```

而不是：

```text
Godot 节点
```

```cpp
struct ChunkData {
    int chunk_x;
    int chunk_y;

    ChunkState state;

    TerrainData terrain;

    std::vector<EntityId> entities;

    std::vector<MachineId> machines;

    std::vector<ConnectorId> connectors;
};
```

---

# ChunkState

```cpp
enum class ChunkState {
    UNLOADED,
    GENERATING,
    GENERATED,
    ACTIVE,
    SLEEPING
};
```

---

# 状态说明

| 状态         | 说明                |
| ---------- | ----------------- |
| UNLOADED   | 未加载               |
| GENERATING | 正在异步生成            |
| GENERATED  | 已生成但未激活           |
| ACTIVE     | 玩家附近完整 Simulation |
| SLEEPING   | 远距离低频 Simulation  |

---

# TerrainData

TerrainData 不包含渲染信息。

只保存：

```text
世界材料信息
```

```cpp
struct TerrainCell {
    TerrainMaterial material;
    uint32_t flags;
};

struct TerrainData {
    int size_x;
    int size_y;

    std::vector<TerrainCell> cells;
};
```

---

# TerrainMaterial

禁止：

```cpp
FLOOR
WALL
```

这种渲染语义。

改为：

```cpp
enum class TerrainMaterial {
    AIR,
    STONE,
    DIRT,
    SAND,
    WATER,
    LAVA,

    ORE_IRON,
    ORE_COPPER,
    ORE_COAL
};
```

---

# TerrainFlags

```cpp
enum TerrainFlags {
    WALKABLE = 1 << 0,
    SOLID    = 1 << 1,
    LIQUID   = 1 << 2,
    MINEABLE = 1 << 3
};
```

# 六、Chunk Streaming 系统

---

# TerrainChunkManager

Godot 中：

```text
只负责 Chunk 可见性
```

不负责：

* 世界逻辑
* 电网
* Tick

---

# 核心职责

```text
1. 跟踪玩家位置
2. 计算可见 Chunk
3. 请求 Chunk 加载
4. 卸载远距离 Chunk
5. 创建 TerrainChunkView
```

---

# 推荐加载范围

```text
View Radius = 3~5 Chunk
```

例如：

```text
5x5 = 25 Chunk
```

---

# TerrainChunkView

这是：

```text
ChunkData 的渲染代理
```

---

# 职责

```text
1. TileMap 渲染
2. Object 渲染
3. Entity RenderProxy 创建
4. Chunk 可见性控制
```

---

# 禁止职责

```text
× 电网 Simulation
× Machine Tick
× 世界数据存储
× Connector 逻辑
```
