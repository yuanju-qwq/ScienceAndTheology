# 科学与神学 — 项目架构总览

## 1. 项目定位

一款 2D 工厂自动化 + 探索游戏，核心玩法受 GregTech 5（工业科技）和 Applied Energistics 2（智能存储与自动合成）启发；魔法系统参考「新生魔艺」(Ars Nouveau) 的符文组合理念，采用祭坛仪式玩法。世界探索部分借鉴 Terraria / Core Keeper 的双层互通地图设计。

技术栈：**Godot 4.6（渲染/表现层） + C++20 GDExtension（模拟核心）**。

## 6. 关键设计决策

### 6.1 引擎无关核心

`snt_core` 静态库不含任何 Godot 头文件依赖，可在纯 C++ 单元测试中独立编译运行。这意味着：
- **可移植**：核心逻辑可运行在无头服务器上
- **可测试**：不需要 Godot 编辑器即可运行单元测试
- **代价**：需要维护 GDExtension 绑定层作为桥梁

### 6.2 异步生成使用 Godot WorkerThreadPool

Chunk 异步生成使用 Godot 内置 `WorkerThreadPool::get_singleton()`，通过 `add_native_task()` 提交 C 函数回调到工作线程。结果通过 mutex 保护的队列返回主线程，在 `process_async_results()` 中消费并 emit `chunk_ready` 信号。

### 6.3 自定义噪声保留

自定义 `NoiseGenerator`（Perlin 分形噪声，~140 行）保留，原因：
- 纯 C++ 内联方法，零虚函数开销
- 栈上构造，天然线程安全（每个 chunk 生成线程创建独立实例）
- 与 Godot `FastNoiseLite` 相比，在异步工作线程上使用 Godot Object 子类存在线程安全风险

> 将来若需要 Cellular/Warp 等高级噪声，可引入 [FastNoiseLite 独立 C++ 头文件](https://github.com/Auburn/FastNoiseLite)（非 Godot 绑定版本）直接嵌入核心层。

### 6.4 适配器接口层（当前为桩）

`src/adapters/` 定义了 4 个纯虚接口（`IWorldAdapter` 等），用于将核心层与特定引擎解耦。当前所有接口仅包含虚析构函数，具体方法声明和实现待后续填充。

### 6.5 神学魔法系统与工业系统：弱依赖、强联动

神学魔法系统（`science_and_theology::magic`）与 GT 工业系统（`science_and_theology::gt`）采用**弱依赖、强联动**设计：

- **弱依赖**：两套体系互不阻塞，纯魔法或纯工业均可独立通关。核心代码不互相 `#include`。
- **强联动**：通过 `EventBus` 事件和 `MaterialRegistry` 物品体系间接交互，交汇点提供超额增益（但不获得也不卡关）。
- **手动/自动化分工**：符文可由机器量产（自动化），仪式必须玩家在祭坛前手动执行（手动）。
- **纯个人属性**：法力不设法力蓄电池方块，不接管道网络。玩家通过道具补充法力。

详见：[神学魔法系统设计](./神学魔法系统设计.md)
